## 0. 이번 단계의 결론

RO-RN은 `예의 바름`, `갈등을 피하는 정도`, `자기주장이 약한 정도`가 아니다.

```text
자신이 원하는 결과와 상대의 선택이 다를 때
→ 더 설득하거나 상대 자체를 문제 삼는 방향이 먼저 떠오르는지
→ 최근 비슷한 상황에서 실제로는 거절·선택 여지를 남기고 비하를 억제했는지
```

RO-RN의 핵심 점수 후보는 실제 나타나는 반응인 `ENACTED_RESPONSE`다. 설득하고 싶은 방향이나 상대 선택을 인정해야 한다는 방향이 먼저 드는지는 상세한 표현 차이를 설명하는 `PROFILE_MODIFIER`로 검증한다.

따라서 아래 사람을 구분할 수 있어야 한다.

> 더 설득할 방법은 떠오르지만, 상대가 스스로 결정할 권리를 고려해 실제로는 설득을 멈추는 사람

이 사람을 단순히 압박형 또는 순응형으로 분류하지 않는다.

## 2. 측정 목표와 금지 해석

### 측정하려는 것

- 제안이나 부탁을 거절당한 뒤 상대의 결정을 실제로 받아들이는 정도
- 공동 계획에서 상대에게 선택 여지를 남기는 정도
- 원하는 결과가 있어도 설득·압박을 이어가는 정도
- 의견 충돌이나 실수 지적에서 문제와 사람 자체를 구분하는 정도
- 설득 충동과 실제 존중 행동이 다르게 나타나는지

### 측정하지 않는 것

- 말을 부드럽게 하는 대화 기술
- 자신의 의견을 작게 말하거나 아예 말하지 않는 태도
- 갈등이 두려워서 물러나는 ER-WD
- 상대 감정이 걱정되는 RO-EC
- 먼저 의견을 말하는 SE-AI
- 설득력·리더십·협상 능력
- 모든 의견에 동의하거나 부탁을 받아주는 순응
- 폭력·자해·범죄·안전 위험 상황에서 필요한 개입

### 코드와의 관계

```text
검증된 RO-RN 실제 반응 문항
→ RO-RN 연속 점수와 측정오차
→ 본인 전용 선택·경계 존중 상세 신호
→ 대표 G/A와 독립된 개인 정밀 리포트 설명
```

RO-RN의 처음 드는 생각과 실제 나타나는 반응은 모두 G/A에 합산하지 않는다. 상대 감정을 먼저 확인하는 RO-EC 반응 modifier만으로도 RO-RN을 높게 확정하지 않는다. `본인 전용`은 사용자에게 숨긴다는 뜻이 아니라 공개 프로필·피드·공유·상대 비교에서 기본 제외한다는 뜻이다.

## 7. HIGH·LOW 비우열 계약

| 방향   | 허용되는 의미                                                                      | 금지되는 단정                                      |
| ------ | ---------------------------------------------------------------------------------- | -------------------------------------------------- |
| `HIGH` | 자신의 입장을 표현해도 상대의 거절·선택 여지를 두고 사람 자체에 대한 평가를 억제함 | 착함, 갈등을 잘 해결함, 배려 깊음, 자기주장이 약함 |
| `LOW`  | 원하는 결과가 중요할 때 설득을 더 이어가거나 갈등에서 사람 평가까지 넓어질 수 있음 | 솔직함, 추진력이 좋음, 강압적 인물, 관계에 위험함  |

LOW를 리더십·직접성의 장점으로 미화하지 않고, HIGH를 침묵·순응·갈등 회피와 같다고 보지 않는다.

## 8. pair 제시와 이유 probe 규칙

- 같은 pair의 A/B 문항은 최소 세 문항 이상 떨어뜨린다.
- 처음 드는 생각은 G/A나 RO-RN 점수에 자동 합산하지 않는다.
- 실제 반응 HIGH가 갈등 회피나 불안에서 나온 것인지 반응만으로 추론하지 않는다.
- 이유 probe는 층위 차이가 최소 2개 상황에서 같은 방향으로 반복될 때 검사 끝에 한 번만 제시한다.
- `답하고 싶지 않아요`와 `상황마다 달라요`를 허용한다.

### 설득 생각은 있지만 실제로 선택을 두는 패턴

```text
더 설득할 방법은 떠오르지만 실제로는 상대의 선택을 그대로 두는 데에는
어떤 이유가 가장 가까운가요?
```

- 상대가 스스로 정할 권리를 중요하게 생각해서
- 관계가 불편해지는 것을 피하고 싶어서
- 더 설명해도 결정이 달라지지 않을 것 같아서
- 내 입장보다 상대 상황을 우선해야 한다고 생각해서
- 상황과 사람에 따라 이유가 달라서
- 답하고 싶지 않아요

### 선택을 인정해야 한다고 생각해도 설득을 이어가는 패턴

```text
상대의 선택을 인정해야 한다고 생각해도 설득을 이어갈 때에는
어떤 이유가 가장 가까운가요?
```

- 결과가 중요하다고 느껴서
- 상대가 내 설명을 충분히 이해하지 못했다고 느껴서
- 내가 더 정확한 정보를 알고 있다고 생각해서
- 역할상 결정을 이끌어야 한다고 생각해서
- 상황과 사람에 따라 이유가 달라서
- 답하고 싶지 않아요

이유 응답은 `NON_SCORING_CONTEXT`다. 사용자가 선택한 이유만 `내가 직접 고른 이유`로 표시하며 코드·facet·비교 점수에는 사용하지 않는다.

## 9. 빠른·정밀 사용 원칙

### 빠른 코어

- 대표 5글자를 만드는 빠른 코어에는 RO-RN을 사용하지 않는다.
- 빠른 결과에서 선택·존엄 존중 수준이나 다층 조합을 확정하지 않는다.
- 별도 상세검사로 이동할 관심이 있는 사용자에게만 후속 경로를 안내한다.

### 정밀 코어

- 선택·거절 존중, 반복 설득 조절, 사람 평가 억제를 모두 포함한다.
- 실제 나타나는 반응 후보 약 6개를 최소 네 상황에서 비교한다.
- 처음 드는 생각 profile modifier는 최소 3개 유효 pair와 2개 반복 차이가 있을 때만 상세 해석 후보가 된다.
- HIGH 행동이 ER-WD 갈등 회피, LOW 행동이 SE-AI·역할 책임만 반영하지 않는지 검증한다.
- 사회적 바람직성 천장 효과와 성별·권력·관계 규범의 문항 기능 차이를 확인한다.

## 10. RO-EC와의 분리

| 관찰                                  | RO-EC                             | RO-RN                                     |
| ------------------------------------- | --------------------------------- | ----------------------------------------- |
| 상대가 속상할까 봐 마음을 먼저 확인함 | 첫 생각 또는 반응 modifier 후보   | 선택권 존중 증거가 없으면 점수화하지 않음 |
| 상대가 거절한 뒤 더 설득하지 않음     | 감정 관심을 뜻하지 않음           | 선택 존중 핵심 행동 후보                  |
| 걱정되어 계속 설득함                  | 공감적 관심이 높을 수 있음        | 선택 존중은 낮게 나타날 수 있음           |
| 사실부터 보지만 거절은 받아들임       | 공감적 관심은 낮은 방향일 수 있음 | 선택 존중은 높은 방향일 수 있음           |

두 신호가 다른 방향으로 나타나는 것을 오류로 보지 않는다. 다만 G/A와 RO-RN을 하나의 `SPLIT_PROFILE` 점수로 묶지 않고 대표 코드와 본인 전용 상세 설명에 각각 보존한다.

| 항목                              | 판정                                  |
| --------------------------------- | ------------------------------------- |

| RO-EC와의 G/A 합산                | `NO_GO_REBASELINE_REQUIRED`           |

연구에서 계속 검토할 핵심:

1. RO-RN 핵심 점수는 실제 나타나는 반응을 우선하고 처음 드는 생각은 profile modifier로 두는 구조
2. 네 주력 상황 P01–P04와 두 탐색 쌍 P05–P06
3. 설득하고 싶은 생각과 실제 존중 행동의 차이를 별도 상세 패턴으로 설명하는 원칙
4. 갈등 회피·포기·역할 책임 같은 행동 이유를 자동 추론하지 않는 원칙

후속 [RO 두 facet·다층 24문항 세트 감사](./NUANG_M03_RO_TWO_FACET_MULTILAYER_SET_BALANCE_AUDIT.md)에서 수량·방향·층위의 구조 균형은 통과했다. 그러나 RO-RN의 낮은 방향은 반복 압박·사람 평가를 포함해 가치중립적인 대표 G/A로 묶을 수 없고, 일부 문항은 도덕적 정답이 지나치게 선명하다. 사용자 승인 A안에 따라 이 후보는 수정·인지 인터뷰를 거친 본인 전용 상세 신호로 연구하며 대표 G/A 점수·글자·선명도에는 연결하지 않는다. 사용자 본인의 정밀 리포트 UI 연결은 검증 후 가능하지만 공개·공유·비교는 기본 제외한다.
