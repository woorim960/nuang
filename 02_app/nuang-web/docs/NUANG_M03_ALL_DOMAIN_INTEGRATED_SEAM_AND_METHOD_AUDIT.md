## 0. 결론

전 5영역 후보 은행은 기계적 기본 품질을 통과했다.

- 연구 후보 150개는 HIGH 75개·LOW 75개다.
- 완전히 같은 질문 문장은 없다.
- 영역 사이에 완전히 같은 상황 라벨도 없다.
- 각 일반 facet은 주력 8개·탐색 4개 구조를 유지한다.
- ER 임상·최근 상태 방화벽, RO 과정층, SM-RL 조건부 지위, RO-RN 비공개 지위가 서로 섞이지 않았다.

초기 감사에서는 대표 G/A의 주증거인 RO-EC `처음 드는 생각` 후보가 3G·3A뿐인 중복성 부족을 발견했다. 사용자가 `RBL-01E-R1` 보강 범위와 문구를 순차 승인했고, 공식 후보 은행 v0.5에 3G·3A를 추가해 6G·6A, 주력 8·탐색 4 구조를 완성했다. 실제 반응 6개는 계속 과정 보조층으로 분리한다.

따라서 현재 판정은 다음과 같다.

```text
PASS — 150개 전체 인벤토리·방향 균형·완전 중복 검사
PASS WITH TEST REQUIREMENTS — 교차 영역 seam·문구·상황 감사
PASS — G/A 주점수 후보 6G·6A·주력 8·탐색 4
NOT READY — 빠른·정밀 최종 문항, 운영 채점, 고객 코드 발급
```

2026-07-18 KST 사용자가 [RO-EC G/A 주점수 후보 보강](./NUANG_M03_RO_EC_CORE_REDUNDANCY_REPAIR.md)의 범위와 여섯 문구를 승인했다. 공식 후보 은행 병합과 자동 재감사를 통과했으므로 `RBL-01E-R1`, `RBL-01`, `ITEM_DESIGN_GATE_COMPLETE`를 닫고 S03-R UI/UX로 복귀한다.

## 2. 대표 코드 증거 밀도 감사

### 2.1 일반 후보 구조

SE·OE·SM·ER의 일반 facet은 각각 다음 구조다.

```text
facet별 후보 12개
= HIGH 6 + LOW 6
= 주력 8 + 탐색 4
```

이는 최종 12개를 모두 운영한다는 뜻이 아니다. 전문가 검토·인지 인터뷰·정량 파일럿에서 일부가 탈락하거나 크게 수정돼도 상황과 방향별 대체 후보를 남기기 위한 연구 풀이다.

### 2.2 RO의 현재 구조

RO-EC의 18개 문장은 두 층으로 나뉜다.

| 층                                          |  수 | 대표 G/A 주점수 | 상세 반응 흐름   |
| ------------------------------------------- | --: | --------------- | ---------------- |
| `REPORTED_FIRST_ORIENTATION` 처음 드는 생각 |  12 | 직접 후보       | 입력층 후보      |
| `ENACTED_RESPONSE` 실제 나타나는 반응       |   6 | 자동 합산 금지  | 보조·과정층 후보 |

실제 반응을 대표 G/A 후보 수에 합치면 “무엇이 먼저 보이는가”와 “어떻게 행동하는가”를 다시 섞게 된다. 이는 두 층을 분리하기로 한 사용자 승인과 반대다.

### 2.3 초기 보강 필요와 해결 결과

초기 6개 처음 생각 후보는 3G·3A로 균형이었지만 다음 문제가 있었다.

- 한 문항이 전문가 검토에서 인접 영역 오염으로 탈락하면 방향·상황 여유가 급격히 줄어든다.
- 인지 인터뷰에서 관계 거리·도움 요청 여부·대화 매체가 해석을 지배해도 대체 문항이 적다.
- 빠른 검사와 정밀 검사가 서로 독립적인 증거를 충분히 나눠 쓰기 어렵다.
- 주점수와 과정 프로필을 같은 문항 수로 보이게 만드는 문서상 착시가 생긴다.

이는 G/A 구조가 틀렸다는 판정이 아니라 `후보 중복성 부족` 판정이었다. 승인된 3G·3A를 보강해 현재 처음 생각 후보는 6G·6A, 주력 8·탐색 4다. M04·M05에서 탈락·수정이 발생해도 방향·상황별 대체 후보를 비교할 수 있다.

### 2.4 `RBL-01E-R1` 보강 계약 이행

보강 후보 6개는 다음 조건을 모두 통과했다.

1. `REPORTED_FIRST_ORIENTATION`만 측정한다.
2. G 방향 3개·A 방향 3개다.
3. 승인된 “마음과 해결이 함께 있는 일상적 관계 상황” 범위를 유지한다.
4. 능력, 착함, 공감 정확도, 해결 능력, 예의, 행동 이유를 묻지 않는다.
5. 기존 P01~P05와 완전히 같은 상황·질문을 반복하지 않는다.
6. 관계 거리, 매체, 도움 요청의 명시성, 문제 긴급도를 분산한다.
7. `contextLabel + 하나의 자연스러운 첫 주의 반응` 구조를 지킨다.
8. 새 ID·revision·방향·상황 버킷·오염 가설을 기록한다.
9. 사용자 승인 전 기존 후보 은행의 승인본을 덮어쓰지 않았고 승인 뒤 v0.5를 새로 발행했다.
10. 자동 감사에서 G/A 주증거 12개·HIGH/LOW 6/6·주력 8/탐색 4를 확인했다.

## 3. 교차 영역 seam 지도

`seam`은 두 영역이 같은 뜻이라는 판정이 아니다. 사용자가 한 상황에서 두 반응을 함께 보일 수 있으므로, 문항이 어느 반응을 기준으로 답하게 하는지 확인해야 하는 경계다.

| 우선 seam            | 혼동 가능한 관찰                 | 반드시 구분할 기준                                     | M04/M05 확인 질문                                          |
| -------------------- | -------------------------------- | ------------------------------------------------------ | ---------------------------------------------------------- |
| SE-RE ↔ ER-WD        | 모임·대화 뒤 피로, 평가 앞 주저  | 사람과 교류한 뒤 에너지 변화인가, 잘못될 걱정인가      | “걱정이 없을 때도 같은가?”                                 |
| SE-AI ↔ ER-WD        | 말·질문을 늦게 꺼냄              | 평소 표현 주도성인가, 부정적 결과 걱정인가             | “할 말은 있지만 걱정 때문에 미루는가?”                     |
| SE-AI ↔ RO 실제 반응 | 먼저 묻거나 기다림               | 말 시작 방식인가, 마음/해결 중 무엇을 먼저 확인했는가  | “먼저 말하지 않은 이유가 필요한가?”                        |
| OE-AE ↔ ER-IR        | 장면·음악의 감정 여운            | 미적 감각에 머무는가, 불편 정서가 커지는가             | “불편하지 않아도 오래 머무는가?”                           |
| OE-CI ↔ ER-WD        | 앞뒤 가능성·미래 장면을 떠올림   | 상상 확장인가, 잘못될 시나리오의 반복인가              | “즐겁거나 중립적인 가능성도 떠올리는가?”                   |
| OE-IE ↔ ER-WD        | 정보를 더 찾음                   | 알고 싶은 관심인가, 불안을 줄이기 위한 확인 반복인가   | “충분히 알아도 걱정 때문에 다시 찾는가?”                   |
| OE-IE ↔ SM-EP/OS     | 자료 확인·비교·정리              | 원리 탐구인가, 일을 끝내기 위한 실행·구조인가          | “해야 할 일이 아니어도 알아보는가?”                        |
| RO-EC ↔ OE-IE        | 상대 이야기의 원인·결과가 궁금함 | 관계 장면의 첫 주의 방향인가, 일반 지적 탐구인가       | “사람의 마음과 해결이 함께 있을 때 무엇이 먼저 보이는가?”  |
| RO-EC ↔ ER-IR/WD     | 상대가 괜찮은지 신경 씀          | 상대에게 향한 짧은 관심인가, 본인의 반복 걱정·동요인가 | “관심이 오래 반복되어 일상을 방해하는가?”                  |
| RO 실제 반응 ↔ RO-RN | 말 걸기·기다리기                 | 마음 확인 순서인가, 상대 선택·경계 존중인가            | “상대가 원치 않을 가능성을 고려했는가?”                    |
| SM-EP ↔ ER-WD        | 일을 시작·계속하지 못함          | 평소 실행 지속인가, 걱정 때문에 생긴 주저인가          | “걱정이 적은 일에서도 같은가?”                             |
| SM-OS ↔ ER-WD/IR     | 반복 확인·정리                   | 구조를 만드는 습관인가, 불안을 낮추기 위한 확인인가    | “실수가 걱정되지 않아도 확인하는가?”                       |
| SM-RL ↔ SE-AI/RO-RN  | 약속·연락·역할 이행              | 전반적 실행 책임인가, 표현 주도성·관계 규범인가        | “혼자 하는 일에서도 같은가?”                               |

M04에서 target 일치만 높고 seam 설명이 틀리면 통과시키지 않는다. M05에서 참가자가 목표 반응 대신 오염 기준으로 답하면 문장이 자연스러워도 수정·탈락 대상이다.

## 4. 상황·환경 커버리지

자동 키워드 집계는 의미 판정이 아니라 놓친 편향을 찾는 경고 장치다. 같은 context pair의 두 후보는 각각 집계되므로 숫자는 고유 장면 수가 아니다.

| 영역 | 현재 집중                        | 합리적 이유                  | 검증해야 할 편향                            |
| ---- | -------------------------------- | ---------------------------- | ------------------------------------------- |
| SE   | 관계·대화 20/24                  | 사람 사이 에너지·표현을 측정 | 말하기 능력, 새 모임 접근성, 대면 중심성    |
| OE   | 디지털·미디어 8/36, 시청각 14/36 | 감각·상상·탐구 장면 확보     | 시청각 접근, 콘텐츠 소비량, 디지털 친숙도   |
| RO   | 관계 16/18, 문제·해결 8/18       | 승인된 마음+해결 관계 범위   | 도움 요청, 친밀도, 예의 규범, 대화 매체     |
| SM   | 과업·결정 24/36                  | 실행·질서·책임 측정          | 직장/학교 지위, 일정 통제권, 돌봄·건강 부담 |
| ER   | 관계 6/24, 과업·평가 8/24        | 좌절·불확실성·평가 장면 확보 | 최근 스트레스, 실제 위험, 수면·건강 상태    |

통합 연구에서 함께 수집하거나 목적표집할 문맥:

- 학생·직장인·구직/이직·프리랜서·돌봄 역할
- 혼자 사는 사람·가족 동거·다양한 친밀 관계
- 대면·메신저·전화 등 소통 매체
- 시각·청각·주의집중·읽기 접근 요구
- 최근 큰 변화·수면 부족·건강·돌봄 부담의 비채점 문맥

환경 정보는 코드를 보정하는 비밀 가중치가 아니다. 특정 문항이 어떤 사람에게만 다른 뜻이 되는지 확인하고 DIF·교체 결정을 위한 연구 정보다.

## 5. 문구 방법효과

### 5.1 자동 감지 결과

| 영역 | 핵심 신호                      | 방향 비대칭                                |
| ---- | ------------------------------ | ------------------------------------------ |
| SE   | `먼저/부터/전에` 6개           | 모두 HIGH                                  |
| OE   | 부정·부재 3개, 제한어 9개      | 모두 LOW; 첫·순서어는 HIGH 1·LOW 1         |
| RO   | 첫·순서어 14개                 | HIGH 5, LOW 9                              |
| SM   | 부정·부재 4개, 순서어 7개      | 부정 표현은 모두 HIGH, 순서어 HIGH 6·LOW 1 |
| ER   | 부정·부재 12개, 반복·지속 11개 | 부정 표현 HIGH 4·LOW 8, 첫·순서어 HIGH 1   |

### 5.2 해석

- SE에서는 `먼저`라는 단어 자체가 E 방향을 알려주는 열쇠가 될 수 있다.
- OE에서는 LOW 문항만 `그 장면만`, `따로 떠올리지 않는다`처럼 제한·부정 문법을 가지면 R/N보다 문장 형식에 반응할 수 있다.
- RO에서는 “첫 주의”를 측정하기 위해 순서어가 필요하지만, 한 방향에만 집중되면 G/A보다 말버릇을 구분할 수 있다.
- SM HIGH는 준비·확인·예방 문구가 많아 성실해 보이는 답을 고르게 할 수 있다.
- ER LOW의 부정형은 C가 “문제가 없는 사람”처럼 보이게 하고 Q의 낙인을 강화할 수 있다.

### 5.3 처리 원칙

1. 숫자를 맞추기 위해 어색한 부정문을 억지로 추가하지 않는다.
2. 의미가 같은 자연스러운 A/B 문구를 인지 인터뷰에서 비교한다.
3. `wording_polarity`, `ordering_cue`, `limiter_cue`, `repetition_cue`를 데이터 필드로 보존한다.
4. 정량 파일럿에서 방향 요인과 문구 방법요인을 구분해 본다.
5. 한 방향의 문법이 쉬워서 생긴 차이를 성향 차이로 해석하지 않는다.

## 6. 가치·낙인·정답처럼 보이는 방향

| 영역·방향              | 정답처럼 보일 위험   | 금지 해석                         |
| ---------------------- | -------------------- | --------------------------------- |
| SE-E                   | 자신감·사교성·리더십 | 사람을 잘 다룸, 사회성이 더 좋음  |
| OE-N                   | 창의성·지적 능력     | 더 똑똑함, 상상력이 우월함        |
| RO-A 및 마음 확인 행동 | 따뜻함·착함·공감     | 더 좋은 사람, 관계 능력이 더 좋음 |
| SM-K 및 RL HIGH        | 생산성·책임감        | 성공 가능성이 높음, M은 무책임함  |
| ER-C                   | 건강·강함·안정       | 정신건강이 더 좋음, Q는 불안정함  |

전문가와 사용자는 각 문항에 대해 다음을 별도로 평가한다.

- 어느 방향이 사회적으로 더 좋아 보이는가?
- 어느 방향이 도덕적으로 더 옳아 보이는가?
- 어느 방향이 능력·건강의 높고 낮음처럼 보이는가?
- 실제 내 모습 대신 되고 싶은 모습을 고르게 하는가?

검사 화면에서는 답을 고를 때 방향별 색·캐릭터 표정·보상·칭찬·경고를 사용하지 않는다. 결과 화면도 두 방향을 장점/단점 서열로 그리지 않는다.

특히 ER의 최근 상태를 Q에 더하지 않고, SM의 바쁜 환경을 M으로 자동 보정하지 않으며, SE의 말하기 접근성을 I로 해석하지 않는다.

| ---------------------------------- | ------------------- | -------------------- | ----------------- |

## 9. 통합 M04 전문가 검토 입력

OE 실행 키트의 익명화·무작위화·Stage 1 잠금 구조를 전 코어에 재사용하되, OE 단독 36문항 packet을 그대로 확장하지 않는다.

### 9.1 실행 전 조건

- `RBL-01E-R1` RO 보강 후보 승인
- 모든 후보의 `candidate_bank_version`과 `revision_id` 잠금
- 주점수·조건부·보조층·비공개 연구 지위 잠금
- 전문가 역할·모집·보상·개인정보 절차의 별도 사용자 승인
- 판정 기준과 제외 규칙 사전등록

### 9.2 두 종류 packet

1. `DOMAIN_COVERAGE_PACKET`
   - 모든 후보가 필요한 수의 독립 검토를 받게 한다.
   - opaque ID만 노출하고 실제 domain·facet·방향·점수 역할은 가린다.
   - Stage 1에서 target mapping, 대안 target, 명확성, 단일 반응, 바람직성, 임상·능력 오염을 평가한다.
   - Stage 1 잠금 뒤 Stage 2에서 의도한 target·방향·층을 공개하고 적합성·수정 범위를 평가한다.

2. `SEAM_PACKET`
   - 3절의 우선 seam별 후보를 섞어 상대 구분성을 본다.
   - 같은 상황·비슷한 동사를 가진 인접 문항을 일부 포함한다.
   - `둘 다 가능`을 오류로 취급하지 않고, 어떤 추가 문구가 target을 분리하는지 기록한다.
   - 주점수 문항과 보조층 문항을 같은 facet처럼 평가하지 않는다.

### 9.3 packet 생성 원칙

- 한 reviewer에게 150개 전체를 한 번에 주지 않는다.
- reviewer별 부담·순서·영역·HIGH/LOW·문구 유형을 균형 배정한다.
- 각 후보는 최소 검토자 수를 충족하고 일부 anchor 문항은 reviewer 사이에 중복한다.
- 정확한 packet 수·문항 수·중복 배정은 RO 보강 후 생성 스크립트가 계산한다.
- 문항 작성자는 독립 reviewer 집계에서 제외하고 adjudication에만 참여한다.
- 기존 OE 사양의 6~8명 독립 전문가, target mapping 75% 이상, 관련성·명확성 중앙값 3 이상은 사전 검토 기준선으로 가져오되 전 코어 workload에 맞춰 심리측정 책임자가 확정한다.

## 10. 통합 M05 모바일 인지 인터뷰 입력

M05는 종이 문항 검토가 아니라 S03-R 실제 모바일 runner에서 수행한다. 다만 참가자 한 명에게 모든 후보를 제시하지 않는다.

### 10.1 세션 구조

```text
연습 2문항
→ 영역 균형 핵심 문항 15~20개
→ seam 비교 모듈 6~10개
→ 판단 어려움·이전/다음·이탈 복원 확인
→ 선택 문항 retrospective probe
→ 코드·점수 미발급, 인터뷰 종료 안내
```

### 10.2 seam 모듈

| 모듈               | 핵심 경계                                |
| ------------------ | ---------------------------------------- |
| X01 사람 사이 행동 | SE-RE·SE-AI ↔ ER-WD·RO 실제 반응         |
| X02 가능성과 걱정  | OE-CI·OE-IE ↔ ER-WD                      |
| X03 마음과 해결    | RO-EC ↔ OE-IE·ER-IR/WD                   |
| X04 실행과 확인    | SM-EP·SM-OS·SM-RL ↔ ER-WD·OE-IE          |
| X05 가치와 문법    | 모든 영역의 HIGH/LOW·부정·순서·제한 문구 |

### 10.3 참가자에게 확인할 것

- 상황 라벨만 읽고 어떤 장면을 떠올렸는가?
- 질문은 무엇을 묻는다고 이해했는가?
- 답할 때 능력·선호·감정·행동·이상적인 모습을 무엇으로 기준 삼았는가?
- 비슷한 두 문항의 차이를 자신의 말로 설명할 수 있는가?
- `판단하기 어려워요`를 언제 선택했고, 저장·복원이 실제로 되는가?
- 한 문장이 특정 직업·관계·장애·생활환경에만 맞는가?
- 어떤 방향이 더 착하거나 건강하거나 유능한 답처럼 보였는가?

M05 소표본으로 어느 구조가 통계적으로 우월하다고 확정하지 않는다. 이해 실패·판단 기준 혼동·접근성 문제를 찾아 revision을 만드는 단계다.

## 11. 빠른·정밀 검사와 점수에 주는 함의

- 빠른 문항 수와 정밀 문항 수를 후보 수로 자동 결정하지 않는다.
- 빠른 검사는 각 대표 영역의 예비 방향만 제공하고 세부 facet·과정 프로필을 확정하지 않는다.
- 정밀 검사는 facet별 충분한 독립 증거와 응답 품질이 있을 때만 대표 코드와 상세 지도를 보여준다.
- RO 실제 반응, RO-RN, SM-RL, ER 최근 상태는 서로 다른 score role을 유지한다.
- 경계 점수는 강제로 한 글자로 고정하지 않고 불확실성·두 방향 유사성을 함께 처리한다.
- M04·M05를 통과해도 M06·M07 정량 파일럿 전에는 가중치·문항 수·경계·quick/full 동등성을 확정하지 않는다.

## 16. 근거와 한계

- [Standards for Educational and Psychological Testing](https://www.testingstandards.net/open-access-files.html) — 용도별 타당성 근거, 공정성, 점수 해석 책임의 기준

- [NIMH, Mental Health Screening](https://www.nimh.nih.gov/health/topics/mental-health-screening) — 일반 성향 설명과 검증된 정신건강 선별을 혼합하지 않는 안전 경계 참고
- [BFI-2 project](https://www.colby.edu/psych/personality-lab/) — broad domain과 facet을 함께 검토하는 성격 측정 구조 참고

위 근거는 뉴앙 문항이 타당하다는 증명이 아니다. 뉴앙의 한국어 문항·5개 코드·빠른/정밀 구조는 독립 전문가 검토, 실제 사용자 인지 인터뷰, 정량 구조·신뢰도·공정성 검증을 별도로 통과해야 한다.
