## 0. 보강 목적

현재 RO-EC 후보 문장 12개 중 대표 G/A를 직접 측정하는 `처음 드는 생각` 주점수 후보는 3G·3A, 총 6개다. 나머지 6개는 `실제 나타나는 반응`을 설명하는 보조층이므로 G/A 주점수 후보 수에 합칠 수 없다.

이번 repair는 G/A의 의미를 넓히지 않는다.

```text
G = 원인·결과·해결할 부분이 먼저 보이는 편
A = 상대의 감정 상태와 안녕이 먼저 보이는 편
```

목표는 같은 구성개념 안에서 탈락·수정 여유와 상황 다양성을 확보하는 것이다. 아래 여섯 문장은 2026-07-18 KST 사용자 승인을 받아 [RO-EC 공식 후보 은행 v0.5](./NUANG_M03_RO_EC_PROCESS_PAIR_CANDIDATE_BANK.md)에 병합됐다.

## 1. 설계 방식

새 후보는 세 상황마다 G/A 문장을 하나씩 만든 matched pair다.

```text
같은 상황
├─ G 후보: 원인·해결에 가는 처음 주의
└─ A 후보: 상대 마음에 가는 처음 주의
```

이 방식은 가까운 관계, 메시지, 의견 요청 같은 상황 차이가 G/A 방향 차이로 오인되는 것을 줄인다. 같은 pair의 두 문항은 후보 비교용이며 실제 한 form에는 함께 넣지 않는다.

## 3. 왜 이 세 상황을 선택했는가

| pair | 보강하는 공백                            | 방향과 분리해야 할 영향                     | 현재 역할 |
| ---- | ---------------------------------------- | ------------------------------------------- | --------- |
| R01  | 가까운 관계·메시지·명시적 도움 요청 없음 | 친밀 의무, 문자 해석, 답장 행동             | 주력 후보 |
| R02  | 덜 가까운 관계·명시적 의견 요청          | 예의 규범, 요청 때문에 해결을 떠올리는 효과 | 탐색 후보 |
| R03  | 여러 사람이 함께 겪는 작은 차질          | 공동 책임, 과업 해결, SM-EP/OS              | 주력 후보 |

기존 은행은 누군가가 힘든 일을 말하거나 지쳐 보이는 대면·일반 관계 장면에 집중돼 있었다. 새 세 상황은 G/A의 본질을 유지하면서 관계 거리, 매체, 도움 요청의 명시성, 공동 과업 여부를 분산한다.

## 5. 문구 균형 감사

### R01

- 두 방향 모두 `처음에는 ... 궁금해진다`를 사용한다.
- G/A 차이는 궁금증의 대상만 `나아질 방법 ↔ 현재 마음`으로 바뀐다.
- 행동, 위로, 해결 제시, 답장 순서는 묻지 않는다.

### R02

- 두 방향 모두 `처음에는 ... 궁금해진다`를 사용한다.
- G/A 차이는 궁금증의 대상만 `일이 생긴 원인 ↔ 당시 상대 마음`으로 바뀐다.
- 명시적 의견 요청이 G를 과도하게 끌어올릴 수 있어 탐색 후보로 둔다.

### R03

- 두 방향 모두 `처음에는 ... 주의가 간다`를 사용한다.
- G/A 차이는 주의 대상만 `어긋난 부분 ↔ 함께한 사람들의 마음`으로 바뀐다.
- 공동 과업 장면이 SM과 섞이는지 seam 검토가 필수다.

세 pair는 문법을 맞췄지만 실제 사용자에게 같은 정도로 자연스럽다는 뜻은 아니다. 인지 인터뷰에서 `처음에는`, `먼저`, `주의가 간다`가 동일한 “처음 드는 생각”으로 이해되는지 비교한다.

## 6. 가치중립 계약

| G 응답에서 허용되는 해석                      | A 응답에서 허용되는 해석                 |
| --------------------------------------------- | ---------------------------------------- |
| 사건의 원인·결과·해결할 부분에 주의가 먼저 감 | 상대의 감정 상태와 안녕에 주의가 먼저 감 |

아래 해석은 금지한다.

- G는 논리적·똑똑함·일을 잘 해결함
- G는 공감이 부족하거나 차가움
- A는 착함·따뜻함·관계 능력이 좋음
- A는 상대의 마음을 정확히 맞히는 능력이 높음

응답 방향에 따라 UI 색·캐릭터 표정·보상·칭찬을 다르게 주지 않는다.

## 8. 승인·병합 처리 기록

사용자가 여섯 문구를 승인해 다음을 수행했다.

1. RO-EC 공식 후보 은행 `m03-ro-ec-process-pairs.v0.5`에 repair 후보 6개를 병합했다.
2. 이전 `m03-ro-ec-process-pairs.v0.4`와 변경 이유를 문서에 보존했다.
3. 통합 자동 감사의 정식 인벤토리에 repair 후보를 포함했다.
4. 대표 G/A 주점수 후보 12개가 6G·6A, 주력 8·탐색 4인지 검증했다.
5. 전 영역 질문·상황 완전 중복과 문구 방향 비대칭을 다시 계산했다.
6. 재감사 통과로 `RBL-01E-R1`과 `ITEM_DESIGN_GATE_COMPLETE`를 닫았다.
7. 다음 작업을 S03-R 빠른·정밀 공통 검사 runner v3 UI/UX 설계로 전환했다.

이번 병합은 연구 후보 문서에만 적용한다. 운영 점수, DB, seed, 앱 화면은 변경하지 않는다.

## 9. 사용자 승인 기록

사용자는 아래 기준으로 세 쌍의 문구를 검토하고 승인했다.

1. 상황이 누구나 떠올릴 수 있을 만큼 구체적인가?
2. 질문이 행동이 아니라 실제 `처음 드는 생각`을 묻는가?
3. G와 A 중 한쪽이 더 착하거나 더 유능한 정답처럼 보이지 않는가?
4. 어색하거나 즉시 이해되지 않는 표현이 있는가?

승인 판정은 여섯 후보 모두 유지하되 R02를 명시적 의견 요청의 상황 지배 효과를 보기 위한 탐색 후보로 두는 것이다. 이 승인은 최종 운영 문항·점수·G/A 발급 승인이 아니다.
