## 1. 이번 검토에서 정할 것

기존의 `함께·탐색, 상대 마음 살피기·꾸준…`은 전체 성향 설명으로 쓰기에는 지나치게 단순하다. 이 표현은 각 글자를 빠르게 훑는 `자리별 요약 태그`로만 사용한다. 결과의 중심에는 사용자가 기억할 역할형 이름과 생활 모습을 떠올릴 수 있는 `한눈에 보는 성향 설명`을 둔다.

```text
ENAKQ
관계를 여는 지휘자
사람들과 함께할 때 활력이 오르고, 보이는 내용 너머의 가능성과 새로운 관점을 더 찾아봐요. 관계 문제에서는 상대가 어떤 마음인지 자연스럽게 살피며, 해야 할 일은 비교적 꾸준히 이어가요. 불편한 일이 생기면 걱정과 감정이 비교적 빠르게 커질 수 있어요.
```

역할형 이름은 검사로 확인한 능력이나 직업이 아니다. 조합의 분위기를 기억하게 하는 브랜드 별칭이다. 따라서 이름이 재미있더라도 `한눈에 보는 성향 설명`에서는 다섯 자리의 실제 의미를 생활 장면과 연결해 보여주고, 자리별 상세 설명에서는 경계와 반대 방향까지 확인할 수 있게 한다.

## 2. 이름 작성 기준

- 첫 번째와 두 번째 자리는 `함께/조용히`, `현실 장면/새 가능성`의 분위기에 반영한다.
- 세 번째와 네 번째 자리는 역할이 관심을 두는 곳과 일상의 흐름에 반영한다.
- 다섯 번째 자리는 이름의 정서적 온도에만 조심스럽게 반영하고, `멘탈이 강함/약함`으로 표현하지 않는다.
- `리더십이 좋다`, `공감 능력이 높다`, `문제 해결을 잘한다`처럼 측정하지 않은 능력을 설명문에서 주장하지 않는다.
- `압도하는`처럼 우열을 만드는 표현은 스타일 참고로만 보고 실제 후보에서는 제외했다.
- 이름 아래에는 항상 한눈에 보는 설명과 자리별 상세 설명을 함께 둬, 이름만 보고 축 의미를 잘못 이해하지 않게 한다.
- 이름은 `관계를 여는 지휘자` 정도의 2~4어절로 제한한다.
- 다섯 방향을 이름 하나에 모두 욱여넣지 않고, 조합에서 가장 기억하기 쉬운 핵심 이미지만 남긴다.
- 한눈에 보는 설명은 `에너지와 관심 → 관계와 일상 → 걱정과 감정 반응` 순서의 세 문장으로 쓴다.
- 단어만 나열하지 않고, 사용자가 평소 모습을 떠올릴 수 있는 행동과 관심 방향으로 설명한다.
- `항상`, `반드시`처럼 단정하지 않고 `편이에요`, `수 있어요`처럼 검사 결과의 범위를 지킨다.

## 4. 구현 결과와 경계

- 역할형 이름과 세 문장 설명은 베타 결과 화면의 앱 데이터 계약에 반영했다.
- 기존의 짧은 직역 문구는 메인 설명에서 제외하고, `마음 먼저`를 제거한 자리별 요약 태그로만 사용한다.
- `NUANG-PROFILE-NAME-CANDIDATE-1.1` migration과 32개 정의를 작성했다.
- 320px 모바일 화면에서 가로 넘침 없이 역할형 이름과 세 문장 설명이 표시되는 것을 확인했다.
- 원격 DB에는 아직 적용하지 않았다. 외부 저장을 다시 명시적으로 승인받은 뒤 candidate 상태로만 저장한다.
- 공유·피드·비교에는 계속 노출하지 않는다.
- 정량 검증 뒤 코드 축이 바뀌면 이름도 같은 release에서 다시 검토한다.
