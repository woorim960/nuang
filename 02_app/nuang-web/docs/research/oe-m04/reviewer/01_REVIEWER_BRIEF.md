# NUANG OE 문항 독립 검토자 안내

문서 버전: `m04-oe-expert-kit.v0.1`  
대상: 승인 후 배정된 M04 reviewer  
주의: 이 문서만으로 모집·연구 참여가 성립하지 않는다. 별도의 자격 확인·동의·보상·데이터 처리 안내가 필요하다.

## 1. 검토 목적

뉴앙은 자기이해를 돕는 성향 검사를 개발하고 있다. 이번 검토는 문장이 좋아 보이는지를 묻는 작업이 아니라, 각 문항이 의도한 한 가지 성향을 다른 성향·능력·환경 영향과 구분해서 묻는지 확인하는 작업이다.

검토 대상은 `분위기와 아름다움 느끼기`, `장면과 이야기 떠올리기`, `원리와 관점 알아보기` 영역에서 작성한 후보 36문항이다. 어떤 문항이 어느 영역을 목표로 했는지는 Stage 1 제출 전 공개하지 않는다.

## 2. 독립 검토 원칙

- 다른 reviewer와 문항·정답 추정·평가 내용을 공유하지 않는다.
- 원문 척도나 기존 뉴앙 문항을 찾아 정답을 맞히려 하지 않는다.
- 문항을 작성자의 의도에 맞춰 좋게 해석하지 않는다.
- 실제 2030 사용자가 읽는 한국어 그대로 평가한다.
- 모호함·불편함·편향 가능성이 있으면 작은 문제라도 근거와 함께 남긴다.
- HIGH와 LOW를 좋은 사람·나쁜 사람, 능력 있음·없음으로 판단하지 않는다.
- 이해상충이나 사전 노출을 발견하면 즉시 운영자에게 알리고 응답을 멈춘다.

## 3. 두 단계 진행

### Stage 1 — blind mapping

`상황 라벨 + 질문 문장`만 보고 가장 가까운 구성개념, 방향, 명확성, 일상 보편성, 척도 적합성, 오염 위험을 평가한다. actual target이나 keyed direction은 모르는 상태다.

제출 후 원본은 수정할 수 없게 잠긴다.

### Stage 2 — target 공개 검토

Stage 1 잠금이 확인된 뒤, 같은 순서의 문항에 target facet·keyed direction·의도한 응답 과정이 공개된다. 이때 target 관련성, 방향 적합성, 범위 기여, 인접 개념 분리 가능성과 최종 권고를 평가한다.

Stage 2에서 생각이 바뀌어도 Stage 1을 수정하지 않는다. 차이는 중요한 연구 자료다.

## 4. 공통 응답 맥락

일반 사용자는 지난 6개월의 평소 모습을 떠올리고, 제시된 상황에서 해당 반응이 자신에게 얼마나 자주 나타나는지 1~5점으로 응답한다고 가정한다. 경험 부족·접근 제한·문장 이해 문제로 판단하기 어려운 경우에는 별도의 `판단하기 어려워요`를 사용할 수 있다.

reviewer는 자신의 성향에 답하는 것이 아니라, 이 방식으로 일반 사용자가 안정적으로 답할 수 있는 문항인지 평가한다.

## 5. 운영자에게 바로 알려야 하는 상황

- actual candidate ID, target facet, HIGH/LOW key가 Stage 1 자료에 노출됨
- 다른 reviewer의 응답이나 집계 결과를 받음
- 배정받지 않은 slot의 packet을 받음
- 파일이 손상되거나 문항이 중복·누락됨
- 참여 중단 또는 데이터 삭제를 요청하고 싶음

노출 사고가 있으면 추측으로 계속 진행하지 않는다. 운영자가 해당 응답의 사용 가능성을 사전등록 규칙에 따라 판단한다.
