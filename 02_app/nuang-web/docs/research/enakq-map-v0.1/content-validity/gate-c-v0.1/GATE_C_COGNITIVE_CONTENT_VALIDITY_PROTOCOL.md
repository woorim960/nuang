# Gate C · 60문항 인지·내용타당도 검증 프로토콜

- 프로토콜: `ENAKQ-GATE-C-COGNITIVE-CONTENT-VALIDITY.v0.1`
- 후보 문항 세트: `NUANG-CORE-GATE-C-COGNITIVE-CANDIDATE-1.0`
- 상태: `PREPARED_NOT_RUN_NOT_EXTERNAL_VALIDATION`
- 고객 채점 사용: 금지
- 운영 DB 연구 자료 저장: 금지
- Gate C 상태: `not_started`

## 1. 이 단계가 성향 지도에 필요한 이유

성향 지도와 비교 리포트는 60문항이 실제 사용자에게 같은 뜻으로 읽히고, 의도한 세부 성향을 묻는다는 근거가 있어야 한다. 내부 전문가 검토만으로는 사용자가 문장을 어떻게 이해하고 어떤 경험을 떠올려 응답했는지 알 수 없다.

Gate C는 정답률을 재는 절차가 아니다. 사용자가 문항을 읽고 다음 네 단계를 거치는 동안 의미가 어긋나는 지점을 찾는다.

1. **이해**: 상황과 질문을 어떤 뜻으로 읽었는가
2. **회상**: 최근 6개월 중 어떤 실제 경험을 떠올렸는가
3. **판단**: 경험의 어떤 부분에 무게를 두었는가
4. **응답 선택**: 그 판단을 5단계 응답 또는 `판단하기 어려워요`로 옮길 수 있었는가

문항이 이 관문을 통과하기 전에는 성향 지도의 설명, 궁합, 관계 조언 또는 고객용 점수의 근거로 승격하지 않는다.

## 2. 검증 후보 60문항 고정

기존 `NUANG-CORE-BETA-1.0`의 60문항을 출발점으로 사용하되, 이미 내부 검토에서 중대한 문제가 확인된 SM-OS 낮은 방향 문항 세 개는 구문이 개선된 후보로 교체한다. 운영 검사 데이터는 바꾸지 않고 Gate C 전용 후보 세트에서만 교체해 버전 추적을 보존한다.

| 기존 beta 문항           | Gate C 후보   | 교체 이유                                                                       |
| ------------------------ | ------------- | ------------------------------------------------------------------------------- |
| `NU-B1-018` · `SMOS-C04` | `SMOS-C10-r2` | 눈에 띄는 일부터 처리한다는 문장이 순서 구조화보다 주의·긴급성·실행 방식과 섞임 |
| `NU-B1-028` · `SMOS-C10` | `SMOS-C13`    | 가까운 곳에 둔다는 문장이 효율적인 고정 체계와 구분되지 않음                    |
| `NU-B1-058` · `SMOS-C02` | `SMOS-C08-r3` | 쓰기 편한 곳이라는 문장이 구조화보다 접근 편의와 환경 조건을 측정함             |

`SMOS-C10-r2`와 `SMOS-C13`은 내부 표적 검토에서 `PASS_TO_COGNITIVE`였고, `SMOS-C08-r3`은 M05 인지 인터뷰용으로 준비됐지만 아직 실제 참여자 검증은 하지 않았다. 세 문항 모두 Gate C에서 동일하게 미검증 후보로 취급한다.

## 3. 폼 설계

60문항을 5개 폼으로 나누고 한 참여자는 한 폼의 12문항만 본다.

- 폼마다 10개 대표 facet을 모두 포함한다.
- 폼마다 높은 방향 6개, 낮은 방향 6개를 포함한다.
- 한 문항은 한 폼에만 들어가며 전체 5개 폼을 합치면 60문항이 정확히 한 번씩 포함된다.
- 같은 facet이 연달아 나오지 않도록 순서를 자동 배치한다.
- 참여자 파일에서는 facet, 방향, 채점 키, 원본 candidate ID를 숨긴다.
- 실제 모바일 검사 러너에서 상황 라벨과 질문을 함께 보여 준다.

세부 성향 이름이나 방향을 먼저 보여 주면 참여자가 의도에 맞춰 설명하려는 편향이 생길 수 있으므로, 진행자용 매핑과 참여자용 문항을 분리한다.

## 4. 참여자 구성과 라운드

이 연구는 통계적 모집단 대표성을 추정하는 설문이 아니라 문항의 문제를 발견하는 목적 표집이다. 한 집단에만 맞는 문장을 통과시키지 않도록 이해 조건과 생활 조건의 차이를 의도적으로 포함한다.

| 구분                 |                      인원 |        폼당 인원 | 목적                                       |
| -------------------- | ------------------------: | ---------------: | ------------------------------------------ |
| 1차 발견 라운드      |                      20명 |              4명 | 오해, 경험 부족, 척도 문제, 환경 혼입 발견 |
| 2차 수정 확인 라운드 |               새로운 20명 |              4명 | 수정 문구와 전체 문항의 반복 문제 확인     |
| 표적 재검증          | 수정 문항당 새로운 4~10명 | 필요에 따라 배정 | S2·S3 또는 특정 하위집단 문제 해결 확인    |

두 정규 라운드를 마치면 각 문항은 최소 8회의 관찰을 갖는다. 수정된 최종 문구는 수정 전 응답을 근거로 통과시킬 수 없고, 새로운 참여자 최소 4명에게 다시 확인한다.

### 목적 표집에서 확인할 차이

- 20대와 30대
- 성별 자기 설명은 선택 입력으로 두되 한 집단에 편중되지 않게 확인
- 학생, 직장인, 구직·전환, 돌봄 등 생활 맥락
- 수도권과 비수도권 등 지역 맥락
- 성향 검사 친숙도 높음·낮음
- 긴 글 읽기와 모바일 설문 친숙도
- 교대근무, 돌봄, 건강, 장애·접근성 도구 등 문항 경험에 영향을 줄 조건

이 정보는 표본의 빈 곳을 찾는 용도이며 개인의 성향 점수나 공개 프로필에 사용하지 않는다.

## 5. 세션 절차

한 세션은 약 55~65분을 기준으로 한다.

1. 연구 목적, 중단 권리, 개인정보 처리 범위를 설명하고 별도 동의를 확인한다.
2. 실제 모바일 UI에서 한 문항을 보여 주고 참여자가 먼저 자연스럽게 응답하게 한다.
3. 진행자는 정답, facet, 방향을 알려 주거나 응답을 유도하지 않는다.
4. 문항별로 이해·회상·판단·응답 선택을 묻는다.
5. 좋은 사람처럼 보이는 답을 골랐는지, 생활·접근 조건이 섞였는지 확인한다.
6. 상황 라벨과 질문의 연결, 모바일 줄바꿈, `판단하기 어려워요` 흐름을 확인한다.
7. 문항을 수정하지 않은 원 기록을 먼저 잠그고, 라운드 단위 분석 후 다음 라운드 문구를 고정한다.

진행자가 임의로 설명을 덧붙여야 이해되는 문항은 통과하지 않는다. 설명을 덧붙인 순간부터는 원문 자체가 이해된 근거가 아니기 때문이다.

## 6. 문제 코드와 심각도

- `S0`: 관찰 메모. 문항 의미나 응답에 영향 없음
- `S1`: 경미한 마찰. 의미는 유지되지만 읽기·회상·표현에 작은 부담
- `S2`: 재료적 문제. 해석, 방향, 응답 또는 하위집단 공정성에 영향을 줌
- `S3`: 중대 문제. 임상·관계 단정, 개인정보 불편, 심각한 오채점 위험

전체 문제 코드는 `generated/internal/issue_codebook.csv`에 고정한다. `S2`와 `S3`는 발생 횟수가 적다는 이유로 무시하지 않는다. 한 사례라도 구성개념 오염이나 안전 문제를 구체적으로 드러내면 수정·보류·제외 판단의 근거가 된다.

## 7. 문항 판정 규칙

최종 판정은 다음 다섯 가지만 사용한다.

- `KEEP_FOR_PILOT`: 최종 문구가 정량 파일럿으로 갈 수 있음
- `COPY_REVISE_RETEST`: 표현을 수정하고 새로운 참여자에게 다시 확인해야 함
- `CONSTRUCT_REWRITE`: 문구가 아니라 측정 대상 또는 장면 설계부터 다시 해야 함
- `HOLD_FOR_SUBGROUP`: 특정 생활·접근 조건에서 공정하게 작동하는 근거가 부족함
- `EXCLUDE`: 현재 측정모형에서 제외하는 편이 안전함

Gate C 전체 통과에는 다음 조건이 모두 필요하다.

1. 1차와 2차 라운드가 모두 완료됐다.
2. 목적 표집의 필수 범위가 채워졌다.
3. 60개 최종 문항이 모두 `KEEP_FOR_PILOT`이다.
4. 문항당 최종 근거가 최소 8건이다.
5. 수정 문항은 최종 수정 뒤 새로운 참여자 최소 4명이 확인했다.
6. 필수 탐침 기록이 비어 있지 않다.
7. 해결되지 않은 S2·S3, 하위집단 공백, 전역 UI 문제가 없다.

숫자만으로 자동 통과시키지 않는다. 상반된 설명과 소수 하위집단의 문제를 원문 근거와 함께 조정하고, 최종 결정 이유를 기록한다.

## 8. 중단·재설계 규칙

- 임상 진단처럼 읽히거나 관계 결과를 단정하면 해당 문항 노출을 즉시 중단한다.
- 불필요한 민감정보 노출 또는 답변 불편이 반복되면 문항과 수집 항목을 함께 재검토한다.
- 같은 UI 문제가 여러 문항에서 반복되면 개별 문항 수정 전에 검사 러너를 고친다.
- 한 facet에서 다수 문항이 인접 facet으로 읽히면 개별 문구가 아니라 측정모형으로 돌아간다.
- `판단하기 어려워요`가 경험 부족을 안전하게 표현하지 못하면 점수 계산을 시작하지 않는다.

## 9. 데이터·개인정보 경계

- 참여자 ID는 가명으로만 기록한다.
- 실명·연락처·원문 음성·영상은 이 생성 패킷에 넣지 않는다.
- 녹음이 필요하면 보관 기간, 접근 권한, 삭제 절차가 포함된 별도 동의를 먼저 승인받는다.
- 연구 응답은 운영 고객 DB와 분리한다.
- 원 기록, 수정 이력, 판정 파일을 덮어쓰지 않고 라운드·revision ID로 버전 관리한다.
- 실제 참여자 모집·연락·보상·연구 실행은 별도 승인 없이는 시작하지 않는다.

## 10. 생성 파일

```text
generated/
├── manifest.json
├── participant/
│   ├── SESSION_GUIDE.md
│   └── FORM_A~FORM_E.csv
├── runner/
│   └── gate-c-runner.json
└── internal/
    ├── item_mapping_and_probes.csv
    ├── issue_codebook.csv
    ├── session_log_template.csv
    ├── item_decision_template.csv
    └── recruitment_coverage_template.csv
```

생성과 무결성 확인:

```bash
npm run research:core:gate-c-kit
npm run research:core:gate-c-kit:check
```

`--check`는 60문항, facet별 3 HIGH·3 LOW, 5개 폼의 12문항·10 facet·6 HIGH·6 LOW, 세 교체 문항, 참여자 파일의 채점 정보 비노출, 필수 탐침, 생성 파일 해시가 현재 규칙과 같은지 확인한다.

개발 환경의 `/research/gate-c/internal`에서는 배정 폼 선택, 가명 세션 설정, 실제 검사 UI와 같은 첫 응답, 진행자 탐침 기록, 로컬 JSON 내보내기를 실행할 수 있다. 이 진행자용 경로는 프로덕션에서 404로 닫히며 서버·Supabase·고객 결과 저장소를 호출하지 않는다.

공개 `/research/gate-c`는 진행자 인터뷰를 대체하지 않는 자가 참여형 보조 트랙이다. 이름·연락처·정확한 생년월일·원 IP를 받지 않고 성인 여부, 연령대, 생활 맥락, 성향검사 경험 정도와 12개 응답만 저장한다. 완료 즉시 문항별 답 변경률, 판단 어려움 비율, 문구 혼란 표시, 첫 응답 시간을 집계해 `추가 표본 필요`, `관찰 유지`, `검토 필요`로 분류한다. 이 자동 분류는 검수 대기열만 갱신하며 운영 문항, 채점 키, 결과 문구, release 상태를 수정할 권한이 없다. 이상하게 빠른 응답은 집계에서 제외하고, 익명 기록은 최대 1년 보관하며 참여자가 받은 삭제 코드로 삭제할 수 있다.

## 11. 방법론 근거와 한계

- CDC CCQDER의 인지 인터뷰는 이해, 회상, 판단, 응답 과정을 탐색하고 목적 표집과 반복적 수집·분석을 사용한다.
- OMB 인지 인터뷰 기준은 연구 설계, 수행, 분석, 보고를 분리하고 투명하게 기록하도록 요구한다.
- COSMIN 내용타당도 틀의 관련성, 포괄성, 이해 가능성을 뉴앙 문항 판정 관점에 적용한다.

이 프로토콜과 내부 AI 검토는 외부 사용자 근거가 아니다. 40명은 문제 발견을 위한 계획값이며 정량 신뢰도·요인 구조·측정불변성·규준을 입증하지 않는다. Gate C를 통과한 뒤 별도의 정량 파일럿과 심리측정 분석으로 이동해야 한다.

공개 자가 참여 자료 역시 인지 인터뷰의 말하기 탐침을 제공하지 않으므로 Gate C `passed` 근거로 단독 사용할 수 없다. 공개 자료는 문제 후보를 더 빨리 찾고 정식 인터뷰의 우선순위를 정하는 보조 신호로만 사용한다.
