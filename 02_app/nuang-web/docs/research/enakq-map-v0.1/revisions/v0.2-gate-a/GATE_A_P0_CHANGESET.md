비교 결과도 삭제하거나 양측이 현재 허용한 정보만으로 다시 만든다.

## AI 사용 계약

허용 후보는 사용자가 직접 요청한 글쓰기 보조와 요약뿐이다. 실제 기능을
도입할 때에도 다음을 모두 지켜야 한다.

1. 사용 전에 입력 범위와 외부 모델 전송 여부, 보관 범위를 보여 준다.
2. 외부 모델 전송은 기본 거부한다.
3. 글·사진·활동 기록으로 성향, 호감, 정신건강, 위험성을 추정하지 않는다.
4. 별도이고 명확한 동의 없이 모델 학습이나 다른 목적으로 재사용하지 않는다.
5. 사용자가 취소하고 결과를 고치고 삭제할 수 있어야 한다.

AI 기능을 제공한다는 이유로 현재 데이터가 외부 모델에 전송되는 것은 아니다.
공급자와 실제 데이터 흐름이 정해진 뒤 별도 구현·법률·보안 검토를 통과해야
활성화할 수 있다.

## 안전 흐름

앱은 게시물이나 활동 기록으로 위기를 자동 추정하지 않는다. 아래 흐름은
사용자의 직접 보고나 명시적인 도움 요청으로만 시작한다.

| 단계          | 성향 콘텐츠 | 우선 행동                                           |
| ------------- | ----------- | --------------------------------------------------- |
| 평소 자기관찰 | 유지        | 선택형 자기관찰을 제공하고 언제든 중단할 수 있게 함 |
| 지원 탐색     | 후순위      | 지역에 맞는 전문 지원 탐색을 제공                   |
| 즉시 안전     | 중단        | 지역별 긴급 지원과 신뢰할 수 있는 사람 연결을 우선  |

세 단계 모두 진단명, 위험 확률, 성향 기반 원인, 관계 판정을 만들지 않는다.
지역별 자원은 최신성, 운영 시간, 접근 실패, 네트워크 실패를 별도로 관리해야
한다.

## 결과 버전 계약

새로 생성되는 결과는 다음 버전을 모두 연결할 수 있어야 한다.

- 설문 버전
- 문항 세트 버전
- 채점 버전
- 경계 판단 규칙 버전
- 사용자 문구 버전
- 성향지도 버전

중요한 기준 변경은 과거 결과를 몰래 덮어쓰지 않는다. 변경 이유를 보여 주고
새 기준으로 다시 검사할지 사용자가 선택할 수 있게 한다. 아직 구현되지 않은
신뢰구간·재검사·추가 질문 규칙은 작동 중인 기능처럼 안내하지 않는다.

## 자동 검증

`trait-map-safety-contract.test.ts`가 다음을 확인한다.

- 9개 데이터 등급의 중복과 기본 비공개
- 공개·비교·커뮤니티 사용의 별도 동의
- 원응답·과정·최근 상태·안전 신호의 공개 차단
- 즉시 안전 신호의 일시적 안전 라우팅 전용 처리
- AI 민감 추론과 외부 전송 기본 차단
- 결과를 만드는 6개 버전 필드
- 변경안 5건의 claim·block·contentKey 일치
- 원본 registry가 승인 상태로 변하지 않았음

## Gate A 완료 조건

이번 저장소 단계에서는 변경안과 실행 계약까지만 만든다. 다음 조건을 모두

2. 개인정보·AI 데이터 흐름과 철회·삭제 구현 검토
3. 지역별 지원 자원과 실패 경로 운영 검토

검토가 필요하므로 이번 변경안이 임의로 확정하지 않는다.
