# ENAKQ 성향지도 상세 템플릿 v1

- 버전: `ENAKQ-MAP-TEMPLATE-1.0-REVIEW`
- 상태: `RESEARCH_PREVIEW_NOT_CUSTOMER_CONTENT`
- 검토 주소: `/map/ENAKQ`
- 목적: 32개 성향 전체를 작성하기 전에 한 역할형의 정보 구조와 모바일 읽기 흐름을 검증한다.

## 화면 순서

1. 5글자 코드와 역할형 이름
2. 긴 설명을 읽는 세 가지 기준
3. 역할형 이름의 의미와 능력 오해 방지
4. 다섯 코드 자리의 접이식 설명
5. 처음 드는 생각과 실제 나타나는 반응의 측정 경계
6. 일상·가족·친구·연인·마음 가는 사람·업무 맥락 전환
7. 근거 범위와 실제 경험 확인 안내

화면은 큰 카드가 연속되는 구조를 피하고, 흰 배경·얇은 선·옅은 보조색과
필요한 부분만 펼치는 흐름을 사용한다. 상황별 설명은 별도 화면으로 이동하지
않고 같은 위치에서 전환해 탐색 부담을 줄인다.

## 데이터 재사용 원칙

- 다섯 자리의 기본 의미는 `candidateAxisCopy` 한 곳에서 가져온다.
- 역할형·상황별 설명은 `content atom`으로 나누고 안정적인 ID를 가진다.
- 관계별 조각은 `relationship_context` 없이는 개인·비교 리포트에서 호출하지 않는다.
- `처음 드는 생각`과 `실제 나타나는 반응`은 대표 코드로 추정하지 않는다.
- 역할형 이름을 리더십·관계 능력·궁합의 근거로 사용하지 않는다.
- 화면의 문구를 수정해도 claim과 원문 block 참조는 유지한다.

## 승인 뒤 확장 순서

1. ENAKQ 문구별 심리·측정·제품 안전·쉬운 문장 검토
2. Gate C와 사용자 읽기 검토에서 발견한 오해 반영
3. 고객 노출 가능한 atom만 `approved` 이후 `published`로 승격
4. ENAKQ를 `/map`의 32개 탐색 목록에 연결
5. 같은 템플릿을 사용해 나머지 31개 역할형을 한 개씩 확장
6. 승인된 공통 조각을 개인 리포트와 나와 비교하기에서 재사용

현재 화면은 구조 검증용이며, 사용자 검증 전 장면 문구를 확정된 성향 사실로
취급하지 않는다.
