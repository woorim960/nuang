# ENAKQ Gate B 5축·세부 성향 측정모형 v0.1

- 계약 버전: `ENAKQ-GATE-B-MEASUREMENT-MODEL.v0.1`
- 상태: `RESEARCH_CANDIDATE_NOT_FOR_VALIDATED_SCORING`
- 고객 코드 순서: `E/I · R/N · G/A · K/M · C/Q`
- 직접 뉴앙 정량 검증: 시작 전
- 운영용 최종 채점 기준: 미확정

## 이번 단계의 결론

다섯 코드 자리는 유지한다. 하지만 현재 50점 단순 분할과 facet 평균은 연구
개발용 계산일 뿐, 최종 글자 발급 기준으로 승인하지 않는다.

현재 대표 코드에 들어가는 세부 성향은 정확히 10개다.

- E/I: `SE-RE`, `SE-AI`
- R/N: `OE-AE`, `OE-CI`, `OE-IE`
- G/A: `RO-EC`
- K/M: `SM-EP`, `SM-OS`
- C/Q: `ER-IR`, `ER-WD`

`RO-RN 선택·존엄 존중`과 `SM-RL 맡은 일 이행`은 대표 코드에 합산하지
않는다. 두 신호는 독립성·안전성·비낙인성이 확인될 때까지 본인 전용 연구
후보다.

## 왜 이 작업이 성향지도보다 먼저인가

성향지도는 검사 결과에서 설명을 꺼내는 지식층이다. 측정층이 불분명하면
지도의 문장이 아름다워도 무엇을 근거로 말하는지 알 수 없다.

예를 들어 `K`를 실행·지속과 질서·구조의 평균으로 발급하면서 맡은 일
이행까지 포함한 설명을 보여 주면, 실제 채점 범위와 사용자 문구가 달라진다.
`A`를 RO 전체라고 부르면 선택 존중과 공감 능력까지 측정한 것처럼 넓어진다.
Gate B는 이처럼 점수보다 앞서 나가는 설명을 막는다.

## 외부 근거를 사용하는 범위

외부 연구는 뉴앙의 구조를 직접 검증하지 않는다. 다음 목적에만 간접적으로
사용한다.

1. 넓은 성격 영역 아래 구분되는 facet을 두는 위계 구조의 선례
2. domain과 facet이 서로 다른 설명력을 가질 수 있다는 근거
3. 내용 균형과 동의 편향을 별도로 다뤄야 한다는 측정 원칙
4. 번역이 아니라 목표 문화와 언어에서 의미 등가성을 확인해야 한다는 원칙
5. 문항의 관련성·포괄성·이해 가능성을 각각 검토해야 한다는 내용타당도 원칙

근거:

- [Soto & John, BFI-2](https://www.colby.edu/wp-content/uploads/2013/08/Soto_John_2017.pdf): 5개 domain 아래 15개 facet의 위계 모형과 내용 균형, 독립 검증 표본을 사용한 개발 절차
- [DeYoung, Quilty, & Peterson, BFAS](https://pubmed.ncbi.nlm.nih.gov/17983306/): Big Five domain과 세부 facet 사이의 10개 aspect 구조
- [International Test Commission 검사 번역·적응 지침](https://www.intestcom.org/files/guideline_test_adaptation_2ed.pdf): 사전 조건, 개발, 경험적 확인, 시행, 채점·해석, 문서화 전 과정의 언어·문화 적응
- [COSMIN 내용타당도 방법론](https://pmc.ncbi.nlm.nih.gov/articles/PMC5891557/): 관련성, 포괄성, 이해 가능성을 구분하고 목표 사용자의 이해를 직접 확인하는 원칙

BFI-2·BFAS 점수를 뉴앙 코드로 변환하지 않는다. A/G는 Agreeableness 전체가
아니며, K/M과 Q/C도 기존 척도의 전체 domain을 그대로 옮긴 것이 아니다.

## 1. E/I — 사람 사이 에너지

### 측정 후보 범위

- `SE-RE 함께할 때의 에너지`: 교류 중·후의 활력 변화와 관여
- `SE-AI 먼저 표현하기`: 필요한 의견·요청·선택지·진행 방법을 먼저 꺼내는 경향

### 포함하지 않는 것

- 친구 수, 인기, 파티 취향
- 소통·발표·설득·리더십 능력
- 낯선 사람에 대한 불안
- 집단의 권한과 문화적 발언 규범

### 경쟁 모형

1. 두 facet이 구분되면서 상위 SE를 이루는 모형
2. 두 facet은 관련되지만 대표 E/I 하나로 묶지 않는 모형

두 facet이 갈리면 평균으로 차이를 숨기지 않는다. 상위요인과 발급 규칙이
검증되기 전에는 E/I가 두 facet 모두를 대표한다고 확정하지 않는다.

## 2. R/N — 생각과 탐색

### 측정 후보 범위

- `OE-AE 미적 경험`: 분위기·음악·장면·환경의 미적 인상에 관심을 두는 정도
- `OE-CI 상상 확장`: 현재 정보 너머의 장면·이야기·가능성을 펼치는 정도
- `OE-IE 지적 탐구`: 필요한 답을 넘어 원리·배경·다른 설명을 탐색하는 정도

### 포함하지 않는 것

- 지능, 이해 속도, 학업 성취
- 창의적 성과와 아이디어 품질
- 예술 능력과 감각 정확도
- 현실성·실용 능력·상식·특정 가치관

### 경쟁 모형

1. 세 facet이 구분되면서 상위 OE를 이루는 모형
2. 세 facet은 관련되지만 상위 R/N으로 묶이지 않는 모형
3. 미적 경험·상상과 지적 탐구가 두 aspect를 이루는 모형

N은 MBTI N과 방향이 반대가 되지 않도록 높은 탐색 방향에 둔다. 그러나
MBTI N 결과를 뉴앙 N으로 변환하지 않는다. R은 익숙하고 구체적인 내용에
관심이 머무는 상대 방향이지 현실 판단 능력을 인증하는 글자가 아니다.

## 3. G/A — 관계에서 관심이 가는 곳

### 측정 후보 범위

대표 G/A에는 `RO-EC` 하나만 사용한다.

- G 방향: 관계 장면에서 일의 원인·결과·해결할 부분이 먼저 주의를 끔
- A 방향: 관계 장면에서 상대의 감정 상태와 필요가 먼저 주의를 끔

### 포함하지 않는 것

- 공감 능력, 착함, 배려, 예절
- 차가움과 문제 해결 능력
- 실제 나타나는 반응
- 선택·거절·경계를 존중하는 행동
- 상대의 마음과 호감

### 경쟁 모형

1. G와 A가 하나의 양극적 주의 방향을 이루는 모형
2. 해결 주의와 마음 주의가 부분적으로 독립적인 두 경향인 모형

현재 G/A는 넓은 RO domain이 아니라 `RO-EC 단일 대표 facet 축`이다.
`RO-RN 선택·존엄 존중`은 가치와 안전 영향이 크므로 G/A에 합산하지 않는다.
처음 든 생각과 실제 나타나는 반응도 같은 점수로 합치지 않는다.

## 4. K/M — 일상을 꾸리는 방식

### 측정 후보 범위

- `SM-EP 실행·지속`: 착수, 중단 뒤 복귀, 지속
- `SM-OS 질서·구조`: 물건·시간·절차의 정돈과 사전 구조화

### 포함하지 않는 것

- 성실함·책임감·도덕성
- 성과와 완성 능력
- 임상적 집행 기능
- M 방향의 유연성·적응력 자동 부여

### 경쟁 모형

1. EP와 OS가 구분되면서 상위 SM을 이루는 모형
2. EP와 OS를 대표 K/M 하나로 묶지 않는 모형
3. EP·OS만 대표 코드로 두고 RL은 외부 연구 신호로 유지하는 모형
4. RL까지 포함한 3-facet 상위 모형

현재 대표 K/M은 EP와 OS만 사용한다. `SM-RL 맡은 일 이행`은 EP와의
중복, 도덕적 자기평가, 사회적 바람직성, 추가 설명력을 확인하기 전에는
본인 전용 연구 신호다.

## 5. C/Q — 걱정과 감정 반응

### 측정 후보 범위

- `ER-IR 감정 동요`: 일상적 불편 정서가 활성화되는 속도와 크기
- `ER-WD 걱정·주저`: 불확실한 결과의 부정적 가능성이 반복되고 선택 전에 주저하는 정도

### 포함하지 않는 것

- 불안장애·우울·정신건강 진단
- 회복 속도와 회복력
- 감정 표현과 말투
- 위험 탐지 능력과 신중함
- 최근 수면·갈등·건강·경제 상태

### 경쟁 모형

1. IR과 WD가 구분되면서 상위 ER을 이루는 모형
2. 두 facet을 대표 C/Q 하나로 묶지 않는 모형
3. 평소 IR·WD와 최근 상태를 별도 변수로 두는 모형

최근 상태는 비채점 문맥으로 분리한다. Q는 정신건강이나 약함을 의미하지
않고 C도 회복력이 좋거나 감정 조절 능력이 높다는 뜻이 아니다.

## facet 불일치와 경계 결과

facet 불일치와 축 경계는 다른 상태다.

- `facet 불일치`: 같은 축 아래 세부 성향들이 서로 다른 방향을 보임
- `축 경계`: 측정오차를 고려할 때 대표 방향을 한쪽으로 구분하기 어려움
- `근거 부족`: 유효 응답, 문항 품질 또는 모델 근거가 발급 조건에 미달함

최종 규칙:

1. facet은 먼저 각각 계산한다.
2. 검증된 상위모형과 가중치가 있을 때만 대표 축을 계산한다.
3. facet이 갈리면 세부 점수를 먼저 보여 주고 대표 글자의 설명 범위를 낮춘다.
4. 경계가 추가 질문 뒤에도 해소되지 않으면 최종 5글자 코드를 발급하지 않는다.
5. 근거가 부족하면 부족한 축만 보완하며 다른 축까지 다시 묻지 않는다.

현재의 50점과 45~55 구간은 개발 시뮬레이션 규칙이다. 최종 경계 폭과 추가
질문 조건은 표본을 본 뒤 편의대로 정하지 않고, 정량 파일럿 전에 분석 계획과
오분류 비용을 사전 등록해 결정한다.

## 빠른 검사와 정밀 검사의 권위

### 빠른 검사

- 예비 5축 방향만 제공
- 세부 성향을 확정 문구로 설명하지 않음
- 정밀 검사와 동일한 확정 코드 권위를 갖지 않음
- 경계가 해소되지 않으면 해당 자리의 최종 글자를 만들지 않음

### 정밀 검사

- 승인된 문항과 유효 응답 조건을 충족한 facet만 계산
- 구조·신뢰도·경계 규칙을 통과한 축만 최종 글자 발급
- facet 불일치와 측정 불확실성을 결과에 함께 표시
- 직접 측정하지 않은 행동·관계 결과·능력을 설명하지 않음

## 정량 파일럿에서 비교할 것

1. 문항 내용의 관련성·포괄성·이해 가능성
2. 각 facet의 목표 적재와 인접 facet 교차 적재
3. 경쟁 모형의 적합도와 홀드아웃 재현
4. 정방향·역방향 문항의 방법 효과와 동의 편향
5. 내적 일관성뿐 아니라 검사-재검사와 측정오차
6. 최근 상태가 Q/C와 다른 축에 미치는 영향
7. 연령·성별·관계·직업 등 주요 집단의 측정동일성 또는 DIF
8. 추가 질문이 실제로 불확실성과 오분류를 줄이는지

특정 적합도 지수나 신뢰도 수치를 결과를 본 뒤 승인선으로 만들지 않는다.
모형, 주요 지표, 보조 지표, 제외 기준, 결측 처리, 다중 비교, 홀드아웃 방식을
분석 전에 고정한다.

## 현재 60문항과의 관계

현재 연구 베타는 10개 대표 facet마다 `HIGH 3 + LOW 3`, 총 60문항으로
구성돼 있다. `RO-RN`과 `SM-RL`은 이 채점 세트에 포함하지 않는다.

이 균형은 필요한 조건이지만 타당성의 증거는 아니다. 내부 검토에서
인지 인터뷰 우선 대상이 된 문항은 수정 상태를 유지하며, 60개 모두 목표
사용자가 의도한 상황과 단일 반응으로 이해하는지 확인해야 한다.

## 운영 잠금

다음을 모두 통과하기 전에는 Gate B 모델을 `validated` 또는 `active`로
바꾸지 않는다.

- 인지 내용타당도
- 정량 구조
- 신뢰도와 시간 안정성
- 공정성과 측정동일성

앱 계약과 DB 트리거가 네 관문을 동일하게 확인한다. 현재 후보 코드 체계와
연결된 문항 은행은 네 상태가 모두 `passed`가 아니면 `validated` 또는
`active`로 변경할 수 없다. 이미 출시된 상태에서 공정성 관문이 누락된
레코드가 발견되면 자동 보정하지 않고 마이그레이션을 중단해 수동 감사를
요구한다.

현재 앱에서 계산되는 코드는 제품 흐름 확인을 위한 candidate 결과다. 이
계약은 아직 검증되지 않은 개발 계산을 과학적으로 확정된 코드로 오인하지
않게 한다.
