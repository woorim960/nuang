# ENAKQ 내부 비판 검토 후 수정·연구 관문

- 상태: `INTERNAL_REVISION_PLAN_V0.1`
- 외부 전문가 검토: `NOT_STARTED`
- 원본 claim registry 자동 변경: `금지`

## 이번 검토가 바꾼 판단

초안의 보호 문구는 대체로 강하지만, 다섯 자리 점수만으로 구체적인 연락,
갈등, 돌봄, 업무 행동과 상대 마음을 설명하는 거리가 너무 멀다. 문장을 더
그럴듯하게 고치는 것만으로는 해결되지 않는다. 먼저 무엇을 실제로
측정했는지 좁히고, 그 다음에 어떤 장면까지 설명할 수 있는지 자료로
확인해야 한다.

내부 역할별 응답 303건을 158개 claim에 연결한 결과는 다음과 같다.

| 조정 결과                      | 개수 | 현재 처리                  |
| ------------------------------ | ---: | -------------------------- |
| `HOLD_FOR_MORE_EVIDENCE`       |   58 | 제품 지식·개인화 노출 보류 |
| `REVISE_BEFORE_NEXT_GATE`      |   50 | 원문과 분리한 변경안 작성  |
| `INTERNAL_ACCEPT_NOT_APPROVED` |   50 | 의미 보존, 운영 승인 금지  |
| `REJECT_OR_REWRITE`            |    0 | 없음                       |

이 수치는 초안이 100개 통과했다는 뜻이 아니다. 보류와 수정은 물론 내부 수용도
외부 독립 검토·한국어 인지 인터뷰·정량 파일럿 전에는 `research_only`다.

## 변경 불변 원칙

1. 내부 에이전트의 `accept`를 `APPROVED`로 바꾸지 않는다.
2. 모든 변경은 `before`, `after`, `reason`, `affected_blocks`,
   `affected_content_keys`, `validation_needed`를 남긴다.
3. 관계 성공, 호감, 상대 속마음, 능력, 정신건강을 코드에서 추론하지 않는다.
4. 현재 상태·구체 장면·처음 든 생각·실제 나타나는 반응은 기본
   `self_private`로 분리한다.
5. 안전한 일반 관계 원칙은 특정 코드의 고유 성향인 것처럼 쓰지 않는다.
6. 미검증 조합 문장은 단일 축 설명과 상호작용 가설로 분해한다.
7. 측정하지 않은 행동은 개인 결과가 아니라 사용자가 확인할 수 있는
   가능성 질문으로만 다룬다.

## 실행 순서

### Gate A — P0 안전·개인정보 변경안

먼저 다룰 critical claim은 다음 5개다.

- `ENAKQ.crush.boundary`: 지속되는 고통과 즉시 안전 위험의 안내를 분리한다.
- `ENAKQ.evidence.ai_boundary`: 외부 모델 전송, 결합 추론, 이차 이용, 삭제
  범위를 명시한다.
- `ENAKQ.evidence.privacy`: 대표 코드와 과정·상태·장면 자료의 동의를 분리하고
  철회 후 파생 비교 결과 처리 규칙을 둔다.
- `ENAKQ.evidence.versioning`: 실제 문항·채점·규칙 버전과 결과를 연결한다.
- `ENAKQ.stress.safety`: 자기관찰, 전문 지원 탐색, 즉시 안전 지원을 서로 다른
  제품 흐름으로 만든다.

동시에 `process.*`와 `stress.*`의 공개 범위를 점검한다. 이 단계에서는 고객
화면을 바로 늘리지 않고 데이터 분류 계약과 안전한 문구 후보를 만든다.

통과 기준:

- 민감 자료 기본값이 `self_private`다.
- 공유·비교는 필드별 명시적 동의와 철회 결과가 정의돼 있다.
- 위기 상황에서 성향 콘텐츠가 중단되고 적절한 지원 흐름으로 이동한다.
- 내부 안전 역할 재검토에서 critical 미해결 항목이 없다.

### Gate B — 다섯 축과 facet 측정모형

관계 문구를 더 만들기 전에 E·N·A/G·K/M·Q/C가 무엇을 포함하고 무엇을
포함하지 않는지 확정한다.

- E: 교류 후 에너지와 표현 시작을 하나의 상위 점수로 묶을 수 있는지
- N: 미적 경험, 상상 확장, 지적 탐구의 위계와 기존 개방성·지성 경계
- A/G: 상대 마음과 원인·해결의 주의 방향이 양극인지, 독립 경향인지
- K/M: 시작, 복귀, 지속, 정리, 계획을 한 축으로 묶을 수 있는지
- Q/C: 정서 활성화, 걱정·주저, 최근 상태, 표현, 회복의 분리

각 축에 포함·제외 정의, facet 불일치 표시, 경계 점수 추가 질문, 미분류와
재검사 규칙을 함께 작성한다.

통과 기준:

- 문항-축-facet 매핑에 고아 문항과 중복 구성개념이 없다.
- 반대 방향이 단일 연속선인지 독립 경향인지 비교 가능한 가설이 있다.
- 대표 글자와 facet이 충돌할 때 억지로 한 글자를 발급하지 않는다.
- 역할명은 채점 근거가 아닌 기억 보조라는 점이 데이터와 UI에서 분리된다.

### Gate C — 한국어 내용타당도·인지 인터뷰

2030 목표 사용자가 문항과 결과 문구를 의도한 기간·장면·구성개념으로
이해하는지 확인한다. 정해진 숫자를 채우는 방식이 아니라 주요 오해가 더
나오지 않을 때까지 반복한다.

반드시 확인할 오해:

- A가 착함·공감 능력, G가 차가움이라는 인상
- K가 성실함·책임감, M이 게으름이라는 인상
- N이 지능·창의성, E가 사교 능력이라는 인상
- Q가 불안장애·예민함·약한 회복력이라는 인상
- 역할명이 리더십·관계 능력·좋은 유형의 순위를 만든다는 인상
- `처음 드는 생각`과 `실제 나타나는 반응`을 구분할 수 있는지

통과 기준:

- 핵심 용어가 별도 설명 없이도 의도한 의미로 재진술된다.
- 가치·능력·임상·관계 결정론 오해가 허용 기준 아래다.
- 상황 라벨과 질문 본문이 중복되지 않고 하나의 장면으로 이해된다.

### Gate D — 5축·facet 정량 파일럿

문항 구조를 먼저 검증한다. 관계별 장면 서사와 32개 역할형 리포트는 이
단계의 주 검증 대상이 아니다.

- 경쟁 요인모형과 홀드아웃 교차검증
- 문항 정보, 응답 범주 작동, 국소의존, 바닥·천장 효과
- 검사-재검사와 최근 상태 변화의 분리
- 인접 성격·상태·능력과의 수렴·변별
- 주요 집단의 측정동일성 또는 DIF
- 경계 사용자의 추가 문항이 불확실성을 실제로 줄이는지 확인

통과 기준과 표본 크기는 결과를 본 뒤 정하지 않는다. 문항 수, 모형 복잡도,
예상 적재량, 허용 오분류 비용을 바탕으로 사전 시뮬레이션하고 분석 계획을
잠근다.

### Gate E — 과정 코드와 조합 가설

`처음 드는 생각 → 실제 나타나는 반응`은 대표 A/G와 분리된 과정 척도로
검증한다. `A→A`, `A→G`, `G→A`, `G→G`는 최소 장면 수, 일치도, 재검사
안정성, 미발급 기준이 확인되기 전 공개하지 않는다.

EA, NK, AQ, EQ, NQ, ENK, NAQ는 단일 축 문장을 합친 설명과 실제
상호작용 가설을 분리한다. 사전 등록, 주효과 대비 증분 설명, 다중검정 통제,
홀드아웃 재현을 통과한 조합만 지식 후보가 된다.

통과 기준:

- 과정 코드의 오분류율과 시간 안정성이 허용 범위다.
- 관찰된 응답 비율을 성격 확률처럼 표현하지 않는다.
- 조합이 단일 축 정보보다 반복 재현되는 추가 설명을 제공한다.

### Gate F — 관계·상황별 검증

가족, 친구, 연인, 호감, 업무, 공부 문장은 관계 종류만으로 발급하지 않는다.
관계 단계, 친숙도, 역할, 권력, 자원, 최근 상태, 직접 반복 행동을 필요한
범위에서 별도로 수집한다. 두 사람 비교는 양측 독립 응답과 공개 동의를
전제로 한다.

통과 기준:

- 관계 행동 claim이 대표 코드 외의 직접 자료를 갖는다.
- 다른 표본 또는 홀드아웃에서 증분 설명이 재현된다.
- 동일 코드와 다른 코드를 궁합·호감·성공 확률로 바꾸지 않는다.
- 안전하지 않은 관계에서 일반 소통을 우선 권하지 않는다.

### Gate G — 외부 독립 검토와 제품 승인

자격·신원·독립성이 확인된 성격심리, 심리측정, 관계/임상 안전 검토자가
수정된 패킷을 서로 독립적으로 검토한다. 불일치는 원응답을 보존한 채
조정하며, 다수결만으로 승인하지 않는다.

외부 검토 뒤에도 다음을 모두 통과해야 고객용 운영 지식이 된다.

- 한국어 인지 인터뷰 결과와 수정 이력
- 정량 파일럿과 재현 결과
- 개인정보·위기 대응 제품 검수
- 결과 문구가 실제 입력과 채점 버전에 연결된 자동 테스트
- claim 단위 승인자, 승인일, 근거 버전, 만료·재검토 조건

## 다음 구현 단위

Gate A 변경안 v0.2는 원본 158개 claim을 덮어쓰지 않은 별도 제안으로
작성했다. 5개 critical claim과 연관된 개인정보·AI 사용·위기 대응·결과 버전
계약은 `revisions/v0.2-gate-a/`와 코드 계약에 기록돼 있다.

다음 작업은 이 변경안의 내부 안전·개인정보 재검토와 **Gate B의 5축
측정모형 문서**다. Gate A의 외부 검토와 실제 데이터 수명주기 구현이 아직
남아 있으므로 v0.2 제안 문구를 운영 승인 상태로 병합하지 않는다.

성향지도 화면, 32개 역할형 이름, 나와 비교하기 리포트는 현재 동작을
깨뜨리지 않되 이 연구 초안을 검증된 사실처럼 확대하지 않는다. 검증된
claim이 늘어날 때마다 contentKey 단위로 교체할 수 있는 구조를 유지한다.
