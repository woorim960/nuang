# ENAKQ 관계·임상 안전 내부 비판 검토 요약

## 상태와 사용 한계

- 상태: `INTERNAL_AGENT_CRITIQUE`
- 검토 범위: `relationship_safety_W1.csv`부터 `W3.csv`까지 배정된 98개 claim
- 이 문서는 AI 내부 에이전트가 수행한 비판 검토이며 외부 전문가 검토나 임상 자문이나 제품 승인이 아니다.
- 이 검토의 `accept`는 현재 문구에서 추가적인 관계·임상 안전 문제를 발견하지 못했다는 뜻에 한정된다. 구성개념 타당도와 측정 타당도와 한국어 인지 인터뷰와 정량 파일럿과 외부 자격 전문가 검토를 대신하지 않는다.
- 외부 전문가 검토 상태는 계속 `EXTERNAL_REVIEW_NOT_STARTED`로 유지해야 한다.

## 중대한 안전 위험

### 1. 검증되지 않은 조합이 관계 불안을 유형화할 수 있음

`N·A·Q`가 연애나 친구 관계에서 좋지 않은 가능성에 오래 머문다는 설명은 애착 불안이나 집착으로 낙인찍지 않는 문구를 포함하고 있다. 그러나 조합 자체가 검증되지 않은 상태에서 개인 결과로 보여 주면 사용자는 늦은 답장과 짧은 말 같은 애매한 신호를 자기 유형의 증거로 반복 확인할 수 있다.

우선 수정 대상:

- `ENAKQ.crush.ambiguity`
- `ENAKQ.crush.context`
- `ENAKQ.crush.uncertainty`
- `ENAKQ.friend.distance`
- `ENAKQ.partner.ambiguity`
- `ENAKQ.stress.crossload`
- `ENAKQ.stress.worry`

이 문장들은 정량 검증 전 개인화 노출하지 않는다. 관계 장면을 직접 반복 측정한 자기보고가 있을 때에도 가능한 가설로 표시하고 상대의 마음과 의도를 추정하는 근거로 쓰지 않는다.

### 2. 현재 상태와 내적 과정이 비교 정보와 섞여 있음

걱정과 감정과 수면과 식사와 일상 기능과 처음 든 생각과 실제 반응은 공개 대표 코드보다 민감하다. 현재 계약의 `comparison_safe` 또는 `public_safe`만으로는 실제 공유 범위를 충분히 보호하지 못한다.

우선 수정 대상:

- `ENAKQ.evidence.privacy`
- `ENAKQ.process.boundary`
- `ENAKQ.process.comparison_boundary`
- `ENAKQ.stress.activation`
- `ENAKQ.stress.communication`
- `ENAKQ.stress.environment`
- `ENAKQ.stress.reactivity_recovery_boundary`
- `ENAKQ.stress.recovery`
- `ENAKQ.stress.self_check`
- `ENAKQ.stress.support_preference`

보호 원칙:

1. 현재 상태와 구체적 장면과 과정 기록은 `self_private`를 기본값으로 한다.
2. 비교에는 사용자가 직접 고른 안정적 지원 선호처럼 최소 필드만 별도 동의로 사용한다.
3. 대표 코드 공개 동의를 세부 과정 공개 동의로 확대 해석하지 않는다.
4. 공유 철회 시 새 접근을 막는 것에 그치지 않고 파생 비교 결과의 삭제 또는 재생성 정책을 둔다.
5. 보관 기간과 접근 기록과 삭제와 외부 모델 전송을 사용자가 확인할 수 있어야 한다.

### 3. 위기 안내가 원칙에 머물고 실행 경로가 부족함

성향 설명이 진단과 상담을 대신하지 않는다는 경계는 강하다. 그러나 일상 기능 저하와 자해·타해 위험과 폭력·위협을 같은 층위에서 다루면 필요한 행동의 긴급도가 흐려질 수 있다.

우선 수정 대상:

- `ENAKQ.crush.boundary`
- `ENAKQ.crush.self_check`
- `ENAKQ.stress.safety`
- `ENAKQ.growth.check`
- `ENAKQ.growth.Q_experiment`

제품 흐름은 다음처럼 분리한다.

- 평소 자기관찰: 선택형 기록과 언제든 중단
- 고통이 지속되거나 일상 기능이 어려움: 성향 설명보다 전문 지원 탐색
- 자해·타해 위험이나 폭력·위협이나 즉각 안전 문제: 성향 콘텐츠를 중단하고 위치에 맞는 긴급 지원과 신뢰할 수 있는 사람 연결

앱은 진단명이나 위험 확률을 추정하지 않는다. 지역별 지원 정보는 최신성과 접근 가능성을 별도로 관리해야 한다.

### 4. 가족 행동 설명이 역할 규범을 강화할 수 있음

가족 연락과 돌봄과 일정 관리를 E·A·K의 자연스러운 표현으로 보여 주면 성별과 출생 순서와 동거와 경제 활동에 따라 이미 불균등한 가족 노동을 성향 장점으로 정당화할 수 있다.

우선 수정 대상:

- `ENAKQ.family.care`
- `ENAKQ.family.contact`
- `ENAKQ.family.friction`
- `ENAKQ.family.role`

가족 콘텐츠는 사용자가 가족이라고 정의한 범위와 실제 역할과 자원과 권한과 반복 행동을 별도로 확인해야 한다. 연락을 적게 하는 것과 돌봄을 맡지 않는 것을 관심 부족으로 평가하지 않고 안전하지 않은 가족 관계에는 소통 지속을 권하지 않는다.

### 5. 역할명이 설명보다 먼저 능력 서사를 만들 수 있음

`관계를 여는 지휘자`를 능력이나 직업이 아니라고 설명해도 제목이 먼저 관계 관리와 지도력을 암시할 수 있다.

우선 수정 대상:

- `ENAKQ.misread.role`
- `ENAKQ.strength.role_boundary`

역할명은 검증된 claim과 시각적으로 분리된 보조 별칭이어야 한다. 인지 인터뷰에서 통솔력과 관계 능력과 좋은 유형의 인상이 반복되면 더 중립적인 이름으로 교체한다.

## 현재 유지할 보호 경계

다음 원칙은 제품 전반의 공통 가드레일로 유지할 가치가 있다.

- 같은 코드와 다른 코드를 호감과 궁합과 관계 성공 확률로 바꾸지 않는다.
- 코드로 애착 유형과 질투와 신뢰와 사랑의 크기와 관계의 미래를 예측하지 않는다.
- 상대의 게시물과 접속 시간과 팔로우 관계와 사진을 결합해 성향과 호감과 정신건강을 추정하지 않는다.
- 본인이 아는 내적 경험과 상대가 관찰한 행동을 구분하며 어느 한쪽을 항상 더 정확하다고 두지 않는다.
- 의도와 영향을 분리하고 코드로 갈등의 승패나 잘못을 정하지 않는다.
- A를 공감 능력과 착함으로, K를 책임감과 성실함으로, N을 지능과 창의성으로, E를 사교성으로, Q와 C를 정신건강과 회복력으로 확대하지 않는다.
- 고용과 승진과 배치와 보험과 신용과 교육 기회와 상대 위험성 판정에 뉴앙 코드를 쓰지 않는다.
- 폭력과 위협과 통제와 반복적 경계 침해가 있는 관계에는 일반적인 소통과 조율을 우선 권하지 않는다.
- 관계 리포트는 상대의 속마음을 맞히는 화면이 아니라 서로 직접 확인할 질문을 제공하는 화면이어야 한다.
- 성향 변화와 성장 기능은 자발적이고 중단 가능하며 본인 전용이어야 하고 코드 방향의 우열과 경쟁 등급을 만들지 않는다.

## 우선 수정과 보류 순서

### P0 — 사용자 노출 전에 반드시 수정

1. `ENAKQ.stress.safety`: 지역별 위기 지원을 포함한 실행 가능한 안전 흐름
2. `ENAKQ.evidence.privacy`와 스트레스·과정 claim: `self_private` 기본값과 철회·파생 삭제 계약
3. `ENAKQ.evidence.ai_boundary`: 결합 추론과 외부 전송과 이차 이용과 삭제 경계
4. `ENAKQ.crush.ambiguity`와 `ENAKQ.partner.ambiguity`와 `ENAKQ.friend.distance`: 검증되지 않은 관계 불안 조합 개인화 중단
5. `ENAKQ.family.role`과 `ENAKQ.family.friction`: 가족 역할 규범과 침해 행동 정상화 방지

### P1 — 인지 인터뷰 전에 수정

1. 대표 코드에서 연락과 제안과 돌봄과 계획 같은 관계 행동을 직접 도출하는 claim
2. Q 자기관찰에서 반추 증가 시 중단 조건과 전문 지원 연결
3. `관계를 여는 지휘자` 역할명의 능력·가치 오인 검사
4. 비교에서 과정 자료와 현재 상태를 별도 동의 없이 노출하지 않는 문구와 화면 구조

### 보류

- `ENAKQ.process.release_threshold`는 `insufficient_evidence`다. 세 쌍의 유효 응답과 두 번의 반복 차이라는 기준은 관계 안전 문구만으로 승인할 수 없다. 독립 심리측정 검토와 정량 파일럿에서 안정성과 오분류율을 확인하기 전 발급하지 않는다.

## 사용자 문구 원칙

1. **사람을 말하지 말고 측정한 범위를 말한다.**
   `ENAKQ는 답장이 늦으면 걱정해요` 대신 `이번 검사에서 불확실한 장면의 걱정 응답이 더 자주 나타났어요`처럼 범위를 밝힌다.

2. **가능한 장면과 실제 경험을 구분한다.**
   일반 예시는 `이런 모습이 나타날 수 있어요`로 제시하고 개인화 문장은 직접 측정되거나 사용자가 보고한 자료가 있을 때만 쓴다.

3. **상대의 마음과 의도를 대신 말하지 않는다.**
   연락과 표정과 말의 길이로 호감과 거절과 애착과 신뢰를 추정하지 않는다. `직접 본 행동`, `내가 떠올린 설명`, `아직 확인하지 않은 부분`을 구분한다.

4. **코드를 가치와 능력으로 바꾸지 않는다.**
   따뜻함과 성실함과 창의성과 리더십과 멘탈 같은 칭찬 또는 낙인 단어를 점수에서 자동 생성하지 않는다.

5. **관계의 보편 규칙을 만들지 않는다.**
   연락 빈도와 가족 돌봄과 연애 공개와 결혼과 성 역할과 갈등 회복의 한 가지 방식을 정상 기준으로 제시하지 않는다.

6. **선택과 거절과 말하지 않을 권리를 함께 쓴다.**
   질문을 권할 때에는 상대가 답하지 않거나 나중에 답하거나 거절할 수 있다는 선택권을 같은 수준으로 보여 준다.

7. **Q를 진단과 약함과 미화로 바꾸지 않는다.**
   Q는 불안장애와 예민함과 감정 조절 실패와 느린 회복을 뜻하지 않으며 위험 감지와 깊은 감수성과 큰 진심을 인증하지도 않는다.

8. **자기관찰에는 중단 기준을 붙인다.**
   기록이 생각을 더 길게 하거나 확인 행동을 늘리거나 일상 기능을 어렵게 하면 중단할 수 있어야 하며 필요한 경우 전문 또는 긴급 지원으로 연결한다.

9. **개인정보 범위를 문장 가까이에 보여 준다.**
   본인 전용과 공개 가능과 두 사람 비교 가능을 한 동의로 묶지 말고 세부 필드마다 공개 여부와 철회 결과를 확인할 수 있게 한다.

10. **안전 경계를 제품 행동으로 구현한다.**
    진단하지 않는다는 면책 문장만 두지 말고 위험 상황에서 콘텐츠를 중단하고 실제 지원으로 연결하는 화면과 데이터 계약을 갖춘다.
