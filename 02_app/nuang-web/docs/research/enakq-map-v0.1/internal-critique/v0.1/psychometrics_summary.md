# ENAKQ 성향지도 심리측정 내부 비판 검토 요약

- 상태: `INTERNAL_AGENT_CRITIQUE`
- 대상 패킷: `ENAKQ-EXPERT-PACKET.v0.1`
- 역할: `psychometrics`
- 검토 범위: W1–W3에 배정된 119개 claim 전수
- 결과 파일: `psychometrics_review.csv`
- 작성 주체: 내부 AI 에이전트
- 외부 전문가 검토 또는 승인: **아님**

이 문서는 실제 자격과 독립성이 확인된 심리측정 전문가의 검토를 대신하지 않는다. 아래 판정은 외부 승인 상태로 병합하지 않고, 문구·측정·파일럿 설계를 더 엄격하게 만드는 내부 반론 자료로만 사용한다.

## 1. 전수 점검 결과

| 판정                    |  수 | 해석                                                                              |
| ----------------------- | --: | --------------------------------------------------------------------------------- |
| `accept`                |  35 | 현재 의미와 범위가 측정 경계 또는 방법론·안전 정책으로 유지 가능                  |

| `reject`                |   0 | 이번 내부 검토에서는 핵심 의미 자체를 폐기하기보다 보류 후 검증할 대상으로 분류함 |

`accept`는 운영 승인이 아니다. 대부분은 “코드로 능력·관계 결과·속마음·회복을 추정하지 않는다”는 경계, 또는 아직 하지 않은 검증을 분명히 말하는 방법론 claim이다. 성향 정의와 구체 행동 문장이 검증됐다는 뜻이 아니다.

## 2. 구조적 측정 문제

### 2.1 다섯 자리와 세부 성향의 위계가 아직 입증되지 않았다

E는 함께할 때의 에너지와 표현 시작, N은 미적 경험·상상 확장·지적 탐구, K는 시작·복귀·지속과 정리·계획, Q는 감정 동요와 걱정·주저를 함께 묶는다. A/G는 관계에서 마음과 원인·해결 중 주의가 향하는 곳이라는 독자 축이다. 이들이 각각 하나의 상위 점수를 형성하는지, 서로 다른 facet을 억지로 합친 것인지, 반대 방향이 실제 양극인지 ENAKQ 자료로 확인되지 않았다.

따라서 대표 글자 발급 전에 다음이 필요하다.

1. 대상 사용자가 각 문항을 의도한 구성개념으로 이해하는지 확인
2. 상관된 facet 모형, 상위요인 모형, bifactor 모형 등 경쟁 구조 비교
3. 총점이 facet 차이를 가리지 않는지 확인
4. 두 방향이 단일 연속선인지, 서로 독립적인 경향인지 비교
5. 경계 점수와 facet 불일치의 표시·추가 질문·미분류 규칙 사전 확정

### 2.2 대표 코드에서 구체 행동으로의 거리가 너무 멀다

가족 연락, 친구 제안, 연인 연락과 갈등, 업무 발언, 공부 방식처럼 맥락 의존적인 행동을 `domain_scores;relationship_context`만으로 발급할 수 없다. 관계 단계, 역할, 기회, 자원, 상대 반응, 현재 상태가 직접 입력되지 않으면 가능한 예시를 개인의 반복 경향처럼 보여주게 된다.

행동 claim에는 최소한 다음 중 해당 정보를 요구해야 한다.

- 행동을 직접 묻는 다상황 반복 자료
- 관련 facet 점수
- 관계 종류뿐 아니라 친숙도·기간·역할·동거·상호작용 조건
- 최근 상태와 상황 강도
- 두 사람을 설명할 때 양측의 독립 응답과 공개 동의
- 가능한 경우 행동 기록 또는 상대 자료와의 준거 확인

### 2.3 “처음 드는 생각 → 실제 나타나는 반응”은 의미 있는 가설이지만 별도 척도다

생각과 행동을 구분하는 개념 자체는 유지할 가치가 있다. 그러나 회고식 자기보고가 실제 “첫 순간”을 얼마나 잘 포착하는지, 동일 장면의 두 응답이 서로 다른 구성개념인지, 사회적 바람직성에 얼마나 좌우되는지 아직 모른다.

특히 “최소 3쌍의 유효 응답과 2회 반복”은 현재 임의 임계값이다. 4개 중 3개를 75% 성격 확률처럼 표시해서도 안 된다. 문항 강도 등가성, 분류 일치도, 검사-재검사, 결측과 무성의 응답의 영향, 오분류 비용을 시뮬레이션과 파일럿으로 보정해야 한다. 과정 자료는 기본 `self_only`로 유지하고 비교 화면에는 별도 동의와 신뢰도 기준이 필요하다.

### 2.4 조합 서사는 주효과를 나열한 것이지 상호작용 근거가 아니다

EA, NK, AQ, EQ, NQ, ENK, NAQ 설명은 각 글자의 의미를 한 장면에 나열한 가설이다. 이것만으로 통계적 상호작용이나 해당 사용자의 행동을 지지하지 않는다. 표본을 본 뒤 그럴듯한 조합만 선택하면 과적합과 다중검정 위험이 크다.

조합 claim은 사전 등록한 소수 가설, 충분한 표본, 주효과 대비 증분 설명, 홀드아웃 재현, facet 기반 분석을 통과하기 전까지 운영 보류한다.

### 2.5 Q는 평소 성향·최근 상태·표현·회복·임상을 가장 엄격하게 분리해야 한다

Q의 활성화, 실제 표현, 걱정 내용, 회복 속도, 필요한 지원은 서로 다른 정보다. 수면 부족·갈등·경제적 어려움 같은 최근 상태가 평소 응답을 올릴 수 있다. Q를 불안 진단, 예민함, 약한 회복력, 위험 탐지 능력으로 바꾸지 않는 현재 경계는 유지한다.

평소 성향 점수와 최근 며칠·2주 상태 척도를 분리하고, 회복·지원은 사용자가 직접 선택하거나 별도 반복 자료가 있을 때만 다룬다. 임상 선별이나 치료 경과 지표로 사용하지 않는다.

### 2.6 자기일치감은 정확성 증거가 아니다

역할명과 관계별 서사는 사용자가 자신과 잘 맞는다고 느끼기 쉽지만, 자기일치감만으로 구조·준거·예측 타당도를 주장할 수 없다. “관계를 여는 지휘자”는 비채점 기억 보조로만 두고 리더십·통솔력·통합 능력으로 해석되는지 인지 인터뷰해야 한다.

### 2.7 불확실성·버전 문구는 실제 채점 구현과 연결돼야 한다

측정 오차와 시간 변동을 언급하는 것은 적절하다. 다만 신뢰구간, 경계 처리, 추가 질문, 재검사, 과거 결과 재해석 규칙이 아직 실행되지 않았다면 “현재 작동하는 기능”이 아니라 “설계 원칙”으로 표시해야 한다. 각 결과에는 문항·채점·규칙 버전과 유효 문항 수를 연결해야 한다.

## 3. 운영 승인 차단 조건

다음 조건을 통과하기 전에는 관련 claim을 `APPROVED` 또는 일반 고객용 사실 문장으로 올리지 않는다.

### 대표 5축과 facet

- 한국어 내용타당도: 관련성·포괄성·이해 가능성
- 경쟁 요인구조 비교와 교차검증
- 문항 수준 품질, 국소독립, 응답 범주 작동, 바닥·천장 효과
- 내적 일관성만이 아닌 검사-재검사와 시간 변동
- 인접 성격·상태·능력 구성개념과의 수렴·변별
- 연령·성별·관계·직업 등 주요 집단의 측정동일성 또는 DIF
- 총점·facet·경계·미분류·추가 질문의 사전 채점 규칙

### 관계·가족·친구·호감·업무·공부 문장

- 해당 행동 또는 선호의 직접 반복 측정
- domain 점수를 넘어선 증분타당도
- 양측 관계 문장은 dyadic 자료와 공개 동의
- 관계 성공·호감·헌신·성과·능력을 예측하지 않는 준거 제한
- 다른 표본 또는 홀드아웃에서의 재현

### 과정 코드

- 첫 주의와 실제 반응의 내용·구조·변별타당도
- 같은 강도의 다상황 문항과 순서·기억 편향 통제
- 분류 임계값의 오분류율·안정성·결측 민감도
- self_only 기본값, 공개 동의, 삭제·갱신 정책
- 관찰 비율과 성격 확률을 구분하는 UI 문구

### 조합과 역할명

- 사전 등록한 상호작용 가설과 다중검정 통제
- 주효과·facet 대비 증분 설명과 재현
- 능력·덕목·관계 결정론 오해율이 허용 기준 아래임을 인지 인터뷰로 확인

### 규준과 백분위

- 목표 모집단을 정의한 대표성 있는 표집
- 가중치·표본오차·집단별 불확실성 문서화
- 앱 이용자 비율과 한국 인구 규준을 명시적으로 분리

## 4. 정량 파일럿 우선순위

### P0. 파일럿 전에 내용타당도와 한국어 인지 인터뷰

- 축·facet별 전문가 검토와 대상 사용자 인터뷰
- 질문을 어떤 기간·관계·장면으로 이해했는지 확인
- A/G, K, Q처럼 새 의미가 큰 자리의 누락 내용과 양극성 집중 확인
- 역할명·코드 문구가 능력·착함·정신건강으로 오해되는지 확인

### P1. 문항 수준 예비 조사와 경쟁 구조 비교

- 사전에 정의한 표집 계획과 결측·무성의 응답 규칙
- EFA용·CFA용 또는 홀드아웃 표본 분리
- 상관 facet·상위요인·양방향 독립 모형 비교
- 총점보다 facet이 더 안정적이면 대표 글자 발급 규칙 재검토
- 필요한 표본 크기는 고정 숫자를 먼저 정하지 말고 문항 수·요인 수·예상 적재량·모형 복잡도에 따른 시뮬레이션으로 산정

### P2. 문항반응과 공정성

- 5점 응답 범주의 순서성과 정보량 확인
- 국소의존·중복 장면·반대 문항 방식 효과 확인
- 주요 사용자 집단의 DIF·측정동일성 확인
- 경계 사용자에게 추가 문항이 실제 분류 불확실성을 줄이는지 적응형 시뮬레이션

### P3. 시간 안정성과 상태 오염

- 적절한 간격의 검사-재검사
- 최근 스트레스·수면·역할 변화와 평소 점수의 분리
- 경계 사용자와 facet 불일치 사용자의 이동 규칙 확인

### P4. 과정 코드 전용 파일럿

- 동일 장면에서 첫 주의와 실제 반응을 충분히 떨어뜨려 측정
- 경험표집 또는 사건 기반 기록과 회고식 응답 비교
- AA·AG·GA·GG 분류 안정성, 최소 장면 수, 오분류 비용 보정
- 개인 리포트 이해도와 비교 공개에 대한 별도 동의 실험

### P5. 행동·관계·업무 준거 검증

- 가족·친구·연인·호감·업무를 서로 다른 준거 연구로 분리
- 자기보고, 가능한 상대 보고, 반복 행동 자료를 구분
- 코드가 상황·역할·기회 이상으로 무엇을 추가 설명하는지 평가
- 예측이 아닌 대화 프롬프트의 유용성은 별도의 제품 실험으로 검토

### P6. 규준은 마지막 단계

초기 앱 표본으로 희귀도·상위 퍼센트·한국인의 분포를 만들지 않는다. 척도 구조와 점수 해석이 안정된 뒤 목표 모집단과 표집 틀을 정의하고 규준 연구를 별도로 수행한다.

- 공부: `ENAKQ.study.pattern`, `ENAKQ.study.support`
- 업무: `ENAKQ.work.collaboration`, `ENAKQ.work.evaluation`, `ENAKQ.work.followthrough`, `ENAKQ.work.ideation`, `ENAKQ.work.learning`, `ENAKQ.work.voice`

`revise` 28개 claim도 수정 요구를 반영하고 해당 게이트를 통과하기 전에는 현재 문구 그대로 승인하지 않는다. 특히 과정 표시, 비교 문구, 지원·성장 프롬프트, 역할명, 불확실성 문구는 “좋은 의도”만으로 운영 승인하지 않는다.

## 6. 결론

현재 초안의 가장 강한 부분은 무엇을 말하면 안 되는지 비교적 구체적으로 정리했다는 점이다. 가장 약한 부분은 대표 축과 facet의 구조가 아직 검증되지 않은 상태에서 관계·업무·과정 장면을 풍부한 개인 서사로 확장한 부분이다.

따라서 다음 개발 우선순위는 문장을 더 많이 만드는 일이 아니다. 먼저 다섯 축과 facet의 내용타당도·구조·안정성·불확실성 규칙을 확인하고, 그 뒤 과정 코드와 행동·관계 claim을 별도 파일럿으로 좁혀야 한다. 외부 독립 전문가 검토와 실제 사용자·정량 자료가 확보되기 전까지 이 결과의 상태는 계속 `INTERNAL_AGENT_CRITIQUE`로 유지한다.
