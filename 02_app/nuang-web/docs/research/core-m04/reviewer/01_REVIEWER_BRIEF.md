# NUANG 전 코어 문항 독립 검토자 안내

문서 버전: `m04-core-expert-kit.v0.1`  
대상: 승인 후 배정된 M04 reviewer  
주의: 이 문서만으로 모집·참여·데이터 제공이 성립하지 않는다.

## 1. 목적

뉴앙은 자기이해용 성향 검사를 개발하고 있다. 이번 과제는 문장이 매력적인지 평가하는 일이 아니라, `상황 라벨 + 질문`이 의도한 하나의 성향을 다른 성향·능력·환경 효과와 구분해 묻는지 확인하는 독립 검토다.

150개 후보는 50문항씩 세 회차로 나뉜다. Stage 1에서는 실제 후보 ID, 목표 영역, HIGH/LOW, 공개·과정·비공개 역할을 알 수 없다.

## 2. 독립성

- 다른 검토자와 문항·정답 추정·평가를 공유하지 않는다.
- 기존 뉴앙 문서나 원척도를 찾아 target을 복원하지 않는다.
- 작성자의 의도에 맞춰 좋게 해석하지 않는다.
- 실제 2030 사용자가 보는 한국어 그대로 평가한다.
- 어느 방향도 더 착함·건강함·유능함을 뜻한다고 가정하지 않는다.
- 사전 노출·이해상충·잘못된 파일을 발견하면 응답을 멈추고 운영자에게 알린다.

## 3. 두 단계

### Stage 1 — blind mapping

상황 라벨과 질문만 보고 첫·두 번째 구성개념, 방향, 명확성, 단일 반응성, 보편성, 척도 적합성, 바람직성, 오염 위험을 판단한다. W1·W2·W3를 모두 제출하고 잠근다.

### Stage 2 — target 공개 검토

세 Stage 1 잠금 뒤 같은 문항의 target·방향·내부 역할이 공개된다. 관련성, 방향 적합성, 범위 기여, 인접 성향 분리, 다음 단계 권고를 평가한다. Stage 1은 수정하지 않는다.

## 4. 일반 사용자 응답 맥락

사용자는 최근 6개월의 평소 모습을 떠올리고, 제시된 상황에서 해당 반응이 실제로 얼마나 자주 나타나는지 1~5로 답한다. 경험 부족·상황 차이·문장 이해·응답 거절에는 별도 `이 상황은 답하기 어려워요`가 있다.

검토자는 자신의 성향에 답하지 않고, 다양한 사용자가 같은 종류의 장면과 반응으로 해석할 수 있는지를 평가한다.

## 5. 즉시 알려야 하는 문제

- Stage 1에 실제 후보 ID·target·HIGH/LOW·score role이 보임
- 세 blind 회차 잠금 전에 Stage 2를 받음
- 다른 검토자의 응답이나 집계를 받음
- 배정되지 않은 slot·wave를 받음
- 중복·누락·손상 또는 참여 중단·삭제 요청이 생김

노출 사고 뒤 추측으로 계속 진행하지 않는다.
