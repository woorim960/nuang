# NUANG MVP Smoke Checklist

작성일: 2026-07-10 KST

이 문서는 MVP 출시 후보를 묶기 전에 매번 같은 순서로 확인할 수 있는 수동 smoke 기준이다. 자동 QA는 `npm run qa:mvp`로 수행하고, 실제 앱 경험은 아래 흐름을 브라우저에서 확인한다.

## 전제

- 로컬 서버는 `http://localhost:3000`에서 열린다.
- Supabase migration은 현재 작업 큐에 완료로 기록된 항목까지 적용되어 있어야 한다.
- 로그인 세션이 필요한 항목은 실제 Kakao 또는 Google 세션으로 확인한다.
- 네이버 로그인, 그룹 검사, 유료 상세검사, 민감/치료성 검사존은 MVP smoke 대상이 아니다.
- 테스트 중 생성한 글, 댓글, 좋아요, 저장, 비교 리포트는 확인 후 정리한다.

## 자동 QA

출시 후보 전에는 아래 명령이 통과해야 한다.

```bash
npm run qa:mvp
```

부분 변경 중에는 최소 아래 명령을 통과시킨다.

```bash
npm run harness:check
npm run typecheck
npm run lint
npm run test
npm run build
```

## 핵심 화면 Smoke

홈:

- `/home`이 모바일 폭에서 바로 열린다.
- 홈은 오늘 시작점, 내 뉴앙 코드 요약, 피드 미리보기, 다음 행동 하나에 집중한다.
- 하단 메뉴와 중복되는 전체 route card 묶음이 보이지 않는다.
- 피드 미리보기는 전체 피드가 아니라 일부 preview만 보여준다.

검사:

- `/assessments`에서 빠른 코어, 정밀 코어, 무료 주제 검사, 별난 성향 연구소가 구분된다.
- 정밀 코어는 유료처럼 보이지 않는다.
- 실행 가능한 무료 주제 검사는 10개만 열린다.
- 아직 열리지 않은 검사는 추후 업데이트로 표시한다.
- 검사 진행 화면은 한 문항 집중형이며 이전/다음 흐름이 깨지지 않는다.

피드:

- `/feed`는 글, 오늘의 질문, 밸런스 게임만 작성 선택지로 보여준다.
- 리포트 공유는 피드 작성기 선택지에 없다.
- 비로그인 작성 시 로그인 화면으로 이동한다.
- 피드 카드에서 좋아요, 댓글, 저장, 관심 없음이 정상 반응한다.
- 밸런스 게임은 투표 후 전체 비율을 보여준다.
- `뉴앙 코드별 통계 보기`는 전체 투표 2명 이상이면 진입 가능하고, 1명 이상 투표한 코드는 비율을 보여준다.
- 코드별 통계는 누가 투표했는지 보여주지 않는다.

성향지도:

- `/map`은 개인 리포트 목록이 아니라 32개 뉴앙 코드 정보 화면이다.
- 사용자-facing 문구는 `뉴앙 코드`, `코드 지도`, `코드 자리`, `세부 신호`를 사용한다.
- `5축`, `10축`, `FOAMT`, `공개 코드` 같은 표현이 보이지 않는다.
- 코드 이름은 `불꽃의 온기 탐험가`, `숲의 새길 개척가`처럼 모티프의 이름 형태다.

마이:

- `/my`는 내 뉴앙 코드와 대표 성향이 중심이다.
- 비회원은 비회원 전용 안내와 로그인 진입만 간결하게 본다.
- 로그인 사용자는 `내 리포트`와 `설정`으로 이동할 수 있다.
- `계정 결과`, `로컬 결과`, `기기 사본`, `공유 링크 관리`, `공개 코드`가 보이지 않는다.
- `/my/reports`에서 검사 결과와 1:1 비교 리포트를 한곳에서 다시 열 수 있다.

## 주요 기능 Smoke

결과 리포트:

- 결과 화면 제목은 `결과 리포트`다.
- 대표 뉴앙 코드와 코드 이름이 첫 화면에서 눈에 띈다.
- 공유 영역은 `공유 주소 복사`와 `피드에 공유`를 제공한다.
- 공유 주소 설명은 30일 자동 만료를 말한다.
- 브라우저 권한 문제로 자동 복사가 실패해도 생성된 공유 주소를 화면에 보여주고 수동 복사를 안내한다.
- 공개 공유 화면은 저장 시점의 예전 이름이 아니라 현재 뉴앙 코드 사전의 코드 이름을 우선 사용한다.
- 피드에 공유한 리포트 preview는 피드 글이 삭제되기 전까지 유지된다.
- 직접 응답, 원점수, OAuth 정보, 이메일, 민감 검사 결과는 공유 preview나 공유 리포트에 보이지 않는다.

1:1 비교:

- 피드에서 상대 프로필을 클릭하면 프로필 이미지, 표시명, 뉴앙 코드, 코드 이름이 보인다.
- `나와 비교하기`를 누르면 비교 리포트가 생성된다.
- 비교 리포트는 서로의 현재 뉴앙 코드를 기준으로 한다.
- 같은 코드 자리와 다른 코드 자리가 서로 다른 피드백으로 나타난다.
- 비교 리포트에는 직접 응답, 원점수, 민감 항목이 보이지 않는다.
- 생성한 비교 리포트는 `/my/reports`에서 다시 열 수 있다.
- 방금 생성한 테스트 비교 리포트만 삭제할 수 있다.

무료 주제 검사:

- 결과 화면에 내부 키 `RO-EC`, `RO-RN` 같은 값이 보이지 않는다.
- 세부 신호는 한국어 이름으로 표시한다.
- 결과는 현재 대표 성향을 더 정교하게 이해하는 참고 결과로 설명한다.
- 피드 공유는 공개 요약만 전송한다.

## GO / NO-GO

GO 조건:

- 고객 공개용 `measurement_release_id`와 승인된 `E/I · R/N · G/A · K/M · C/Q` runtime이 일치
- `npm run qa:mvp` 통과
- 핵심 화면 smoke 통과
- 로그인 기반 피드 작성과 1:1 비교 생성이 통과
- 공유 주소 생성과 피드 공유가 통과
- 직접 응답과 민감 정보가 공개 화면에 노출되지 않음

NO-GO 조건:

- legacy `S/T · C/V · F/O · D/A · P/E` 또는 provisional 문항·채점이 공식 대표 결과로 노출됨
- measurement release ID 없는 결과가 공개 프로필·공유·피드·비교로 전달됨
- 공개 코드 입력 또는 발급 UI가 다시 노출됨
- 비로그인 사용자가 회원 기능을 사용할 수 있음
- 피드 작성 실패 후 사용자가 다음 행동을 알 수 없음
- 비교 리포트가 같은 문장 반복으로 보임
- 공유 리포트에 직접 응답, 원점수, 민감 정보가 보임
- 정책 문서가 미완성인 상태로 실제 배포를 진행하려 함
