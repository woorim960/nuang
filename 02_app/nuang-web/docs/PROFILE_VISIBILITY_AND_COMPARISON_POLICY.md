# Profile Visibility and 1:1 Comparison Policy

작성일: 2026-07-04 KST

## 핵심 원칙

1:1 비교는 상대에게 매번 승인 요청을 보내는 기능이 아니다. 상대가 미리 설정해 둔 공개 프로필 범위 안에서 바로 비교한다.

단, 비교는 공개된 요약만 사용한다. 직접 응답, 원점수, 전체 점수 벡터, 민감 검사, 도움 연결 기록, 계정 식별 정보는 기본 비공개이며 공개 범위를 넘어 추정하지 않는다.

RO-RN 선택·존엄 존중은 사용자 본인의 정밀 리포트에만 제공하는 `본인 전용 상세 신호`다. 대표 G/A에 합산하지 않으며 공개 프로필·피드·공유·비교에서 기본 제외한다.

## 공개 수준

| 수준      | 의미                                             | 초기 MVP 처리                      |
| --------- | ------------------------------------------------ | ---------------------------------- |
| `public`  | 뉴앙 프로필/코드/링크로 접근한 사람이 볼 수 있음 | 기본 프로필과 성향지도 요약 기본값 |
| `limited` | 지정된 링크, 코드, 관계 범위로만 접근            | 후속 구현 후보                     |
| `private` | 본인만 볼 수 있음                                | 민감·개인·원자료 기본값            |

주의:

- 여기서 `public`은 검색엔진 인덱싱 공개를 뜻하지 않는다.
- 공개 프로필 페이지와 공유 링크에는 `noindex` 유지 여부를 별도 출시 게이트에서 확인한다.

## 기본 공개 매트릭스

| 항목                           | 기본값    | 1:1 비교 사용 | 이유                                                                 |
| ------------------------------ | --------- | ------------- | -------------------------------------------------------------------- |
| 기본 프로필                    | 공개      | 가능          | 비교 상대 식별과 프로필 표시                                         |
| 대표 성향                      | 공개      | 가능          | 비교의 핵심 요약                                                     |
| 성향지도 5영역                 | 공개      | 가능          | 공통점·차이 설명의 기본 입력                                         |
| 승인된 코드 기여 상세 요약     | 공개      | 가능          | 대표 코드의 근거가 되는 비민감 상세만 사용                           |
| RO-RN 선택·경계 존중 상세 신호 | 본인 전용 | 미사용        | 관계 안전·낙인·사회적 바람직성 위험이 있어 상대 공개와 비교에서 제외 |
| 빠른 코어 예비 결과            | 비공개    | 미사용        | 예비 결과는 비교 기준으로 부족                                       |
| 별난 성향 연구소 결과          | 비공개    | 미사용        | 생활형 주제이며 코어 비교와 분리                                     |
| 직접 문항 응답                 | 비공개    | 차단          | 원자료 보호                                                          |
| 원점수·전체 점수 벡터          | 비공개    | 차단          | 과도한 추론 방지                                                     |
| 민감 검사·위기·치료 주제       | 비공개    | 차단          | 고보호 데이터                                                        |
| 도움 연결 이용 기록            | 비공개    | 차단          | 위기·복지 이용 기록 보호                                             |
| 연령대 등 인구통계             | 비공개    | 미사용        | 비교 목적에 불필요                                                   |
| 계정 식별 정보                 | 비공개    | 차단          | 개인정보 보호                                                        |

## 1:1 비교 흐름

1. 내 정밀 코어 결과가 있어야 한다.
2. 상대의 공개 프로필 스냅샷이 있어야 한다.
3. 시스템은 상대가 공개한 항목만 읽는다.
4. 비공개 항목은 `비공개`로 처리하고 추정하지 않는다.
5. 결과는 공통점, 차이, 오해 조건, 대화 질문, 조절 선택지만 제공한다.
6. 궁합 점수, 관계 성공 예측, 순위, 위험 인물 판정은 제공하지 않는다.

## 설정 UX 요구사항

- 마이 > 공개 범위 설정에서 항목별 토글을 제공한다.
- 기본 프로필, 대표 성향, 승인된 코드 기여 성향지도는 기본 `public`으로 둔다.
- 민감 항목은 기본 `private`이며 기본 토글이 꺼져 있어야 한다.
- 민감 항목을 공개하는 UX는 MVP에서 제공하지 않는다.
- `본인 전용`은 사용자 자신의 정밀 리포트에는 보여주되 다른 사람에게 자동 공개하지 않는다는 뜻이다.
- 공개 범위를 바꾸면 새 비교부터 즉시 반영한다.
- 이미 생성된 공개 파생 결과는 공개 범위 변경 시 접근 재평가 또는 비활성화 대상이다.

## 데이터 설계 요구사항

필요 엔터티 후보:

- `profile_visibility_setting`: 계정별 현재 공개 범위 설정
- `profile_visibility_release`: 공개 범위 정책 버전
- `profile_public_snapshot`: 비교에 사용할 공개 요약 스냅샷
- `comparison_report`: 내 결과와 상대 공개 스냅샷의 비교 결과
- `visibility_audit_event`: 공개 범위 변경, 비공개 전환, 삭제, 접근 차단 이벤트

서버 구현 원칙:

- 비교 생성 시 상대의 현재 공개 스냅샷 버전을 기록한다.
- 직접 응답과 원점수는 공개 스냅샷에 넣지 않는다.
- 비공개 전환, 결과 삭제, 계정 삭제 시 관련 공개 스냅샷과 비교 파생 결과를 재평가한다.
- 공개 스냅샷 payload에는 이메일, provider id, token hash, 직접 응답, raw score payload를 넣지 않는다.

## 공개 payload 계약

코드 기준:

```text
src/features/together/public-comparison-contract.ts
src/features/together/public-comparison-contract.test.ts
```

공개 프로필 스냅샷 계약:

- contract version: `public-profile-snapshot.v0.1`
- visibility policy: `profile-visibility.v0.1`
- 포함: 기본 표시 이름, motif, 대표 성향 코드/이름, 5자리 summary, 승인된 코드 기여 상세 summary
- 제외: RO-RN 본인 전용 상세 신호, 이메일, provider id, 계정 식별 정보, 직접 응답, 원점수 payload, 민감 검사, 도움 연결 기록, token hash

1:1 비교 리포트 계약:

- contract version: `public-comparison-report.v0.1`
- 상대 추가 승인 요청 없이 `published_profile_scope`만 사용한다.
- 제공: 공통점, 차이점, 대화 시작 질문, 조절 가이드
- 미제공: 궁합 점수, 관계 성공 예측, 순위, 비공개 항목 추정

공개 프로필 카드 계약:

- contract version: `public-profile-card.v0.1`
- 공개 스냅샷을 입력으로 사용한다.
- 제공: 표시 이름, 캐릭터 motif, 대표 성향 코드/이름, 승인된 5자리 주요 요약
- 미제공: RO-RN 본인 전용 상세 신호, 직접 응답, 원점수, 이메일/provider id, 계정 식별 정보, 민감 검사, 도움 연결 기록, token hash
- 로컬 preview는 `아직 미공개` 상태를 표시하고 실제 공유/게시 기능을 열지 않는다.

비교 생성 write 순서:

```text
1. ensure_viewer_full_core
2. read_target_public_snapshot
3. validate_snapshot_policy_version
4. project_allowed_fields
5. build_comparison_report
6. record_comparison_audit_event
```

## 나중에 사용자 확인이 필요한 선택지

현재 구현 전제는 위 매트릭스로 충분하다. 다만 실제 서버 기능을 열기 전에 아래는 최종 선택이 필요하다.

- 공개 수준 이름: `전체 공개`, `코드/링크 공개`, `비공개`처럼 사용자에게 보일 문구
- 프로필 검색 기능을 MVP에 포함할지 여부
- 비교 열람 기록을 상대에게 보여줄지 여부
- 14~18세 사용자에게도 기본 프로필·성향지도 공개 기본값을 동일하게 적용할지 여부
- 공개 범위 변경 시 과거 비교 리포트를 즉시 닫을지, 다음 열람부터 재평가할지 여부
