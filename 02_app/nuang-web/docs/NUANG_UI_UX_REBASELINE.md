# NUANG UI/UX Rebaseline

작성일: 2026-07-11 KST

이 문서는 NUANG MVP의 전반 UI/UX를 홈 중심으로 다시 정렬하기 위한 기준선이다. 목표는 단순히 화면을 예쁘게 바꾸는 것이 아니라, 사용자가 매일 들어와도 부담 없이 머물고 신뢰를 잃지 않게 만드는 것이다.

## 방향

- 홈은 메뉴판이 아니라 오늘의 개인화 대시보드다.
- 피드는 작성보다 읽기, 반응, 저장, 코드별 공감으로 체류를 만든다.
- 검사는 기능 목록이 아니라 나를 더 알아가는 콘텐츠 라이브러리다.
- 성향지도는 개인 그래프가 아니라 32개 코드 세계를 둘러보는 탐색 공간이다.
- 마이는 데이터 관리 화면이 아니라 내 코드, 리포트, 설정을 안정적으로 다시 여는 개인 공간이다.

## 첫 화면 원칙

첫 화면은 아래 순서로 구성한다.

1. 오늘의 상태를 짧게 말한다.
2. 바로 이어갈 행동을 하나 제안한다.
3. 개인화된 리포트나 코드로 연결한다.
4. 피드 체류 루프로 넘긴다.
5. 공유·비교·개인정보 신뢰 장치를 짧게 보여준다.

## UI 원칙

- 페이지 섹션은 얇은 선과 full-width band를 중심으로 구성한다.
- 반복 항목, 모달, 도구 표면에만 카드 형태를 쓴다.
- 둥근 장식, 과한 그림자, 큰 그라데이션은 최소화한다.
- 캐릭터 이미지는 감정적 초점으로 쓰되 화면마다 하나의 역할만 맡긴다.
- 버튼은 명령, 링크는 이동, 스위치는 설정이라는 역할을 섞지 않는다.
- 긴 설명보다 짧은 제목, 보조 문장, 명확한 다음 행동을 우선한다.
- 트렌디함은 색보다 리듬, 여백, 타이포, 반응 가능한 밀도에서 만든다.

## 홈 1차 개편 기준

홈은 다음 구조를 기준으로 한다.

- `오늘의 리듬`: 앱 전체의 현재 맥락을 한 문장으로 설명한다.
- `오늘의 추천`: 진행 중인 검사, 최근 결과, 첫 검사 중 하나만 제안한다.
- `오늘의 메뉴`: 내 성향, 내 리포트, 피드 반응으로 이어지는 핵심 행동 3개를 보여준다.
- `오늘의 질문`: 피드의 질문형 콘텐츠가 있을 때만 별도 band로 강조한다.
- `피드 미리보기`: 오늘 읽을 3개 항목을 세로 목록으로 보여준다.
- `신뢰 안내`: 직접 응답, 원점수, 민감 항목 비노출을 짧게 확인시킨다.

## 체류 루프

사용자의 하루 루프는 아래 흐름을 목표로 한다.

```text
홈에서 오늘의 추천 확인
→ 검사 또는 리포트 열기
→ 피드 질문/밸런스 게임 읽기
→ 반응 또는 저장
→ 마이 리포트에서 다시 확인
→ 다음 방문 때 이어갈 행동 추천
```

## 다음 개편 순서

1. 피드: 읽기 밀도, 반응 affordance, 저장한 글 회수 흐름 개선
2. 마이: 내 코드 요약, 내 리포트, 공개 범위, 데이터 권리의 정보 계층 정리
3. 검사: 코어, 무료 주제, 연구소를 콘텐츠 허브처럼 재배치
4. 성향지도: 32개 코드 탐색과 내 코드 연결의 리듬 개선
5. 공통 UI: 버튼, 섹션, 리스트, empty state, privacy note 패턴 통일
