## 0. 현재 권장 구조

### 고객에게 보여주는 코드 자리 순서

```text
E / I · R / N · G / A · K / M · C / Q
```

읽는 순서:

```text
사람과 에너지 → 생각과 정보 → 관계에서 먼저 보는 것 → 일상을 꾸리는 방식 → 마음의 반응
```

### 코어 성향 영역 수

`5개를 유지`한다.

단, 이 결론은 현재 문항과 각 영역의 세부 성향 구조가 이미 타당하다는 뜻이 아니다.

- 다섯 코드 자리는 뉴앙의 공개 코어에 적합한 가장 강한 현재 가설이다. Big Five 다섯 broad domain을 일대일로 복제한다고 주장하지 않는다.
- 각 domain의 세부 성향 구조는 별도로 정하고 검증한다. 모든 영역을 동일하게 두 facet으로 맞추지 않는다.
- RO 구성개념 검토에서 `D = Direct`를 폐기하고, 관계 상황에서 원인·해결과 상대 마음 중 무엇이 먼저 보이는지를 나타내는 `G/A`를 확정했다. RO-RN 선택·존엄 존중은 G/A에 합산하지 않는 본인 전용 상세 신호다.
- ER 구성개념 검토에서 기존 C/V를 `C/Q`로 교체하고 `C = Calm`, `Q = Quick-reacting`으로 확정했으며, 긍정 감정의 풍부함과 분리했다.
- SM 구성개념 검토에서 현재 문항이 실행·지속과 질서·구조를 측정하지만 낮은 방향에 MBTI P의 유연성을 부여한다는 모순을 확인했다. J/P를 폐기하고 `K = 꾸준히 이어감 / M = 상황에 따라 달라짐`을 사용자 승인했다.
- OE 구성개념 검토에서 높은 방향 N은 MBTI N과 같은 방향을 유지하되 동일한 검사로 해석하지 않기로 했다. 낮은 방향 R은 현실적 능력이 아니라 `익숙하고 구체적인 대상에 관심이 머무는 경향`으로 좁혀 사용자 승인했다.
- OE는 사용자 결정에 따라 `미적 경험·상상 확장·지적 탐구` 3-facet을 채택하고, 파일럿에서 세 facet의 구분성·신뢰도·상위 OE 구조를 검증한다.
- 여섯 번째 성향 후보는 코어 코드에 넣지 않고 별도 상세검사로 검증한다.

고객 자리 순서 `SE → OE → RO → SM → ER`는 유지한다. 현재 작업 범례는 `E/I · R/N · G/A · K/M · C/Q`다. `R/N`은 MBTI의 `S/N`과 같은 좌우 문법을 활용해 익숙하고 구체적인 대상에 관심이 가는 R을 먼저, 새로운 관점과 가능성에 관심이 가는 N을 뒤에 둔 표기다. N은 MBTI N과 같은 방향이지만 같은 검사 결과가 아니며, R은 현실성·실용 능력을 뜻하지 않는다.

## 1. 코드 순서 검토

### 1.1 비교한 두 순서

안 A — 순서 결정 당시 내부 측정 순서를 고객에게도 유지하는 경우:

```text
E/I · C/Q · J/P · G/A · R/N
사람 → 마음 → 일상 → 관계 → 생각
```

안 B — 순서 결정 당시 익숙한 성격 코드 문법을 활용한 고객 순서:

```text
E/I · R/N · G/A · J/P · C/Q
사람 → 생각 → 관계 → 일상 → 마음
```

권장: `안 B`

아래 순서 비교의 J/P는 당시 위치 학습 효과를 검토한 표기다. 후속 M01-SM 감사에서 현재 측정 의미와 맞지 않아 문자쌍은 폐기 후보가 됐지만, 네 번째 `SM 자리` 자체는 유지한다.

## 2. 안 B를 권장하는 이유

### 2.1 이미 학습된 자리 기억을 일부 재사용할 수 있다

MBTI 사용자는 다음 순서에 익숙하다.

```text
E/I → S/N → T/F → J/P
에너지 → 정보 → 결정 → 생활
```

뉴앙 안 B는 같은 개념은 아니지만 가장 가까운 주제를 같은 위치에 둔다.

| 위치 | MBTI의 익숙한 질문           | 뉴앙의 질문                                                                         | 효과                                                |
| ---- | ---------------------------- | ----------------------------------------------------------------------------------- | --------------------------------------------------- |
| 1    | 에너지가 어디로 향하는가     | 사람과 함께/혼자서 언제 충전되는가                                                  | E/I의 기존 이해를 활용                              |
| 2    | 어떤 정보를 먼저 보는가      | 새로운 관점·가능성과 익숙하고 구체적인 대상 중 어디에 관심이 가는가                 | N 방향의 기존 이해를 활용하되 동일 검사 오해를 차단 |
| 3    | 결정할 때 무엇을 먼저 두는가 | 마음과 해결이 함께 있는 관계 상황에서 원인·해결과 상대 마음 중 무엇이 먼저 보이는가 | 코드 위치만 활용, G/A는 T/F와 동일하지 않음         |
| 4    | 외부 생활을 어떻게 꾸리는가  | 실행·지속과 생활 구조가 어떻게 나타나는가                                           | 네 번째 위치는 유지하되 문자쌍 재설계               |
| 5    | 해당 없음                    | 불편한 상황에서 걱정과 감정이 얼마나 쉽게 커지는가                                  | 뉴앙 고유 영역임을 명확히 표시                      |

안 A에서는 익숙한 N 계열 주제가 다섯 번째, J/P가 세 번째로 이동하고 새로운 C/Q가 두 번째에 들어간다. 사용자는 새로운 글자뿐 아니라 기존에 알던 글자의 위치까지 다시 배워야 한다.

안 B가 더 잘 기억될 것이라는 판단은 사용자 친숙성을 이용한 UX 가설이다. 뉴앙 코드 순서 자체를 비교한 실증 연구는 없으므로 출시 전 A/B 이해도 검증을 거쳐야 한다.

### 2.2 자기소개 문장으로 자연스럽다

사람이 자신의 성향을 소개할 때 다음 흐름은 하나의 짧은 이야기로 연결된다.

1. 나는 사람 속에서 충전되는가, 혼자서 충전되는가
2. 새로운 관점·가능성과 익숙하고 구체적인 대상 중 어디에 관심이 가는가
3. 관계에서 원인·해결과 상대 마음 중 무엇이 먼저 보이는가
4. 계획을 정리해 마무리하는가, 상황을 열어두고 조정하는가
5. 불편한 상황에서 걱정과 감정이 빨리 커지는가, 비교적 차분하게 반응하는가

`사람 → 생각 → 관계 → 일상 → 마음`은 내부 심리측정 용어 없이도 실제 생활 이야기로 설명할 수 있다.

### 2.3 뉴앙의 차별점은 마지막 자리에서 분명하게 남는다

MBTI에 직접 대응하지 않는 ER 마음의 반응을 다섯 번째 자리에 배치하면 다음 효과가 있다.

- 처음 네 자리의 익숙한 읽기 흐름을 크게 깨지 않는다.
- 다섯 번째 글자에서 뉴앙이 추가로 보는 핵심을 자연스럽게 설명할 수 있다.
- C/Q가 다른 체계의 글자를 바꿔 쓴 것이 아니라 뉴앙 고유 영역임을 알 수 있다.

단, 고객 화면에서 `MBTI에 감정 축 하나를 더했다`고 말하지 않는다. 뉴앙은 독자 문항과 연속 점수, 세부 성향을 사용하는 별도 측정 모델이다.

### 2.4 측정 순서와 표시 순서를 분리할 수 있다

내부 domain ID와 데이터 저장 순서는 유지할 수 있다.

```text
내부 계산: SE → ER → SM → RO → OE
고객 코드: SE → OE → RO → SM → ER
```

코드를 배열 index에 직접 의존시키지 않고 `code_position_id`로 조립하면 된다.

| 고객 자리 | domain ID | 후보 글자 |
| --------- | --------- | --------- |
| 1         | SE        | E / I     |
| 2         | OE        | R / N     |
| 3         | RO        | G / A     |
| 4         | SM        | K / M     |
| 5         | ER        | C / Q     |

내부 측정 구조를 억지로 바꾸지 않고 고객의 학습 비용만 줄이는 방식이다.

### 2.5 motif도 코드 순서에서 분리해야 한다

현재 motif는 기존 코드의 첫 두 자리인 SE+ER 조합에 의존한다. 새 순서에서는 SE가 1번, ER이 5번이 된다.

motif는 문자열 앞 두 글자가 아니라 domain ID로 계산한다.

```text
motif = motifBy(SE letter, ER letter)
```

예:

- E + Q → 불꽃
- E + C → 햇살
- I + Q → 물결
- I + C → 숲

사용자는 motif 계산법을 외울 필요가 없다. 캐릭터와 코드 이름은 canonical profile ID를 통해 같은 의미로 유지한다.

## 3. 순서 변경 당시 예시 — 운영 사용 금지

아래 예시는 자리 순서를 검토할 때 J/P를 임시로 사용한 역사적 예시다. M01-SM 검토 후 새 코드·마이그레이션·공유 URL의 근거로 사용하지 않는다.

### 3.1 기존 코드에서 새 후보로

| 기존 v0.1 | v2 내부 순서 후보 | v3 고객 자리 예시 |
| --------- | ----------------- | ----------------- |
| `SVODE`   | `IVJDN`           | `INGJQ`           |
| `TVOAE`   | `EVJAN`           | `ENAJQ`           |
| `SCFAP`   | `ICPAR`           | `IRAPC`           |

G/A와 나머지 네 자리의 구성개념·채점이 모두 승인되기 전에는 위 예시를 운영 데이터 migration이나 공유 URL에 사용하지 않는다.

### 3.2 고객이 읽는 모습

```text
E  N  A  J  Q
함께 · 탐색 · 마음·선택 · 정돈 · 걱정·감정 빠름
```

한 문장:

> 사람들과 함께할 때 에너지가 살아나고, 가능성과 의미를 탐색하며, 관계에서는 상대의 마음과 선택권을 살피는 편이에요. 일상을 정돈해 가고, 불편한 상황에서는 걱정이나 감정이 비교적 빨리 커질 수 있어요.

반대 방향 예시:

```text
I  R  G  P  C
혼자 · 현실 · 해결·결과 · 유연 · 차분
```

### 3.3 코드 표면 설계 원칙

- 다섯 글자는 동일한 크기와 간격으로 보여준다.
- `4글자 + 보너스 1글자`처럼 구분선을 넣지 않는다.
- 각 글자 아래에는 초기 학습 기간 동안 쉬운 한국어를 붙인다.
- 한 글자를 누르면 반대 방향, 세부 성향, 선명도를 펼친다.
- 같은 글자를 썼다는 이유로 MBTI 결과와 같다고 설명하지 않는다.

## 4. 순서 결정용 사용자 검증

안 A와 안 B를 같은 디자인으로 비교한다.

### 검증 과제

1. 10초 동안 코드를 본 뒤 다섯 자리의 주제를 순서대로 말할 수 있는가
2. 본인의 코드를 10분 뒤 다시 입력할 수 있는가
3. 친구에게 코드와 의미를 한 문장으로 설명할 수 있는가
4. 새 SM 문자를 보고 실행·지속과 질서·구조의 방향을 정확히 설명하는가
5. G/A와 C/Q를 MBTI 글자로 억지 변환하지 않는가
6. 뉴앙을 `MBTI+1`이라고 오해하지 않는가

### 결정 원칙

- 안 B가 회상과 설명 정확도에서 우세해야 한다.
- MBTI 동일 검사 오해가 늘어나면 FAQ·초기 설명을 수정한다.
- 설명 보강 후에도 동일 검사 오해가 높으면 공유 글자 또는 순서를 다시 검토한다.
- 개발 편의 때문에 안 A를 선택하지 않는다.

## 5. 다섯 성향 영역이 최적인가

### 5.1 결론의 범위

`다섯 영역은 뉴앙의 공개 코어 코드에 가장 적합한 현재 선택`이다.

다만 다음 두 문장은 다르다.

- 허용: `다섯 영역은 현재 이론, 제품 목적, 기억 부담을 함께 고려한 최선의 코어 가설이다.`
- 금지: `사람의 성격은 과학적으로 정확히 다섯 가지뿐이다.`

성격 구조에는 Big Five 외에도 HEXACO 같은 대안이 있다. 무엇이 최적인지는 사용 목적과 측정 증거에 따라 달라진다.

## 6. 4개·5개·6개 영역 비교

| 구조 | 장점                                                           | 손실·위험                                                                | 뉴앙 적합성                      |
| ---- | -------------------------------------------------------------- | ------------------------------------------------------------------------ | -------------------------------- |
| 4개  | 코드가 짧고 MBTI 학습을 활용하기 쉬움, 16조합                  | broad domain 하나를 버려야 함; MBTI식 4축은 정서 반응 영역이 없음        | 낮음                             |
| 5개  | broad personality를 폭넓게 다루면서 5글자·32조합으로 유지 가능 | 한 글자 더 학습해야 함; 각 축의 한국어 검증 필요                         | 가장 높음                        |
| 6개  | HEXACO의 Honesty-Humility 같은 별도 영역을 포함 가능, 64조합   | 코드·지도·커뮤니티 세분화가 커짐; 도덕적 자기평가와 사회적 바람직성 위험 | 코어에는 낮음, 상세검사에는 가능 |

5개를 유지하는 이유:

- 4개로 줄이면 ER 정서 반응 또는 다른 broad domain 하나를 제외해야 한다.
- ER은 관계 속 오해, 스트레스 반응, 자기이해에서 중요한 차이를 제공한다.
- 6개는 64개 조합이 되어 코드 이름·캐릭터·커뮤니티 통계가 지나치게 분산된다.
- Honesty-Humility처럼 도덕 평가로 읽히기 쉬운 축은 공개 프로필 코드보다 동의 기반 상세검사에 더 적합하다.
- 5개는 측정 범위와 기억·공유 부담 사이의 가장 좋은 현재 균형이다.

## 7. 5개 domain별 유지 판정

| 영역                  | 현재 두 세부 성향                                 | 유지 판정                                        | 반드시 해결할 문제                                                                 |
| --------------------- | ------------------------------------------------- | ------------------------------------------------ | ---------------------------------------------------------------------------------- |
| SE 사람 사이 에너지   | 교류 활력, 주도적 표현                            | 유지                                             | 두 facet이 하나의 E/I 방향을 구성하는지 검증                                       |

### 7.1 SE

Big Five Extraversion과 MBTI E/I 모두에 가장 안정적으로 대조되는 영역이다. 뉴앙은 교류 활력과 주도적 표현을 분리해 더 구체적으로 설명할 수 있다.

판정: `유지`

### 7.2 ER

MBTI에 직접 없는 Negative Emotionality 계열을 포함해 뉴앙의 차별점이 된다. 공개 코어는 감정 동요와 걱정·주저의 2-facet으로 유지하고, Depression 관련 내용은 별도 웰빙·전문 영역으로 분리한다. C/Q는 `차분하게 반응 / 걱정·감정이 빨리 커짐`으로 설명하며 `예민함/둔함`, 긍정 감정의 풍부함 또는 건강·질병처럼 읽히지 않게 한다. Q는 행동이나 판단 속도가 빠르다는 뜻으로 사용하지 않는다.

판정: `domain 유지, C/Q 사용자 승인`

### 7.3 SM

Conscientiousness 계열은 실행과 계획, 약속, 마무리 등 일상에서 관찰하기 쉽고 사용자 행동과 연결하기 좋다. 후속 M01-SM 감사에서 실행·지속과 질서·구조는 상위 영역 가설을 유지할 근거가 있으나, 낮은 점수가 MBTI P의 유연성·선택지 개방을 뜻하지 않는다는 점을 확인했다. BFI-2의 Responsibility 범위를 반영한 제3 facet 후보도 파일럿에서 비교한다.

판정: `domain 유지, K/M 사용자 승인, 사용자 이해 검증 필요`

### 7.4 RO

Agreeableness 계열은 관계 중심 서비스에 중요한 연구 배경이다. 그러나 공감적 관심과 선택·존엄 존중을 한 글자에 합치면 낮은 선택 존중이 `목표 지향`으로 미화되거나 G 사용자가 낙인찍힐 수 있다. G/A는 RO-EC 관계 주의 방향으로 좁히고, RO-RN은 사용자 본인의 상세 리포트에서 별도로 검증한다.

판정: `코드 자리 유지, G/A RO-EC 전용 승인, RO-RN 분리 검증 필요`

### 7.5 OE

Openness/Intellect 계열은 관심사, 콘텐츠 탐색, 대화 주제와 연결되므로 커뮤니티 경험에 유용하다. 현재 문항은 미적 경험·상상과 지적 탐구에 대한 관심을 묻지만, 낮은 방향 리포트가 현실 감각·실용 능력·안정적인 판단까지 추론하는 오류가 있다. 이를 제거하고, 사용자 결정에 따라 미적 경험·상상 확장·지적 탐구의 3-facet을 채택한다. 파일럿은 두 구조의 재선택이 아니라 세 facet이 실제로 구분되면서도 상위 OE를 구성하는지 검증한다.

N은 MBTI N과 같은 높은 방향을 유지하지만 같은 구성개념이나 검사 결과로 변환하지 않는다. R은 `Real-world / 익숙하고 구체적인 것에 관심`이라는 기억용 후보이며, 현실성·실용 능력·상식·정확성을 뜻하지 않는다.

판정: `domain·3-facet·R/N 사용자 승인, 구조와 이해도 검증 필요`

## 8. 모든 영역을 두 facet씩 구성하는 것이 최적인가

아직 확정할 수 없다.

근거가 되는 모델:

- Big Five Aspect Scales는 각 broad domain 아래 두 개의 관련 aspect, 총 10개를 제안하고 별도 표본에서 검증했다.
- BFI-2는 각 domain 아래 세 facet, 총 15개를 사용한다.

따라서 `5 domain`에는 강한 이론적 근거가 있지만 `각 domain당 정확히 2 facet`만이 유일한 정답은 아니다.

특정 domain에서 두 facet 구조를 선택하려면 다음이 확인되어야 한다.

- 두 facet이 broad domain의 핵심 범위를 충분히 대표한다.
- 두 facet이 서로 구분된다.
- 두 facet이 같은 상위 domain을 구성한다.
- 다른 domain facet과 교차 적재가 과도하지 않다.
- 한국어 2030 사용자에게 같은 의미로 해석된다.
- 두 facet으로 만든 domain이 실제 생활·관계 결과를 적절히 설명한다.

OE는 사용자 결정에 따라 세 facet으로 확장했다. 다른 domain도 근거가 있을 때만 세 facet으로 늘린다. 5글자 코드는 domain 다섯 개를 유지하면서 내부 해상도만 다르게 구성할 수 있다.

## 10. 확정 로드맵

1. 고객 자리 순서를 `SE → OE → RO → SM → ER`로 유지하고, 현재 작업 범례를 `E/I · R/N · G/A · K/M · C/Q`로 관리
2. M01-SE·RO·ER·SM 구성개념과 문자 승인 상태 유지
3. M01-OE의 3-facet 승인 상태를 유지하고 R/N 의미 사용자 승인
4. 다섯 domain 코드의 의미·글자·자리 정합성 최종 감사
5. 독립 전문가 blind mapping
6. 2030 인지 인터뷰
7. 코드 순서 A/B 이해·회상 테스트
8. 정량 파일럿에서 5자리 구조·각 자리의 요인구조·교차오염 검증
9. 알파벳·순서·채점 경계 최종 release
10. versioned DB migration과 앱 UI 반영
