## 0. 이번 단계의 결론

RO-EC는 `공감 능력`, `친절함`, `실제로 많이 도와주는 정도`를 재는 항목이 아니다.

```text
다른 사람의 어려움이나 부담을 접한 상황
→ 사실·과제·해결에만 주의가 머무는지,
→ 그 사람의 감정 상태와 안녕에도 자연스럽게 주의가 가고
→ 이해하거나 지지하고 싶은 마음이 생기는지를 관찰
```

첫 후보 은행은 6개 상황 버킷, 총 12문항으로 구성했다.

| 구분                   |  수 |
| ---------------------- | --: |
| 높은 RO-EC 방향 `HIGH` |   6 |
| 낮은 RO-EC 방향 `LOW`  |   6 |
| 주력 인지 인터뷰 후보  |   8 |
| 경계 확인용 탐색 후보  |   4 |

`HIGH`는 A의 `상대 마음이 먼저 보이는 편`, `LOW`는 G의 `원인·해결이 먼저 보이는 편`에 해당하는 초기 증거 후보다. 어느 방향도 더 착하거나 유능하다는 뜻이 아니다.

12문항은 모두 연구 후보다. 아직 빠른 2문항, 정밀 약 6문항, G/A 코드 산출 문항으로 확정하지 않는다.

## 1. 측정 목표와 금지 해석

### 측정하려는 것

- 힘든 이야기를 들을 때 상대가 느낀 마음에 주의가 가는 정도
- 표정·말투·행동에서 부담이 보일 때 상대의 안녕이 신경 쓰이는 정도
- 해결할 일과 감정적 지지가 함께 필요한 장면에서 감정 상태도 살피는 정도
- 가까운 관계의 의무가 약해도 다른 사람의 상태를 더 이해하고 싶어지는 정도
- 명시적인 요청이 없어도 지지 의향이 생기는지에 대한 탐색
- 대화가 끝난 뒤에도 상대의 안녕이 다시 떠오르는지에 대한 탐색

### 측정하지 않는 것

- 상대 감정을 정확히 맞히는 능력
- 감정에 함께 휩쓸리는 정서 전염
- 위로·상담·대화 기술
- 실제로 제공한 돈·시간·도움의 양
- 가족·연인·친구에 대한 의무감
- 자기 필요를 포기하는 자기희생
- 문제 해결 능력, 논리성, 판단력
- 상대의 거절과 경계를 존중하는 RO-RN
- 먼저 말을 걸거나 표현하는 SE-AI
- 걱정이 오래 지속되는 ER-WD 또는 감정 반응 강도 ER-IR

### 코드와의 관계

```text
RO-EC 여러 문항의 유효 응답
→ RO-EC 연속 점수와 측정오차
→ 관계 주의 방향 점수와 경계 상태 검증
→ 세 번째 G/A 글자 후보
```

후속 A안 승인으로 G/A는 RO-EC만 사용한다. RO-RN은 별도 본인 전용 상세 신호이며 이 v0.1 문항으로는 어떤 결과도 발급하지 않는다.

## 2. 공통 응답 방식

모든 문항은 아래 형식으로 답한다.

```text
최근 6개월 동안 비슷한 상황이 생겼을 때의 평소 모습

1 거의 그렇지 않아요
2 드문 편이에요
3 반반이에요
4 자주 그래요
5 거의 항상 그래요

이 상황은 답하기 어려워요
```

`LOW` 문항에 높은 값을 선택하는 것은 감정 신호보다 사실·현재 할 일·해결 방법에 주의가 먼저 가는 경험의 증거 후보다. 냉정함, 이기심, 공감 능력 부족 또는 문제 해결 능력을 뜻하지 않는다.

## 6. HIGH·LOW의 비우열 계약

| 방향   | 허용되는 의미                                                | 금지되는 단정                                     |
| ------ | ------------------------------------------------------------ | ------------------------------------------------- |
| `HIGH` | 다른 사람의 감정 상태와 안녕에도 주의가 자연스럽게 감        | 착함, 공감 능력·상담 능력이 높음, 실제로 잘 도움  |
| `LOW`  | 감정 신호보다 사실·과제·해결 방법에 주의가 먼저 머물 수 있음 | 차가움, 이기적임, 공감 능력이 없음, 논리적·유능함 |

`HIGH=좋은 사람`, `LOW=문제 해결형`처럼 양쪽에 측정하지 않은 가치·능력을 붙이지 않는다. 리포트에서는 관찰된 주의 방향만 설명한다.

## 7. 후보 쌍 중복과 동시 사용 제한

| 후보 쌍     | 중복 위험                                         | 처리                                                    |
| ----------- | ------------------------------------------------- | ------------------------------------------------------- |
| `C01 / C02` | 힘든 이야기에서 감정과 사건 정보에 가는 주의      | 인지 인터뷰 form을 분리하고 둘 다 높은 응답의 이유 확인 |
| `C03 / C04` | 지쳐 보이는 사람과 현재 과제 사이의 주의          | 일의 긴급도와 관계 친밀도를 통제할 문구 비교            |
| `C05 / C06` | 해결 장면에서 감정과 해결 방법의 우선순위         | 짧은 검사에 동시에 넣지 않고 더 순수한 문장 경쟁        |
| `C07 / C08` | 덜 가까운 사람의 이야기에서 마음과 사건 중심 듣기 | 비교 문법·예의 규범 오염이 낮은 쪽 우선                 |
| `C09 / C10` | 요청 없는 부담 신호에 대한 접근과 대기            | RO-RN·SE-AI 오염이 크면 둘 다 공개 코어에서 제외        |
| `C11 / C12` | 대화 이후 후속 관심과 주의 전환                   | ER 걱정·기억 오염이 크면 둘 다 제외                     |

같은 장면의 정반대 문장을 짧은 검사에 함께 넣어 응답 일관성을 인위적으로 높이지 않는다.

| -------------------------- | ------------------ | --------------------------------------------------- |
| 상황 라벨 40자 이하        | `PASS`             | 모두 권장 범위 안                                   |
| 질문 문장 70자 이하        | `PASS`             | 모두 한 문장                                        |
| 질문의 빈도·`~하는 편이다` | `PASS`             | 응답 척도와 의미 중복 없음                          |

| 인접 성향 오염             | `REVIEW_REQUIRED`  | C09·C10 RO-RN/SE-AI, C11·C12 ER                     |
| HIGH/LOW 문구 효과         | `REVIEW_REQUIRED`  | HIGH=마음, LOW=사실·과제로 응답 전략이 보일 수 있음 |

자동 lint 통과는 내용타당도나 타당성 검증 완료가 아니다.

## 9. 예상 응답 해석

예시 `C01`:

```text
누군가 힘들었던 일을 이야기할 때
그 사람이 어떤 마음이었는지에 주의가 간다.
```

| 응답        | 허용되는 가설                                             | 금지되는 해석                        |
| ----------- | --------------------------------------------------------- | ------------------------------------ |
| 1           | 힘든 이야기를 들을 때 감정 상태에 주의가 가는 경험이 드묾 | 공감 능력이 없음, 냉정함             |
| 3           | 사람·상황·관계에 따라 감정에 가는 주의가 비슷하게 갈림    | 성향이 없음, 거짓 응답               |
| 5           | 상대가 느낀 마음에 주의가 가는 경험이 반복됨              | 상대 감정을 정확히 맞힘, 상담을 잘함 |
| 판단 어려움 | 이런 대화를 거의 하지 않음·표현 이해 문제·응답 근거 부족  | 3점 또는 LOW 증거                    |

예시 `C04`:

```text
함께 있던 사람이 평소보다 지쳐 보일 때
하던 일에 주의를 계속 둔다.
```

| 응답        | 허용되는 가설                                                          | 금지되는 해석           |
| ----------- | ---------------------------------------------------------------------- | ----------------------- |
| 1           | 부담 신호가 보이면 하던 일보다 상대 상태로 주의가 옮겨가는 경우가 많음 | A 확정, 착한 사람       |
| 3           | 일의 중요도·관계·상황에 따라 주의 방향이 비슷하게 갈림                 | 애매한 사람             |
| 5           | 부담 신호가 보여도 현재 과제에 주의를 유지하는 경험이 반복됨           | 이기적임, 집중력이 좋음 |
| 판단 어려움 | 떠올린 일의 긴급도 차이가 너무 크거나 장면 경험이 부족함               | 3점 대체 또는 LOW 확정  |

## 10. 인지 인터뷰 설계

### 10.1 공통 질문

1. 어떤 관계·장소·어려움의 강도를 떠올렸는가?
2. 문장을 자신의 말로 다시 설명하면 무엇인가?
3. `마음`, `상태`, `괜찮은지`, `신경이 간다`를 각각 어떻게 이해했는가?
4. 선택한 1·3·5 응답을 뒷받침하는 최근 실제 경험은 무엇인가?
5. 관심이 생긴 것과 실제로 도운 행동을 구분해 답했는가?
6. LOW 문항을 문제 해결 능력·집중력·경계 존중의 장점으로 답했는가?
7. HIGH 문항을 착하고 따뜻하게 보이기 위해 높여 답했는가?
8. `반반이에요`와 `이 상황은 답하기 어려워요`를 구분할 수 있는가?

### 10.2 문구 A/B 확인

| 개념                | A                                    | B                                         | 확인 목표                            |
| ------------------- | ------------------------------------ | ----------------------------------------- | ------------------------------------ |
| emotional attention | `어떤 마음이었는지에 주의가 간다`    | `어떤 마음이었는지 자연스럽게 궁금해진다` | 주의와 단순 호기심·OE 탐색 구분      |
| wellbeing attention | `괜찮은지 신경이 간다`               | `힘든 상태인지 마음이 간다`               | 쉬운 한국어·걱정·도덕성 해석 차이    |
| fact-first          | `무슨 일이 있었는지부터 파악한다`    | `일어난 일을 중심으로 듣는다`             | 시간 순서와 중요도 해석 차이         |
| solution-first      | `어떻게 해결할지에 주의가 먼저 간다` | `해결할 일을 정리하는 데 집중한다`        | 주의 방향과 문제 해결 능력 오염 차이 |
| support inclination | `상태를 물어보고 싶어진다`           | `말을 걸어보고 싶어진다`                  | 관심과 SE-AI·참견 규범의 구분        |

한 참여자에게 같은 장면의 양방향 문장을 연속해서 보여주지 않는다.

### 10.3 두 개의 균형 form

| Form A | 방향   | Form B | 방향   |
| ------ | ------ | ------ | ------ |
| C01    | `HIGH` | C02    | `LOW`  |
| C04    | `LOW`  | C03    | `HIGH` |
| C05    | `HIGH` | C06    | `LOW`  |
| C08    | `LOW`  | C07    | `HIGH` |
| C09    | `HIGH` | C10    | `LOW`  |
| C12    | `LOW`  | C11    | `HIGH` |

각 form은 HIGH 3개·LOW 3개다. form 점수를 비교하지 않고 문장 이해·회상·오염 탐색에만 사용한다.

## 11. 전문가 blind mapping 사양

검토자에게 목표 facet을 표시하지 않고 다음을 받는다.

- 가장 가까운 상위 영역·세부 신호와 두 번째 후보
- 표적이 감정 주의, 안녕 관심, 해결 우선, 선택 존중, 걱정, 표현 시작 중 무엇인지
- HIGH/LOW 채점 방향
- 상황의 보편성·강도·관계 의존성
- 사회적 바람직성·능력·문화 오염
- 수정 없이 사용 / 수정 후 사용 / 제외

우선 혼동:

- C01·C02: RO-EC ↔ OE 관점 탐색·경청 방식
- C03·C04: RO-EC ↔ 관찰력·ER 걱정·현재 과제 집중
- C05·C06: RO-EC ↔ SM 실행·문제 해결 능력
- C07·C08: RO-EC ↔ 관계 친밀도·예의 규범
- C09·C10: RO-EC ↔ RO-RN·SE-AI
- C11·C12: RO-EC ↔ ER-WD·기억·반추

통과선은 결과를 보기 전에 통합 M04에서 심리측정 책임자가 사전 등록한다.

## 12. 빠른·정밀 검사 후보 구성 원칙

### 빠른 검사

RO-EC 최소 2문항은 다음을 만족해야 한다.

- 같은 상황 버킷을 반복하지 않음
- HIGH·LOW 방향을 모두 검토하되 단순 부정 역문항은 사용하지 않음
- `힘든 이야기`와 `해결·부담 신호` 중 서로 다른 장면을 포함
- 가족·연인·친구 중 한 관계만 전제하지 않음
- C09·C10은 RO-RN·SE-AI와 분리되기 전 점수에 넣지 않음
- C11·C12는 ER 걱정·반추와 분리되기 전 점수에 넣지 않음

아래는 인지 인터뷰 전의 비교 조합이며 최종 shortlist가 아니다.

| 조합 | 후보      | 장점                                         | 위험                                |
| ---- | --------- | -------------------------------------------- | ----------------------------------- |
| A    | C01 + C06 | 감정 주의 + 해결 우선, 서로 다른 장면과 방향 | HIGH/LOW의 가치 차이가 보일 수 있음 |
| B    | C03 + C08 | 보이는 부담 + 덜 가까운 관계                 | 관찰력·관계 거리 영향               |
| C    | C07 + C04 | 이해 의향 + 현재 과제 주의                   | 친밀도·일의 중요도 영향             |

### 정밀 검사

약 6문항을 선정한다면 최소 네 버킷을 포함하고 같은 유사 쌍을 모두 넣지 않는 안부터 비교한다. 실제 선정은 아래 근거 뒤에만 가능하다.

- RO-RN과 분리되면서 RO 상위 영역을 함께 구성하는가?
- OE 관점 탐색·SE-AI·ER 걱정·SM 실행과 교차 적재가 수용 가능한가?
- HIGH와 LOW가 내용보다 문구 방식으로 갈리지 않는가?
- 낮은 방향 응답이 문제 해결 능력이나 경계 존중만 반영하지 않는가?
- 높은 방향 응답에 천장 효과와 사회적 바람직성 효과가 크지 않은가?
- 관계 친밀도·성별 규범·돌봄 역할에 따라 문항 기능이 달라지지 않는가?

## 14. 사용자 검토가 필요한 결정

다음 네 항목을 검토한 뒤 RO-RN 후보 문항 은행으로 이동한다.

1. RO-EC를 `착함`이 아니라 `상대의 감정 상태와 안녕에 자연스럽게 가는 주의·지지 의향`으로 유지한다.
2. 6개 버킷·12문항을 후보 풀로 보존하되 빠른·정밀 문항은 아직 고르지 않는다.
3. C09·C10은 선택·경계 존중과, C11·C12는 걱정·반추와 구분하기 위한 탐색 후보로만 둔다.
4. C08의 비교 문법과 HIGH 문항의 사회적 바람직성은 승인과 별개로 인지 인터뷰 A/B에서 반드시 경쟁 문구를 검증한다.

승인 전에는 운영 코드·DB·검사 화면을 변경하지 않는다.
