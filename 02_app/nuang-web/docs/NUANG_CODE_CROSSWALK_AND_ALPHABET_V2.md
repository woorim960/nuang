## 0. 결론

차기 뉴앙 코드의 권장 후보는 현재 자리 순서를 유지한 아래 조합이다.

```text
E / I · V / C · J / P · A / D · N / R
```

| 자리 | 권장 후보 | 고객에게 보이는 뜻 | 결정 상태 |
|---|---|---|---|
| 1 사람 사이 에너지 | `E / I` | 함께하며 충전 / 혼자서 충전 | SE 구성개념 승인 대기 |
| 2 마음의 반응 | `V / C` | 선명한 반응 / 차분한 반응 | ER 구성개념 검증 대기 |
| 3 일상을 꾸리는 방식 | `J / P` | 정돈해 마무리 / 열어두고 조정 | SM 구성개념 검증 대기 |
| 4 관계 방식 | `A / D` | 상대를 살피며 조율 / 내 기준을 직접 표현 | RO 구성개념 검증 대기 |
| 5 생각의 방향 | `N / R` | 가능성과 의미 탐색 / 현실과 적용 중시 | OE 구성개념 검증 대기 |

이 조합의 장점:

- 열 개 글자가 서로 중복되지 않는다.
- MBTI에서 이미 익숙한 글자는 비슷한 의미일 때만 사용한다.
- MBTI `F`와 `P`를 다른 뜻으로 쓰던 v1의 충돌을 제거한다.
- MBTI에 없는 정서 반응은 `V/C`, 관계 방식은 `A/D`라는 뉴앙 고유 언어로 분리한다.
- 현재 5자리 순서를 유지하므로 데이터 migration 범위를 불필요하게 키우지 않는다.

단, 이 알파벳은 아직 코드 release가 아니다. 각 축의 구성개념이 M01에서 확정되어야 글자도 최종 확정할 수 있다.

## 1. MBTI와 뉴앙의 모델 차이

MBTI는 네 개의 선호쌍을 조합한 유형 모델이다.

- E/I: 에너지와 주의가 향하는 방향
- S/N: 정보를 받아들이는 방식
- T/F: 결정을 내릴 때 먼저 두는 기준
- J/P: 외부 생활을 조직하는 방식

뉴앙은 다섯 성향 영역을 연속 점수로 측정하고, 5글자 코드는 그 결과를 기억하기 쉽게 압축하는 표현층으로 설계한다.

따라서 두 코드 사이에는 `개념적으로 가까운 부분`이 있을 수 있지만, 아래는 허용하지 않는다.

- 뉴앙 코드에서 MBTI 코드를 계산하기
- MBTI 코드에서 뉴앙 코드를 계산하기
- 같은 알파벳이면 같은 검사 결과라고 설명하기
- MBTI 유형 뒤에 글자 하나만 붙인 검사라고 설명하기

성격 5요인과 MBTI의 비교 연구 및 MBTI 측의 요약 자료에서는 다음 관계가 반복 보고됐다.

- MBTI E/I ↔ Big Five Extraversion
- MBTI S/N ↔ Big Five Openness
- MBTI T/F ↔ Big Five Agreeableness
- MBTI J/P ↔ Big Five Conscientiousness
- Big Five Neuroticism에 직접 대응하는 MBTI 축은 없음

뉴앙의 현재 5영역도 이 다섯 성격 영역을 바탕으로 한 가설이므로 개념적 대조는 가능하다. 그러나 상관은 동일성을 뜻하지 않는다.

## 2. 자리별 대조

### 2.1 뉴앙 E/I ↔ MBTI E/I

대응 수준: `가장 높음`

공통점:

- 사람·외부 상호작용에서 에너지가 살아나는지
- 혼자 생각하고 정리할 때 에너지가 회복되는지
- 생각을 말하며 정리하는지, 정리한 뒤 말하는지
- 먼저 반응하거나 시작하는 경향

뉴앙의 추가 구분:

- `함께할 때의 에너지`
- `먼저 표현하기`

MBTI의 공식 E/I 설명에도 상호작용에서 에너지를 얻는 경향과 주도적 시작이 함께 포함된다. 따라서 첫 자리는 `E/I`를 공유할 수 있는 가장 강한 후보다.

차이:

- MBTI는 선호 유형 언어다.
- 뉴앙은 여러 문항에서 두 세부 성향과 연속 점수·불확실성을 계산한다.
- 뉴앙 E가 곧 MBTI E라는 주장은 할 수 없다.

### 2.2 뉴앙 V/C ↔ MBTI 직접 대응 없음

대응 수준: `뉴앙 고유 축`

현재 뉴앙이 측정하려는 내용:

- 감정 반응이 빠르고 크게 올라오는 정도
- 걱정과 위축이 나타나는 정도
- 불편함·위험 신호를 얼마나 빠르게 감지하는지
- 흔들린 뒤 차분하게 돌아오는 방식

이 내용은 성격 5요인의 Negative Emotionality/Neuroticism 영역과 가깝다. 공식 MBTI 설명에도 이 Big Five 영역에 직접 대응하는 선호쌍이 없다고 명시되어 있다.

권장 글자:

- `V = Vivid`: 감정 신호가 선명
- `C = Calm`: 감정의 온도를 차분히 정리

주의:

- V를 `감정이 풍부한 좋은 사람`으로 표현하지 않는다.
- C를 `감정이 둔한 사람`으로 표현하지 않는다.
- 반응 강도와 걱정이 한 축을 이루는지는 M01-ER에서 검증한다.

### 2.3 뉴앙 J/P ↔ MBTI J/P

대응 수준: `비교적 높지만 검증 필요`

공통점:

- 계획과 구조를 선호하는지
- 일을 결정하고 마무리하는지
- 선택지를 열어두고 상황에 맞춰 조정하는지
- 변화와 즉흥성에 편안함을 느끼는지

현재 뉴앙 세부 성향:

- 실행과 마무리
- 정리와 계획

MBTI J/P는 조직·계획·종결 대 유연성·선택지 개방을 설명하므로, 기존 `O/F`보다 `J/P`가 사용자에게 더 즉시 이해될 가능성이 높다. 또한 F를 MBTI의 Feeling과 다른 Flexible이라는 뜻으로 쓰던 v1 충돌도 해소된다.

차이:

- 뉴앙은 실행 지속과 실제 정리 습관까지 측정하려 한다.
- MBTI J/P는 외부 생활에 접근하는 선호이며 과업 성과나 성실성 자체가 아니다.
- SM의 두 세부 성향이 한 축을 이루는지 확인하기 전 J/P를 확정하지 않는다.

권장 글자:

- `J`: 정돈해 결정하고 마무리
- `P`: 열어두고 상황에 맞춰 조정

고객 화면에서는 `판단형/인식형` 같은 번역어를 사용하지 않고 생활 문장으로 설명한다.

### 2.4 뉴앙 A/D ↔ MBTI F/T와 일부만 겹침

대응 수준: `부분 대응, 독자 글자 유지`

현재 뉴앙이 측정하려는 내용:

- 상대의 감정과 부담을 살피는 정도
- 상대의 거절과 선택권을 존중하는 정도
- 사실·선택·내 기준을 먼저 보는 정도
- 원하는 방향을 분명하게 말하는 정도

MBTI F는 결정을 내릴 때 사람과 가치에 두는 비중을 설명하므로 뉴앙 A의 공감 측면과 일부 겹친다. 그러나 MBTI T는 객관적 논리와 비인격적 기준을 중심으로 결정하는 선호다. 뉴앙의 D 후보인 직접 표현·거리 조절·내 기준과 같은 개념은 MBTI T와 동일하지 않다.

따라서 `F/T`를 사용하면 검사를 실제보다 더 같아 보이게 한다. 이 자리에는 뉴앙 고유 글자인 `A/D`를 유지하는 편이 안전하다.

권장 기억 단서:

- `A = Attuned`: 상대 반응을 살피며 조율
- `D = Direct`: 내 기준과 선택을 직접 표현

중요한 구성개념 문제:

- 현재 RO-EC의 높은 방향은 공감·배려다.
- 현재 RO-RN의 높은 방향은 거절과 선택권 존중이다.
- 둘 모두 높을 때 A로 계산되지만, 기존 D 설명에는 `거리와 선택 존중`이 들어가 있어 RO-RN의 높은 방향과 충돌한다.

따라서 A/D의 최종 뜻은 M01-RO에서 두 세부 성향과 낮은 방향을 다시 정의한 뒤 확정한다. 기억하기 쉬운 대칭을 만들기 위해 낮은 점수를 근거 없이 `독립적이고 솔직한 장점`으로 바꾸지 않는다.

### 2.5 뉴앙 N/R ↔ MBTI N/S

대응 수준: `비교적 높지만 완전 일치 아님`

공통점:

- 가능성, 패턴, 의미, 상상을 탐색하는지
- 구체적인 사실과 실제 적용을 우선하는지
- 아이디어와 원리를 넓혀 보는지
- 현실에서 바로 쓸 수 있는 설명을 선호하는지

현재 뉴앙 세부 성향:

- 감각과 상상
- 아이디어 탐구

N은 MBTI에서도 가능성·패턴·상상·아이디어와 연결되므로 같은 방향에 사용할 수 있다.

반대 방향에 MBTI의 S를 그대로 쓰지 않는 이유:

- MBTI Sensing은 오감을 통한 구체적 정보 수집이라는 고유한 이론적 의미가 있다.
- 뉴앙의 낮은 OE 점수가 곧 감각 기능 선호를 뜻하지 않는다.
- 현재 뉴앙이 말하려는 것은 복잡한 가능성보다 현실성과 적용 가능성을 우선하는 경향에 더 가깝다.

권장 글자:

- `N = New possibilities`: 가능성과 의미 탐색
- `R = Real-world`: 현실과 적용 중시

`R`이 실제 사용자에게 자연스럽게 기억되는지는 별도 이해도 테스트가 필요하다.

## 3. 이전 안의 판정

### v0.1 — S/T · C/V · F/O · D/A · P/E

판정: `교체 필요`

- 첫 S/T가 외향·내향 방향을 즉시 알려주지 못한다.
- 글자의 영어 기억 단서가 고객 문구와 약하게 연결된다.

### v1 — E/I · V/C · O/F · A/D · N/P

판정: `사용하지 않음`

- 뉴앙 F는 Flexible이지만 MBTI F는 Feeling이다.
- 뉴앙 P는 Practical이지만 MBTI P는 Perceiving이다.
- 익숙한 글자를 다른 뜻으로 사용해 오히려 오해를 늘린다.

### v2 — E/I · V/C · J/P · A/D · N/R

판정: `권장 후보`

- MBTI 글자를 같은 방향의 개념에만 사용한다.
- 완전히 같지 않은 관계 축은 A/D로 분리한다.
- MBTI에 없는 정서 반응은 V/C로 분리한다.
- OE의 현실 방향은 Sensing과 동일시하지 않고 R로 구분한다.

## 5. 고객에게 보여주는 방식

코드를 처음 보는 화면에서는 글자만 던지지 않는다.

```text
E  V  J  A  N
함께 · 선명 · 정돈 · 살핌 · 탐색
```

반대 방향 예시:

```text
I  C  P  D  R
혼자 · 차분 · 유연 · 직접 · 현실
```

최초 결과의 한 줄 설명:

```text
EVJAN
사람들과 함께할 때 에너지가 살아나고,
마음의 신호가 선명하며 일상을 정돈해 가는 편이에요.
```

글자를 누르면 아래 순서로 펼친다.

1. 쉬운 한국어 한 단어
2. 생활 속 한 문장
3. 반대 방향과의 차이
4. 두 세부 성향
5. 현재 방향의 선명도와 불확실성

## 6. MBTI 비교 안내 문구

고객 FAQ 초안:

> 뉴앙 코드와 MBTI는 일부 성향 주제가 겹치지만 같은 검사는 아니에요. E/I처럼 비슷한 방향도 있지만, 뉴앙은 마음이 흔들릴 때의 반응을 별도로 보고 각 성향을 연속적인 정도와 세부 모습으로 설명해요. 같은 글자라도 한 검사 결과를 다른 검사 결과로 바꿔 읽을 수는 없어요.

금지 문구:

- `뉴앙은 MBTI에 감정 축을 더한 검사예요.`
- `EVJAN은 ENFJ와 같아요.`
- `뉴앙 E면 MBTI도 E예요.`
- `MBTI보다 정확해요.`
- `MBTI를 대체해요.`

## 7. 확정 순서

코드 글자를 먼저 고정하고 문항을 그 뜻에 맞추지 않는다.

1. M01-SE 승인 → E/I 후보 확정
2. M01-ER → V/C 의미 검증
3. M01-SM → J/P 공유 가능성 검증
4. M01-RO → A/D의 실제 양극과 facet 구조 재설계
5. M01-OE → N/R 의미 검증
6. 5개 축 blind naming test
7. 코드 조합 회상·설명 테스트
8. 측정 검증 후 code scheme release
9. 버전 migration과 UI 반영

현재는 E/I를 포함한 모든 글자를 `candidate`로 저장하고 운영 코드에는 반영하지 않는다.

## 8. 이해도 검증 과제

알파벳 후보를 10초 동안 보여준 뒤 다음을 확인한다.

- E와 I 방향을 올바르게 설명하는가
- J/P를 MBTI와 같은 방향으로 이해하되 같은 검사라고 오해하지 않는가
- A/D를 T/F로 잘못 바꾸어 말하지 않는가
- N/R에서 R을 현실·적용 방향으로 기억하는가
- V/C가 감정이 좋고 나쁨이 아니라 반응 방식이라는 점을 이해하는가
- 5글자 전체를 10분 뒤 회상하는가
- 코드 아래의 한국어를 보지 않고도 각 글자의 짧은 의미를 말할 수 있는가

비교할 후보:

- v0.1 기존안
- v2 권장안
- 구성개념 검토 과정에서 나오는 대체 글자안

## 9. 구현 전 필수 데이터 계약

- `code_scheme_version`
- `canonical_profile_id`
- `stored_code_at_completion`
- `display_code_current`
- `domain_scores`
- 검증된 경우에만 `facet_scores`
- `code_uncertainty_by_position`
- `code_alias_from_version`

문자열 코드가 사용자의 성향 원본이 되어서는 안 된다. 원본은 release에 연결된 연속 점수와 측정 근거이며, 코드는 버전별 표시값이다.
