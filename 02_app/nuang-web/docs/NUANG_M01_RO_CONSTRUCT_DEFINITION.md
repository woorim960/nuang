## 0. 이번 단계의 결정 요약

세 번째 대표 코드는 마음과 해결이 함께 있는 일상적 관계 상황에서 `무엇이 먼저 눈에 들어오거나 궁금해지는가`를 나타낸다.

```text
G = 원인·해결이 먼저 보이는 편
A = 상대 마음이 먼저 보이는 편
```

최종 결정:

- 고객 영역명은 `관계에서 먼저 보는 것`이다.
- 내부 대표 구성개념은 `RO 관계 주의 방향`이다.
- G/A의 측정 원본은 `RO-EC 관계 주의 방향` 하나다.
- `G = Goal-oriented`는 원인·결과·해결할 부분에 주의가 먼저 가는 방향이다.
- `A = Attentive`는 상대의 감정 상태와 안녕에 주의가 먼저 가는 방향이다.
- G/A는 문제 해결 능력, 공감 능력, 논리성, 착함, 의사결정 방식 전체를 뜻하지 않는다.
- `RO-RN 선택·존엄 존중`은 중요한 관계 행동 신호지만 G/A 점수·글자·선명도에 합산하지 않는다.
- RO-RN은 검증 후에도 개인 정밀 리포트의 서술형 상세 신호가 기본이며 공개·공유·비교에서는 제외한다.
- 검증된 RO-EC process pair에서만 `G → A` 같은 관계 반응 흐름을 후보로 만든다.
- 대표 G/A와 과정 흐름의 G/A는 모두 같은 `원인·해결 ↔ 상대 마음` 범위를 사용한다.
- 기존 D/A와 RO-EC+RO-RN 상위 점수 제안은 폐기한다.

이 결정으로 `사실·해결을 먼저 보면서도 상대의 선택은 존중하는 사람`을 G이자 낮은 존중으로 오해하지 않는다.

## 1. 왜 기존 D/A 구조를 그대로 사용할 수 없는가

현재 데이터 구조에서는 아래 두 내용이 모두 RO의 높은 방향으로 채점된다.

- 다른 사람의 감정과 부담을 살피는 경향
- 상대의 거절·선택권을 인정하고 강요하지 않는 경향

그런데 기존 코드 사전의 D는 다음처럼 설명된다.

- 상대가 스스로 결정할 여지를 존중한다.
- 가까운 관계에서도 각자의 영역이 필요하다.
- 강요하지 않는다.

첫 번째와 세 번째 설명은 현재 `RO-RN`의 높은 방향과 같다. 즉 같은 행동을 세부 점수에서는 높은 RO로, 코드 설명에서는 낮은 RO인 D로 해석하는 모순이 있다.

또한 `내 기준을 직접 표현한다`는 내용은 RO의 반대편이 아니라 `SE-AI 주도적 표현`에 더 가깝다. 직접 말하면서도 상대의 거절을 존중할 수 있고, 말을 부드럽게 하면서도 반복 설득으로 압박할 수 있다. 표현의 선명도와 상대 권리 존중은 별개의 관찰 대상이다.

따라서 글자를 먼저 지키기 위해 구성개념을 바꾸지 않는다. 측정 의미를 먼저 확정한 뒤 그 의미를 정확히 기억시키는 글자를 선택한다.

## 2. 이론적 위치와 뉴앙의 해석

### 2.1 두 세부 성향 구조의 근거

Big Five Aspect Scales 연구는 Agreeableness 아래에 서로 관련되지만 구분되는 `Compassion`과 `Politeness`가 존재할 수 있다는 경험적 선례를 제시한다.

- Compassion은 다른 사람의 감정과 안녕에 대한 비교적 자동적인 관심·돌봄과 관련된다.
- Politeness는 다른 사람의 필요와 권리를 고려하고 공격적·강압적 반응을 억제하는 경향과 관련된다.

이 구조는 RO-EC와 RO-RN의 출발점으로 사용할 수 있다. 다만 공개된 영어 척도의 구조가 뉴앙의 한국어 문항과 2030 사용자 집단에서 그대로 재현된다는 뜻은 아니다.

### 2.2 Big Five의 세부 성향을 대표 코드에 그대로 묶지 않는다

BFI-2는 Agreeableness를 `Compassion`, `Respectfulness`, `Trust`의 세 facet으로 구분한다. 뉴앙의 현재 RO 구조에는 일반적인 대인 신뢰가 별도로 없다.

BFAS와 BFI-2의 위계 구조는 `공감적 관심`과 `선택·존엄 존중`을 서로 구분해 연구할 근거를 제공한다. 그러나 이론적으로 관련된다는 사실만으로 두 신호를 고객용 G/A 한 글자에 합칠 수는 없다.

제품 구조는 아래처럼 확정한다.

- 대표 G/A: RO-EC의 관계 주의 방향
- 본인 전용 상세 신호: RO-RN 선택·존엄 존중
- 별도 상세 관계검사: 일반적 대인 신뢰, 관계별 신뢰, 신뢰 회복 방식

뉴앙의 세 번째 자리는 Big Five Agreeableness 전체를 그대로 재현한다고 주장하지 않는다. 관련 연구는 문항 범위와 인접 성향을 검토하는 근거이며, 실제 G/A 구조는 한국어 문항에서 별도로 검증한다.

### 2.3 MBTI T/F와 같은 축이 아니다

MBTI의 T/F는 결론을 내릴 때 논리·비인격적 사실과 사람·가치 중 무엇에 더 무게를 두는지를 설명한다. G/A는 의사결정 선호 전체가 아니라 마음과 해결이 함께 있는 관계 상황에서 처음 주의가 가는 대상을 측정한다.

두 체계 사이에 경험적 관련이 있을 수는 있지만 다음과 같이 동일시하지 않는다.

- A = MBTI F 아님
- G = MBTI T 아님
- 원인·해결이 먼저 보이는 것 = 낮은 존중 아님
- 상대 마음이 먼저 보이는 것 = 모든 결정을 사람 중심으로 내리는 것 아님

## 3. 명칭 체계 제안

| ID    | 내부 명칭      | 고객에게 보이는 쉬운 명칭 | 대표 코드 포함 | 판정                |
| ----- | -------------- | ------------------------- | -------------- | ------------------- |
| RO    | 관계 주의 방향 | 관계에서 먼저 보는 것     | G/A 자체       | 확정                |
| RO-EC | 관계 주의 방향 | 원인·해결 / 상대 마음     | 포함           | 확정                |
| RO-RN | 선택·존엄 존중 | 선택과 경계 존중하기      | 제외           | 본인 전용 상세 신호 |

`관계 조율 방식`은 갈등 해결 능력과 선택 존중까지 대표 글자에 포함하는 것처럼 넓다. `관계에서 먼저 보는 것`은 현재 G/A가 실제로 측정할 관심 순서를 직접 설명한다.

`공감적 배려`는 높은 사람이 더 도덕적이라는 인상을 줄 수 있다. `공감적 관심`은 상대의 상태에 주의와 관심이 자연스럽게 향하는 경향이라는 측정 대상을 더 직접적으로 표현한다.

`존중·비강압`은 한쪽은 가치, 다른 한쪽은 행동 억제로 문법이 섞여 있다. `선택·존엄 존중`은 거절과 선택권, 비하 억제라는 공통 대상을 드러낸다.

## 4. G/A 대표 구성개념 정의

### 4.1 한 문장 정의

`RO 관계 주의 방향`은 마음과 해결이 함께 있는 일상적 관계 상황에서 상대의 감정 상태와 안녕, 또는 사건의 원인·결과·해결할 부분 중 어디에 주의가 먼저 향하는지를 나타내는 연속 성향이다.

### 4.2 측정하는 것

- 다른 사람이 힘든 일을 말할 때 마음과 사건 중 무엇이 먼저 궁금해지는가
- 감정과 해결이 함께 필요한 상황에서 무엇이 먼저 눈에 들어오는가
- 상대 마음을 확인하는 반응과 해결 내용을 다루는 반응이 어떤 순서로 나타나는가

### 4.3 측정하지 않는 것

- 공감 정확도나 마음 읽기 능력
- 상담·위로·갈등 해결 기술
- 착함, 도덕성, 좋은 사람 여부
- 친절한 말투나 예절 지식만
- 타인을 실제로 도울 수 있는 시간·돈·권한
- 자기희생, 순응, 부탁을 모두 들어주는 성향
- 직접 표현, 솔직함, 리더십, 자기주장
- 건강한 경계 설정 능력
- 애착 유형, 친밀감 욕구, 특정 관계의 만족도
- 일반적 대인 신뢰 또는 특정 인물에 대한 신뢰
- 거절과 선택권을 존중하는 정도
- 비하·강요·압박을 억제하는 정도

### 4.4 점수 해석

G/A 연속 점수는 승인된 RO-EC 문항만으로 계산한다. RO-RN, 대인 신뢰, 갈등 회피, 주도적 표현 점수는 G/A의 값이나 선명도를 바꾸지 않는다.

- G 방향: 원인·결과·해결할 부분이 먼저 보이는 응답이 상대적으로 많음
- A 방향: 상대의 감정 상태와 안녕이 먼저 보이는 응답이 상대적으로 많음
- 경계: 두 방향이 비슷하거나 측정오차 범위가 양쪽에 걸침
- 증거 부족: 응답 수·품질이 발급 기준을 충족하지 못함

글자는 연속 점수를 기억하기 위한 표현층이다. 단순 원점수 50점을 경계로 사용하지 않고 M07 파일럿에서 규준·정보량·측정오차를 함께 검증한다.

## 5. RO-EC 공감적 관심

### 5.1 정의

다른 사람의 감정 상태와 안녕에 자연스럽게 주의를 두고, 이해·위로·지지하려는 마음이 생기는 경향이다.

### 5.2 상대적으로 높은 방향에서 관찰할 수 있는 모습

- 힘들어 보이는 사람의 상태가 자연스럽게 눈에 들어온다.
- 상대의 경험을 이해하려고 듣는 편이다.
- 가능한 범위에서 정서적 지지나 도움 의향이 생긴다.
- 문제 해결 전후에 상대가 어떻게 느끼는지도 살핀다.

### 5.3 상대적으로 낮은 방향에서 관찰할 수 있는 모습

- 감정 신호보다 사실, 과제, 해결할 문제에 주의가 먼저 갈 수 있다.
- 다른 사람의 정서 상태가 행동의 우선순위에 미치는 영향이 상대적으로 작을 수 있다.
- 위로나 지지 표현이 자연스럽게 떠오르지 않을 수 있다.

낮은 방향만으로 아래 내용을 추론하지 않는다.

- 냉정하다 또는 이기적이다.
- 공감 능력이 없다.
- 문제 해결 능력이 좋다.
- 논리적이거나 독립적이다.
- 타인의 경계를 잘 존중한다.

### 5.4 포함 경계

포함:

- 정서적 관심, 동정적 반응, 안녕에 대한 관심
- 상대 경험을 이해하려는 자발적 주의
- 지지하고 싶어지는 경향

제외:

- 상대 감정을 정확히 맞히는 능력
- 감정에 같이 휩쓸리는 정서 전염
- 실제 도움의 양
- 가까운 사람에게만 느끼는 의무감
- 자기 필요를 포기하는 자기희생

### 5.5 처음 드는 생각과 실제 나타나는 반응을 분리한다

RO-EC의 핵심은 상대의 감정 상태와 안녕에 자연스럽게 가는 관심이다. 그러나 사용자가 보고한 처음 드는 생각과 실제 나타나는 반응은 같지 않을 수 있다.

```text
원인·해결이 먼저 보임
→ 상대에게 미칠 영향을 고려함
→ 실제로는 마음을 먼저 확인함
```

이 경우 처음 드는 생각은 RO-EC의 낮은 방향 증거 후보, 실제 나타나는 반응은 별도 `ENACTED_RESPONSE` profile modifier 후보다. 실제 나타나는 반응 하나만 보고 RO-EC를 높게 덮어쓰거나 RO-RN을 높게 확정하지 않는다.

반대로 상대의 마음이 먼저 보이지만 실제로는 필요한 해결을 먼저 말할 수도 있다. 이것도 모순이나 거짓 응답이 아니라 검증해야 할 다층 반응 패턴이다.

행동 이유는 `배려해서`, `상대가 상할까 봐`처럼 문항에 포함하지 않는다. 반복된 층위 차이가 확인된 뒤 사용자가 별도 비채점 probe에서 직접 고른 이유만 상세 리포트의 맥락으로 사용한다.

이 다층 프로필은 새로운 코드 자리가 아니다. 대표 G/A와 과정 흐름은 모두 RO-EC의 `원인·해결 ↔ 상대 마음` 범위를 사용하되, 대표 글자는 처음 드는 생각의 안정적인 경향을 요약하고 과정 흐름은 처음 드는 생각과 실제 나타나는 반응의 차이를 설명한다.

## 6. RO-RN 선택·존엄 존중

### 6.1 정의

의견이나 원하는 결과가 다르더라도 상대의 선택권과 존엄을 인정하고, 비하·강요·과도한 압박으로 자신의 뜻을 관철하려는 행동을 억제하는 경향이다.

### 6.2 상대적으로 높은 방향에서 관찰할 수 있는 모습

- 거절을 받으면 상대의 결정을 인정하려 한다.
- 공동 결정에서 상대가 말하고 선택할 여지를 남긴다.
- 갈등 중에도 상대 자체를 깎아내리는 표현을 억제한다.
- 원하는 결과가 있어도 압박의 강도를 조절한다.

### 6.3 상대적으로 낮은 방향에서 관찰할 수 있는 모습

- 원하는 결과가 중요할 때 설득이나 압박을 오래 이어갈 가능성이 상대적으로 높을 수 있다.
- 의견 충돌에서 상대의 선택권보다 자신의 목표를 우선할 수 있다.
- 갈등의 강도가 높아지면 비하적·적대적 표현을 억제하기 어려울 수 있다.

낮은 방향만으로 아래 내용을 추론하지 않는다.

- 자기주장이 강하다.
- 솔직하고 직접적으로 말한다.
- 독립적이다.
- 경계를 잘 세운다.
- 리더십이나 추진력이 좋다.

이 특성들은 별도 구성개념과 문항으로 측정해야 한다.

### 6.4 포함 경계

포함:

- 선택권과 거절 존중
- 강요·압박·비하 억제
- 공동 결정에서 상대 발언권 인정

제외:

- 갈등 자체를 피하는 행동
- 부탁을 거절하지 못하는 순응
- 자기 의견을 작게 말하는 태도
- 모든 의견에 동의하는 태도
- 예의 바른 어휘만 사용하는 말투
- 안전·폭력·법적 의무 상황에서 필요한 개입

### 6.5 설득 충동과 실제 존중 행동을 분리한다

RO-RN의 핵심은 상대의 선택권과 존엄을 실제 반응에서 인정하고 압박·비하를 억제하는 경향이다. 그러나 더 설득하고 싶은 생각과 실제 나타나는 반응은 다를 수 있다.

```text
결정을 바꿀 설득 방법이 떠오름
→ 상대가 스스로 정할 권리를 고려함
→ 실제로는 그 선택을 그대로 둠
```

이 경우 처음 드는 생각은 `REPORTED_FIRST_ORIENTATION` profile modifier, 실제 나타나는 반응은 RO-RN 핵심 증거 후보다. 반대로 상대 선택을 인정해야 한다고 생각하면서도 결과가 중요할 때 설득을 계속할 수 있다.

실제로 설득을 멈춘 행동만 보고 선택 존중의 이유를 확정하지 않는다. 갈등 걱정, 포기, 역할 종료, 관심 부족도 같은 행동을 만들 수 있다. 반복된 층위 차이 뒤 사용자가 별도 비채점 probe에서 직접 선택한 이유만 리포트 맥락으로 사용한다.

RO-EC의 `상대 마음을 먼저 확인하는 행동`은 감정에 대한 반응 순서이고, RO-RN의 `거절 뒤 선택을 그대로 두는 행동`은 선택권과 압박 조절이다. 앞의 행동만으로 뒤의 점수를 대신하지 않는다.

## 7. 대표 코드와 본인 전용 상세 신호를 섞지 않는다

RO-EC와 RO-RN을 모두 측정하더라도 하나의 상위 점수로 평균 내지 않는다.

| 대표 G/A | RO-RN 상세 신호         | 허용되는 설명                                                                | 금지되는 단정               |
| -------- | ----------------------- | ---------------------------------------------------------------------------- | --------------------------- |
| A 방향   | 선택 여지를 자주 남김   | 상대 마음이 먼저 보이고, 의견이 달라도 선택 여지를 남기는 반응이 자주 나타남 | 좋은 사람, 관계 능력이 높음 |
| A 방향   | 설득을 이어갈 때가 있음 | 상대 마음은 먼저 보이지만 중요한 결과에서는 설득이 길어질 수 있음            | 위선적, 오지랖형            |
| G 방향   | 선택 여지를 자주 남김   | 원인·해결이 먼저 보이면서도 상대의 결정은 인정하는 반응이 자주 나타남        | 차갑지만 독립적, 낮은 공감  |
| G 방향   | 압박이 길어질 때가 있음 | 원인·해결이 먼저 보이고 일부 갈등에서는 설득이 길어질 수 있음                | 무례한 유형, 위험 인물      |

RO-RN은 본인에게 숨기는 값이 아니다. 검증을 통과하면 정밀 리포트에서 쉬운 문장과 상황별 근거로 보여주되, 대표 글자·선명도·공개 프로필·피드 공유·상대 비교에는 기본 포함하지 않는다.

### 7.1 G/A 글자 부여 규칙

G/A는 측정 원본이 아니라 연속 점수를 기억하기 위한 표현층이다. 유효한 검사에서는 세 번째 자리를 비워두지 않는다.

```text
승인된 RO-EC 응답이 유효함
→ 관계 주의 방향 연속 점수 계산
→ 중심점보다 G 방향이면 G, A 방향이면 A
→ 신뢰구간·응답 품질로 선명도와 세부 설명 추가
```

| 상태                      | 코드 글자                     | 사용자 설명                                | 후속 행동                               |
| ------------------------- | ----------------------------- | ------------------------------------------ | --------------------------------------- |
| G 방향이 선명함           | G                             | 원인과 해결할 부분이 먼저 보이는 편이에요. | 관계 반응 흐름은 정밀검사에서 별도 검증 |
| A 방향이 선명함           | A                             | 상대가 어떤 마음인지 먼저 보이는 편이에요. | 관계 반응 흐름은 정밀검사에서 별도 검증 |
| 중심점 부근/신뢰구간 교차 | 점추정치가 가까운 쪽 G 또는 A | G와 A가 비슷해요.                          | 필요할 때 2~4개 적응형 추가 문항 권장   |
| 응답 부족·품질 실패       | 코드 미발급                   | 결과를 만들기 위한 응답이 더 필요해요.     | 추가 응답 또는 재검사                   |

내부 상태는 `CLEAR`, `NEAR_BOUNDARY`, `INSUFFICIENT_EVIDENCE`를 사용한다. `SPLIT_PROFILE`은 G/A와 RO-RN의 차이를 뜻하지 않는다. 처음 드는 생각과 실제 나타나는 반응이 유효하게 갈리는 경우에는 별도 `PROCESS_FLOW` 상태로 보존한다.

중심점과 선명도 수치는 M07 파일럿에서 규준과 측정오차를 기준으로 확정한다. 단순 원점수 50점이나 정확히 절반 응답을 그대로 경계로 사용하지 않는다.

## 8. 인접 구성개념과의 경계

| 인접 성향            | 혼동되는 이유                                   | RO와 구분하는 질문                                                        |
| -------------------- | ----------------------------------------------- | ------------------------------------------------------------------------- |
| SE-AI 주도적 표현    | 강하게 말하는 모습이 압박처럼 보일 수 있음      | 먼저·분명히 말했는가가 아니라 상대의 거절과 선택 여지를 남겼는가          |
| ER-WD 걱정·위축      | 갈등을 피하는 사람이 비강압적으로 보일 수 있음  | 두려워서 말을 못 했는가, 말할 수 있어도 상대 권리를 인정했는가            |
| ER-IR 감정 반응 강도 | 화가 빠르게 오르면 비하·압박이 늘 수 있음       | 감정이 얼마나 빨리 올랐는가와 그 감정을 타인에게 어떻게 행동했는가를 분리 |
| SM-EP 실행·지속      | 실제 도움 행동은 성실성과 자원 영향을 받음      | 돕고 싶었는가와 실제로 끝까지 도왔는가를 분리                             |
| OE 관점 탐색         | 다른 관점을 생각하는 것이 공감처럼 보일 수 있음 | 여러 설명을 떠올렸는가와 상대 안녕에 관심이 갔는가를 분리                 |
| 대인 신뢰            | 상대를 믿으면 부드럽게 대할 가능성이 있음       | 타인이 선의라고 믿는가와 선택권을 존중하는가를 분리                       |
| 갈등 대처 기술       | 잘 훈련된 대화는 존중처럼 보일 수 있음          | 문장 기술이 아니라 반복되는 자연스러운 경향을 측정                        |

핵심 사례:

- 의견을 먼저 분명히 말하면서 상대의 거절을 인정하면 `SE-AI 높음 + RO-RN 높음`일 수 있다.
- 아무 말도 하지 않았지만 속으로 상대를 존중한 것이 아니라 갈등이 두려웠다면 `ER-WD`의 영향일 수 있다.
- 상대를 걱정해 계속 설득했다면 `RO-EC 높음 + RO-RN 낮음`일 수 있다.

## 9. 문항이 다뤄야 할 상황 범위

### 9.1 RO-EC 상황 표본

- 다른 사람이 힘든 이야기를 꺼낼 때
- 표정이나 행동에서 부담이 보일 때
- 가까운 사람과 덜 가까운 사람의 어려움을 접할 때
- 해결할 일과 감정적 지지가 함께 필요한 때

피해야 할 편향:

- 가족·연인처럼 의무가 강한 관계만 사용
- 심각한 사고·질병처럼 누구나 바람직하게 답하기 쉬운 극단 상황
- 돈·시간·직업 권한 때문에 도움 가능성이 크게 달라지는 상황
- `착한 사람이라면`이라는 도덕 평가가 드러나는 문장

### 9.2 RO-RN 상황 표본

- 사소한 선호가 다를 때
- 공동 계획을 정할 때
- 부탁이나 제안을 거절당했을 때
- 중요한 의견 충돌이 생겼을 때
- 상대의 실수나 문제를 지적할 때

피해야 할 편향:

- 폭력·자해·범죄·안전 위험 때문에 개입이 필요한 상황
- 교사·상사·보호자처럼 역할 권한이 핵심인 상황만 사용
- 한 번 더 설명하는 정상적 설득과 강압을 구분하지 못하는 상황
- 명백한 모욕 문구만 제시해 정답이 보이는 문항

## 10. 리포트 주장 계약

### 10.1 검증 후 말할 수 있는 내용

RO-EC 높은 방향:

- 상대의 감정과 부담에 주의가 자연스럽게 가는 편
- 이해하거나 지지하려는 마음이 비교적 쉽게 생기는 편

RO-EC 낮은 방향:

- 감정 신호보다 사실이나 해결할 문제에 주의가 먼저 갈 수 있음
- 위로나 지지가 자연스럽게 떠오르기까지 시간이 필요할 수 있음

RO-RN 높은 방향:

- 거절과 선택권을 인정하려는 경향
- 의견 차이에서도 압박과 비하를 억제하려는 경향

RO-RN 낮은 방향:

- 원하는 결과가 중요할 때 설득·압박이 길어질 수 있음
- 갈등이 강해지면 상대 선택 여지를 좁힐 가능성이 상대적으로 높을 수 있음

### 10.2 말하면 안 되는 내용

- 공감 능력이 높다/낮다.
- 좋은 사람, 배려 깊은 사람, 이기적인 사람이다.
- 관계를 잘한다/못한다.
- 솔직하다, 논리적이다, 독립적이다.
- 자기 경계를 잘 지킨다.
- 상대를 믿는다/의심한다.
- 갈등 해결 능력이나 상담 능력이 좋다.
- 가족·친구·연인과 잘 맞는다.
- 애착 유형이나 정신건강 상태가 어떻다.

### 10.3 기존 리포트 문구 판정

| 기존 문구                               | 판정           | 이유                                             |
| --------------------------------------- | -------------- | ------------------------------------------------ |
| `상대 마음과 내 기준을 함께 살피는 편`  | 수정 필요      | 내 기준은 현재 RO에서 측정하지 않음              |
| `관계의 안전감을 만드는 데 강점`        | 삭제           | 관계 결과·능력으로 과도하게 확장                 |
| `배려하다 내 필요를 뒤로 미룸`          | 별도 증거 필요 | 자기희생·거절 곤란을 측정하지 않음               |
| `솔직한 선택과 거리 조절을 중시`        | 삭제           | 낮은 RO에서 솔직함·거리 조절을 추론할 수 없음    |
| `과한 책임을 떠안지 않음`               | 삭제           | 경계 설정과 책임 범위를 측정하지 않음            |
| `상대 감정보다 사실과 선택을 먼저 봄`   | facet별 분리   | 사실 우선은 낮은 EC 후보지만 선택 존중은 높은 RN |
| `내 기준이나 원하는 방향을 분명히 말함` | 삭제           | SE-AI와 겹치며 낮은 RN의 뜻이 아님               |

현재 `core-result-copy.v0.1.json`과 `report-copy.ts`의 RO 문구는 운영 승계 대상이 아니다. 후속 리포트 재작성 단계에서 이 계약에 맞춰 교체한다.

## 11. 현재 RO 12문항 감사

판정 표기:

- `REWRITE`: 구성개념 후보는 관련 있으나 문구를 그대로 사용하지 않음
- `DUPLICATE`: 다른 문항과 내용 범위가 지나치게 겹침
- `CONTEXT_RISK`: 상황에 따라 의미가 크게 달라짐
- `DESIRABILITY_RISK`: 좋은 사람처럼 답하려는 영향이 큼
- `CROSS_LOAD_RISK`: 다른 성향이나 능력과 함께 측정될 위험

### 11.1 RO-EC 6문항

| 문항    | 현재 핵심                                 | 감사 판정                     | 핵심 문제                                                             |
| ------- | ----------------------------------------- | ----------------------------- | --------------------------------------------------------------------- |
| ROEC-01 | 가까운 사람의 힘든 이야기를 이해하며 듣기 | `REWRITE · CROSS_LOAD_RISK`   | 가까운 관계에 한정되고, `이해`가 공감 정확도·경청 기술로 읽힐 수 있음 |
| ROEC-02 | 어려운 사람에게 도움이 필요한지 살피기    | `REWRITE · DESIRABILITY_RISK` | 도움 행동·경계 감시·실제 자원과 섞일 수 있음                          |
| ROEC-03 | 속상한 사람을 그냥 지나치지 않기          | `REWRITE · DESIRABILITY_RISK` | `그냥 지나치기보다`가 바람직한 정답을 강하게 암시                     |
| ROEC-05 | 다른 사람의 부담을 덜어주고 싶음          | `REWRITE · DESIRABILITY_RISK` | 자기희생과 도움 의무, 사회적 바람직성 영향이 큼                       |
| ROEC-07 | 위로나 지지를 표현하기                    | `REWRITE · CROSS_LOAD_RISK`   | 마음의 관심보다 표현 기술·문화·SE-AI를 함께 측정할 수 있음            |
| ROEC-10 | 감정보다 해결할 일에 먼저 집중            | `REWRITE · CONTEXT_RISK`      | 공감과 해결을 강제 양자택일로 만들며 둘 다 할 수 있는 사람을 오분류   |

RO-EC의 현재 문항은 높은 방향 5개, 반대 채점 1개다. 높은 방향 문항이 도움·위로·관심처럼 명백히 바람직한 행동에 몰려 있어 응답 편향을 통제하기 어렵다.

### 11.2 RO-RN 6문항

| 문항    | 현재 핵심                               | 감사 판정                                 | 핵심 문제                                                      |
| ------- | --------------------------------------- | ----------------------------------------- | -------------------------------------------------------------- |

### 11.3 문항 세트 전체 판정

- 12문항 모두 `OWNER_REVIEW_REQUIRED` 상태를 유지한다.
- 기존 JSON의 `qa_checks: PASS`를 측정 승인으로 해석하지 않는다.
- 어떤 문항도 현재 문구 그대로 운영 코어에 확정하지 않는다.
- M02에서 상황 표본과 행동 초점을 먼저 정한 뒤 M03에서 새 후보군을 작성한다.
- 같은 뜻의 정·역문항을 단순 부정문으로 뒤집지 않는다.
- 노골적으로 좋은/나쁜 행동 대신 선택이 실제로 갈릴 수 있는 중간 강도의 상황을 사용한다.

## 12. 빠른 코어 기존 RO 4문항 판정

현재 빠른 코어는 다음 네 문항을 사용한다.

| 문항    | 세부 성향      | 방향      | 판정        |
| ------- | -------------- | --------- | ----------- |
| ROEC-01 | 공감적 관심    | 높은 방향 | 재작성 필요 |
| RORN-02 | 선택·존엄 존중 | 높은 방향 | 재작성 필요 |
| ROEC-05 | 공감적 관심    | 높은 방향 | 재작성 필요 |
| RORN-03 | 선택·존엄 존중 | 높은 방향 | 재작성 필요 |

네 문항 모두 과거 2-facet 높은 방향의 사회적으로 바람직한 진술이다. 이 구성은 다음 위험이 있다.

- 문장에 동의하는 습관과 실제 성향을 구분하기 어렵다.
- 좋은 사람처럼 보이려는 응답을 통제하기 어렵다.
- G/A와 무관한 RO-RN 문항이 대표 글자에 섞인다.
- RO-EC 두 문항도 마음 쪽 정답만 있어 G 방향을 안정적으로 구분하기 어렵다.

현재 네 문항은 모두 대표 G/A 산출에서 제외한다. 새 빠른 코어는 RO-EC의 G/A 양쪽 방향을 서로 다른 중간 강도 상황에서 최소 4문항으로 비교한다. RO-RN 문항은 대표 코드가 아닌 별도 상세검사 후보로 이동한다.

빠른 코어는 RO-EC 응답의 정보량과 품질이 승인 기준을 만족할 때만 G/A 한 글자를 포함한 `예비 5글자 코드`를 제공한다. 중심점 부근이면서 측정 신뢰도가 낮거나 응답 품질이 부족할 때만 결과 직전에 2~4개의 짧은 적응형 문항을 추가할 수 있다. 빠른 결과에서는 관계 반응 흐름이나 RO-RN 상세 신호를 확정하지 않는다.

## 13. 코드 문자 최종 설계

### 13.1 D 폐기

`D = Direct`는 직접 표현을 뜻하며 RO-RN의 낮은 방향과 동일하지 않다. SE-AI와 구성개념이 겹치고, 낮은 RO에서 측정하지 않은 솔직함·자기주장을 장점으로 만들어낸다.

판정: `REJECT_AS_RO_LOW_SYMBOL`

### 13.2 비교한 문자 후보

| 후보                          | 장점                                              | 탈락 또는 선택 이유                                    | 판정 |
| ----------------------------- | ------------------------------------------------- | ------------------------------------------------------ | ---- |
| D/A — Direct/Attuned          | 기존 초안과 연결됨                                | Direct는 SE-AI이며 관계 주의 방향과 다름               | 탈락 |
| T/F — Thinking/Feeling        | MBTI 학습을 그대로 활용                           | 의사결정 선호 전체로 오해되고 MBTI 자동 변환 위험      | 탈락 |
| H/G — Harmony/Goal            | 한국어 설명이 쉬움                                | Harmony는 갈등 회피·순응까지 포함                      | 탈락 |
| O/G — Others/Goal             | 양쪽 대상이 직관적                                | Others-first는 자기희생·타인 우선으로 과장             | 탈락 |
| G/A — Goal-oriented/Attentive | 원인·해결과 상대 마음의 주의 방향을 기억하기 쉬움 | RO-EC 범위로 제한할 때 양쪽을 가치중립적으로 설명 가능 | 선택 |

### 13.3 G 의미

`G = Goal-oriented`

고객 표기: `원인·해결이 먼저 보이는 편`

정의:

> 마음과 해결이 함께 있는 관계 상황에서 상대의 감정 상태보다 일어난 일의 원인·결과·해결할 부분에 주의가 먼저 가는 경향

G가 뜻하지 않는 것:

- 목표 달성 능력이 높음
- 논리적이거나 똑똑함
- 독립적이거나 솔직함
- 경계를 잘 설정함
- 상대를 존중하지 않음
- 원하는 결과를 반드시 관철함

### 13.4 A 의미

`A = Attentive`

고객 표기: `상대 마음이 먼저 보이는 편`

정의:

> 마음과 해결이 함께 있는 관계 상황에서 일어난 일의 원인·해결보다 상대의 감정 상태와 안녕에 주의가 먼저 가는 경향

A가 뜻하지 않는 것:

- 공감 정확도나 상담 능력이 높음
- 모든 부탁을 들어줌
- 자기 욕구를 희생함
- 갈등을 피함
- 더 착하거나 도덕적임
- 상대의 거절과 선택권을 반드시 존중함

### 13.5 범례와 코드 위치

```text
고객 코드 자리: 사람 → 생각 → 관계 → 일상 → 마음
domain 위치:     SE → OE → RO → SM → ER
현재 작업 범례: E/I · R/N · G/A · K/M · C/Q
```

세 번째 자리는 MBTI의 T/F와 코드 위치만 비슷하다. G/A는 결정 논리 전체가 아니라 마음과 해결이 함께 있는 관계 상황에서 무엇이 먼저 보이는지를 설명하므로 T/F와 같은 결과라고 말하지 않는다.

초기 학습 UI에서는 글자만 보여주지 않는다.

```text
G  원인·해결이 먼저 보이는 편
A  상대 마음이 먼저 보이는 편
```

## 14. 후속 데이터·화면 설계 계약

이 단계에서는 구현하지 않지만 실제 개발 때 아래 정보를 분리해 저장해야 한다.

### 14.1 필요한 버전 필드

- `construct_definition_version`
- `item_bank_version`
- `scoring_model_version`
- `report_copy_version`
- `code_scheme_version`

### 14.2 RO 결과에 필요한 값

- RO-EC 처음 드는 생각 연속 점수와 표준오차/신뢰구간
- G/A 글자 값과 상태: `CLEAR`, `NEAR_BOUNDARY`, `INSUFFICIENT_EVIDENCE`
- G/A 선명도와 적응형 추가 문항 필요 여부
- RO-RN 본인 전용 상세 신호의 별도 연속 점수·측정오차·발급 상태
- RO-RN이 대표 G/A 계산에 포함되지 않았음을 나타내는 scoring contract version
- 처음 드는 생각과 실제 나타나는 반응의 별도 점수·측정오차·차이 반복 여부
- 다층 프로필 상태와 프로필 모델 버전
- 사용자가 직접 선택한 비채점 이유의 존재 여부; 원문 동기 자동 추론 금지
- 응답 품질 신호: 결측, 지나치게 동일한 응답, 비정상적으로 빠른 응답 등

### 14.3 결과 화면 원칙

- 대표 글자와 함께 `원인·해결 ↔ 상대 마음`의 쉬운 설명을 보여준다.
- RO-RN은 `본인 전용 상세 신호`로 명확히 구분하고 대표 코드 근거처럼 배치하지 않는다.
- 빠른 코어는 정밀 코어보다 낮은 해상도와 확신도를 명시한다.
- `당신은 이런 사람`보다 `현재 응답에서는 이런 경향이 더 자주 보였다`를 사용한다.
- 관계 조언은 점수만으로 상대의 의도나 궁합을 단정하지 않는다.
- 처음 드는 생각과 실제 나타나는 반응이 다를 때 본성·가식·가면으로 표현하지 않는다.
- 이유는 사용자가 직접 고른 경우에만 `내가 직접 고른 이유`로 표시한다.

## 15. 사용자 승인 기록

2026-07-17 KST의 초기 승인은 기존 D/A 모순과 두 관계 신호의 분리 필요성을 확정한 설계 이력으로 보존한다.

1. 기존 D 설명은 RO-EC와 RO-RN이 갈린 상태를 낮은 RO로 잘못 해석했으므로 폐기한다.
2. RO-EC와 RO-RN은 서로 다른 신호로 측정한다.
3. 현재 legacy 문항과 빠른 코어 문항은 자동 승계하지 않는다.

2026-07-18 KST 사용자는 RO 24문항 감사의 A안을 승인해 다음을 최종 설계 방향으로 확정했다.

1. 고객 영역을 `관계에서 먼저 보는 것`으로 좁힌다.
2. RO-EC를 `공감적 관심 / 상대 마음 살피기`로 바꾼다.
3. 대표 G/A는 RO-EC의 관계 주의 방향만으로 산출한다.
4. `G = 원인·해결이 먼저 보이는 편`, `A = 상대 마음이 먼저 보이는 편`으로 설명한다.
5. RO-RN은 G/A에 합산하지 않고 본인 전용 상세 신호로 분리한다.
6. 본인 전용은 사용자에게 숨긴다는 뜻이 아니라 공개 프로필·피드·공유·비교에서 기본 제외한다는 뜻이다.
7. 대표 G/A와 `G → A` 과정 흐름은 같은 의미 범위를 사용한다.
8. 실제 점수 경계와 문항은 파일럿 검증 전까지 운영에 사용하지 않는다.
