# NUANG Assessment Architecture

작성일: 2026-07-10 KST

이 문서는 뉴앙의 핵심 기능인 `성향 검사` 제품 구조를 고정한다. 이후 검사 홈, 결과 리포트, 공유 리포트, 성향지도, 피드, 1:1 비교, 마이 리포트는 이 문서를 기준으로 판단한다.

## 핵심 원칙

뉴앙은 검사가 중심인 성향 기반 SNS다. 피드와 비교 기능은 검사를 대체하는 기능이 아니라, 검사 결과를 더 자주 다시 보게 만드는 확장 경험이다.

검사 경험의 우선순위:

1. 시작이 쉬워야 한다.
2. 짧고 재미있는 검사가 많아야 한다.
3. 정밀 코어는 서비스의 기본권처럼 항상 무료여야 한다.
4. 공식 대표 성향 코드는 정밀 코어에서 만든다.
5. 무료 검사들은 오래 머물 이유를 만들고 성향 해상도를 높인다.
6. 무료 주제 검사는 공식 대표 성향 코드를 즉시 흔들지 않지만, 승인된 성향 증거가 충분히 쌓이면 현재 대표 코드를 갱신할 수 있다.
7. 상세검사는 더 깊은 해석을 원하는 사람을 위한 확장 경험이다.
8. 어떤 검사도 진단, 치료, 위험 인물 판정, 궁합 점수처럼 보이면 안 된다.

## 코어 검사

### 빠른 코어

역할:

- 처음 온 사용자가 가장 빨리 뉴앙을 이해하는 입구다.
- 20문항 내외로 끝나야 한다.
- 예비 뉴앙 코드와 예비 코드 요약을 제공한다.
- 정밀 코어로 자연스럽게 이어지는 다리 역할을 한다.

금지:

- 정밀 코어와 같은 권위를 주지 않는다.
- 1:1 비교의 기준으로 쓰지 않는다.
- 유료 상세검사로 강하게 몰아붙이지 않는다.

### 정밀 코어

역할:

- 뉴앙의 공식 대표 성향을 만드는 기준 검사다.
- 60문항 내외로 구성한다.
- 뉴앙 코드, 32성향 이름, 코드 지도, 세부 신호를 제공한다.
- 공개 프로필과 1:1 비교의 기준이다.
- 검증된 경우 `처음 드는 생각`과 `실제 나타나는 반응`의 차이를 상세 프로필로 설명한다.

가격:

- MVP뿐 아니라 앞으로도 무료로 운영한다.
- 정밀 코어를 결제벽 뒤에 두지 않는다.
- 정밀 코어가 무료여야 사용자가 성향 코드로 소통하고 피드/비교 경험이 빠르게 성장한다.

### 세부 성향 지도

정밀 결과는 대표 5글자 아래에서 다섯 코드 자리 모두의 세부 성향을 보여준다.

| 코드 자리 | 세부 성향                                               |
| --------- | ------------------------------------------------------- |
| `E/I`     | 함께할 때의 에너지, 먼저 표현하기                       |
| `R/N`     | 미적 경험, 상상 확장, 지적 탐구                         |
| `G/A`     | 원인·해결과 상대 마음의 관계 주의 방향                  |
| `K/M`     | 시작하고 이어가기, 정리하고 계획하기, 맡은 일 이행 후보 |
| `C/Q`     | 감정 동요, 걱정·주저                                    |

- 세부 성향 지도는 시간 순서나 원인·결과를 뜻하지 않는다.
- 세부 성향이 갈리면 대표 글자만으로 차이를 숨기지 않는다.
- 세부 영문 코드는 의미·기억성·가치중립성 검증 전까지 만들지 않는다.
- 빠른 코어에서는 세부 성향을 확정적으로 노출하지 않는다.
- RO-RN 선택과 경계 존중하기는 G/A에 합산하지 않는 본인 전용 상세 신호다. 정밀 리포트에서는 보여줄 수 있지만 공개·공유·비교에서 기본 제외한다.

### 다층 성향·반응 프로필

사람은 처음 드는 생각과 실제 나타나는 반응이 다를 수 있다. 뉴앙은 이를 모순이나 거짓 응답으로 처리하지 않는다.

```text
처음 드는 생각
→ 실제 나타나는 반응
→ 사용자가 직접 고른 반응 이유
```

- 처음 드는 생각과 실제 나타나는 반응은 별도 문항·별도 점수 역할로 검증한다.
- 반응 이유는 사용자가 직접 선택한 비채점 맥락이며 코드에 반영하지 않는다.
- 빠른 코어는 다층 프로필을 확정하지 않는다.
- 정밀 코어도 서로 다른 상황의 유효 pair가 최소 3개 있고 차이가 반복될 때만 상세 설명 후보를 만든다.
- 다층 프로필은 여섯 번째 축이나 새 코드 글자가 아니다.
- 공개 프로필·피드·1:1 비교에는 기본적으로 노출하지 않는다.
- RO-EC 관계 반응 흐름은 대표 G/A와 동일한 `원인·해결 ↔ 상대 마음` 의미축을 사용하며, 검증을 통과한 정밀 결과에서만 `G → A`처럼 화살표를 포함해 설명한다.
- `Q → C`와 `K → M`은 각각 별도의 문항·안전·환경 오염 검증이 필요한 후속 연구 후보이며 현재 발급하지 않는다.
- E/I와 R/N은 현재 과정 화살표가 아니라 세부 성향 지도로 설명한다.

공통 측정 계약은 [NUANG_MULTILAYER_TRAIT_RESPONSE_PROFILE_MODEL.md](./NUANG_MULTILAYER_TRAIT_RESPONSE_PROFILE_MODEL.md)를, 전체 결과 계층은 [NUANG_ALL_CODE_DETAIL_PROFILE_ARCHITECTURE.md](./NUANG_ALL_CODE_DETAIL_PROFILE_ARCHITECTURE.md)를 따른다.

## 무료 주제 검사

무료 주제 검사는 뉴앙의 체류 시간을 만드는 핵심 콘텐츠다. 사용자는 `공식 대표 성향`을 한 번 보고 끝나는 것이 아니라, 여러 주제 검사를 가볍게 해보며 자신을 계속 발견해야 한다. 코어 검사가 사용자의 대표 성향을 완성한다면, 무료 주제 검사는 그 대표 성향 안에서 어떤 생활 패턴, 관계 방식, 감정 반응, 취향 신호가 강한지를 더 구체화한다.

원칙:

- 3~8문항 중심으로 짧게 만든다.
- 결과는 한 화면에서 바로 읽힌다.
- 결과는 낙인보다 대화 소재가 되어야 한다.
- 공식 대표 성향 코드를 즉시 흔들지 않는다.
- 승인된 축·세부 성향 매핑과 충분한 유효 가중치가 있을 때 현재 대표 코드를 갱신할 수 있다.
- 성향 해상도, 세부 해석, 추천 피드, 다음 추천 검사의 품질을 높인다.
- 사용자가 검사를 더 수행할수록 `내 성향을 더 정확히 이해하고 있다`는 느낌을 줘야 한다.
- 민감하거나 치료적인 주제는 열지 않는다.
- 피드 카드, 오늘의 질문, 공유 리포트와 연결할 수 있다.

초기 무료 검사 카테고리:

| 카테고리      | 예시                                            | 목적                         |
| ------------- | ----------------------------------------------- | ---------------------------- |
| 관계          | 대화 온도, 사과 방식, 거리감 리듬, 갈등 후 회복 | 친구/연인/가족 대화 소재     |
| 일상          | 충전 루틴, 아침/밤 리듬, 집중 전환, 정리 방식   | 매일 다시 해보기 좋은 소재   |
| 감정          | 기분 전환 방식, 서운함 표현, 위로 받는 방식     | 자기 이해와 피드 소재        |
| 사회적 에너지 | 모임 후 회복, 새 사람 대하는 방식, 팀플 역할    | SNS 공유와 비교 전 대화 소재 |
| 취향형 성향   | 계획 여행/즉흥 여행, 카페 자리 취향, 알림 대응  | 가볍고 재미있는 바이럴 소재  |
| 성장          | 피드백 받는 방식, 목표 유지 방식, 미루기 패턴   | 자기관리형 콘텐츠            |

무료 검사 운영 방식:

- 매주 1~3개 새 주제 또는 시즌 주제를 추가할 수 있게 설계한다.
- 홈에는 오늘의 추천 1개만 노출한다.
- 검사 탭에는 카테고리별로 정리하되, 첫 화면 선택지는 많지 않게 유지한다.
- 피드에는 결과 공유 카드나 오늘의 질문으로 일부 재사용한다.

성향 구체화 방식:

- 무료 주제 검사 결과는 기본적으로 `성향 보강 신호`로 취급한다.
- 성향 보강 신호는 검사 관찰 점수와 유효 가중치로 누적되며, 충분한 근거가 쌓이면 현재 대표 성향 코드에 반영될 수 있다.
- 보강 신호는 관계, 일상, 감정, 사회적 에너지, 취향, 성장 같은 영역별 해석을 더 정확하게 만든다.
- 사용자가 같은 영역의 검사를 여러 번 수행하면 해당 영역의 해석 신뢰도와 추천 정확도가 올라간다.
- 성향지도에서는 공식 코드 지도와 구분되는 `세부 힌트` 또는 `최근 더 선명해진 부분`으로 표현한다.
- 피드 추천에서는 무료 검사 신호를 사용해 오늘의 질문, 추천 주제, 결과 공유 카드의 노출 순서를 조정할 수 있다.
- 1:1 비교에는 기본적으로 정밀 코어와 공개된 공식 축만 사용한다. 무료 주제 검사 신호를 비교에 넣으려면 별도 공개 범위와 사용자 승인 후 설계한다.

대표 코드 갱신 원칙:

- 대표 코드는 `현재 가장 가까운 대표 성향`이며 평생 고정 유형처럼 표현하지 않는다.
- 현재 성향 점수는 여러 검사 관찰 점수의 가중 평균으로 계산할 수 있다.
- 유효 가중치는 측정량, 구성개념 직접성, 응답 품질, 승인 상태, 최근성, 반복 할인을 반영한다.
- 결제 여부는 항상 가중치 1.0이며, 유료라는 이유만으로 더 정확한 사람처럼 취급하지 않는다.
- 반대 방향 증거가 충분할 때만 코드가 바뀌고, 작은 변화는 경계 또는 최근 변화 안내로 표시한다.
- 현재 결과와 과거 결과를 모두 보존한다.

## 별난 성향 연구소

별난 성향 연구소는 무료 주제 검사 중 더 유별나고 가벼운 실험 영역이다.

역할:

- 사용자가 재미로 눌러보는 공간이다.
- 결과 이름과 문구가 SNS 공유 소재가 되기 좋게 만든다.
- 기본적으로 성향지도, 대표 성향 코드, 1:1 비교 기준에는 반영하지 않는다.
- 다만 특정 별난 검사가 실제 성향 축과 직접 매핑된다는 승인을 받으면 낮은 가중치의 성향 증거로 반영할 수 있다.

톤:

- 귀엽고 가볍지만 유치하면 안 된다.
- AI가 만든 농담처럼 보이는 과한 문구를 피한다.
- 결과는 생활 속 모습, 장점, 조심할 점, 작은 실험으로 구성한다.

## 상세검사

상세검사는 정밀 코어 이후 더 깊은 해석을 원하는 사용자를 위한 후속 확장이다.

현재 결정:

- 상세검사는 지금 바로 MVP 핵심 기능으로 열지 않는다.
- 상세검사 가격, 결제, 환불, 리포트 범위는 별도 승인 후 결정한다.
- 정밀 코어의 무료 권리를 침해하지 않는다.

상세검사 후보:

| 상세검사                 | 기반                       | 결과                                     |
| ------------------------ | -------------------------- | ---------------------------------------- |
| 관계 상세 리포트         | 정밀 코어 + 관계 문항      | 관계 거리감, 대화 방식, 갈등 회복 가이드 |
| 자기관리 상세 리포트     | 정밀 코어 + 실행/리듬 문항 | 목표 유지, 미루기, 루틴 설계             |
| 감정 반응 상세 리포트    | 정밀 코어 + 감정 문항      | 감정 반응 패턴, 회복 실험                |
| 커뮤니케이션 상세 리포트 | 정밀 코어 + 대화 문항      | 말하기/듣기/피드백 스타일                |

상세검사 금지:

- 병명, 장애, 치료 필요 여부를 판정하지 않는다.
- 1:1 비교의 필수 조건으로 만들지 않는다.
- 상세검사를 하지 않은 사용자를 열등하게 보이게 하지 않는다.

상세검사 비교 확장 원칙:

- MVP에서는 상세검사나 유료검사 기반 1:1 비교를 열지 않는다.
- 추후 상세검사 비교를 제공하더라도 두 사람이 모두 같은 검사를 수행한 경우에만 비교한다.
- 한 사람만 수행한 검사는 비교하지 않는다.
- 서로 다른 검사 결과끼리 억지로 연결하지 않는다.
- 두 사람 모두 해당 결과의 비교 사용을 허용한 경우에만 별도 비교 리포트를 만든다.

## 검사 홈 정보 구조

검사 탭의 첫 화면은 선택지를 많이 주지 않는다.

권장 구조:

1. 오늘 바로 시작할 검사 1개
2. 코어 검사
3. 무료 주제 검사
4. 별난 성향 연구소
5. 도움 연결 허브
6. 상세검사 준비 영역은 필요할 때만 하단에 조용히 노출

첫 화면에서 피해야 할 것:

- 모든 검사 카테고리를 한꺼번에 펼치는 것
- 유료 상세검사를 과하게 강조하는 것
- 검사 후 이동 가능한 모든 기능을 설명하는 것
- 치료성/위험성 검사를 재미형 검사처럼 배치하는 것

## 결과 사용 기준

| 결과             | 마이 리포트 | 성향지도                          | 공유 리포트 | 피드 소재 | 1:1 비교                                      |
| ---------------- | ----------- | --------------------------------- | ----------- | --------- | --------------------------------------------- |
| 빠른 코어        | 가능        | 예비 보기                         | 가능        | 가능      | 불가                                          |
| 정밀 코어        | 가능        | 공식 반영                         | 가능        | 가능      | 가능                                          |
| 무료 주제 검사   | 가능 후보   | 세부 힌트와 현재 성향 증거로 보강 | 가능 후보   | 가능      | 불가 기본                                     |
| 별난 성향 연구소 | 가능 후보   | 미반영                            | 가능 후보   | 가능      | 불가                                          |
| 상세검사         | 가능        | 미정                              | 별도 승인   | 제한      | MVP 불가, 추후 동일 검사 수행자끼리 가능 후보 |

## 개인정보와 안전

- 무료 주제 검사와 별난 성향 연구소는 직접 응답을 공개 리포트나 피드에 넣지 않는다.
- 무료 주제 검사의 성향 보강 신호는 사용자에게 이로운 해석, 현재 대표 코드 갱신, 추천 품질 개선에만 사용하며, 공개/비교 확장은 별도 승인 전에는 열지 않는다.
- 민감 검사와 도움 연결 이용 기록은 결과, 피드, 비교, 공유에 섞지 않는다.
- 자살, 자해, 우울, ADHD, 중독, 트라우마, 약물, 폭력 등은 재미형 검사로 열지 않는다.
- 임상 선별 도구를 열려면 출처, 목적, 한계, 비저장 기본값, 긴급 안내, 전문가 상담 권고를 별도 승인 게이트로 둔다.

## 다음 개발 순서

1. 검사 홈 문구와 구조를 이 문서 기준으로 정리한다.
2. 동적 성향 증거 모델의 검사별 가중치와 코드 변경 조건을 승인받는다.
3. 무료 주제 검사 카테고리와 축·세부 성향 매핑을 추가한다.
4. 별난 성향 연구소와 무료 주제 검사를 구분한다.
5. 무료 주제 검사 결과 저장/공유/성향 구체화 범위를 설계한다.
6. 정밀 코어 기반 1:1 비교를 구현한다.
7. 상세검사는 결제/상품 정책 승인 후 설계한다.
