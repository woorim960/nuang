# NUANG MVP 측정 베타 게이트

작성일: 2026-07-18 KST  
상태: `INTERNAL_BETA_RUNNABLE_EXTERNAL_RELEASE_BLOCKED`

## 지금 완료된 것

- 신규 고객 코드 순서 `E/I · R/N · G/A · K/M · C/Q`를 versioned candidate release로 고정했다.
- 150개 후보 문항을 revision 단위로 DB에 저장했다.
- 대표 코드에 사용하는 공개 10개 facet에서 `3 HIGH + 3 LOW`씩 60개 연구 베타 세트를 만들었다.
- `SM-RL`과 `RO-RN`은 대표 코드 채점에서 제외했다.
- 60개 문항 모두 상황 라벨과 질문을 분리했고, item ID·문항 release·코드 release·채점 release를 추적한다.
- 내부 자동 감사, 60문항 완료, 새 5글자 코드 계산, 결과 이동을 localhost에서 확인했다.
- DB의 candidate/beta release는 활성 release가 아니며, 세 검증 게이트가 모두 통과하기 전에는 활성화 RPC가 실패한다.

## 내부 검토 결과의 정확한 의미

현재 60개 중 내부 AI dry-run 판정은 다음과 같다.

- `PASS_TO_COGNITIVE`: 49개
- 인지 인터뷰에서 문구를 확인하거나 수정해야 하는 문항: 11개
  - `COPY_REVISE`: 7개
  - `HOLD_FOR_ACCESS`: 2개
  - `HOLD_FOR_RISK_REVIEW`: 1개
  - `CONSTRUCT_REWRITE`: 1개

이 11개는 일반 사용자에게 검증된 문항으로 공개할 수 있다는 뜻이 아니다. 통제된 인지 검토에서 오해 원인을 확인하기 위한 probe이며, 외부 베타 전 문구 판정과 revision을 잠가야 한다.

## 출시 단계

### 1. 내부 실행 — 현재 가능

- 개발 전용 URL에서 60문항과 예비 코드 흐름 확인
- 공유·피드·비교·공개 프로필 연결 금지
- 결과 문구는 `연구 중인 예비 신호`로 제한

### 2. 동의한 인지 검토 참여자 — 다음 필수 단계

- 11개 우선 문항과 나머지 대표 문항의 상황·질문·척도 이해 확인
- 본인 말로 다시 설명, 떠올린 상황, 응답 이유, 도덕적 정답처럼 느꼈는지 수집
- 큰 수정은 새 item revision으로 만들고 다음 회차에서 다시 확인

통과 조건:

- 의도한 상황과 단일 반응을 같은 의미로 이해한다.
- 반복되는 오해, 선택 장애, 접근성 위험이 남지 않는다.

### 3. 정량 파일럿 — 인지 검토 통과 후

- 개발 표본과 확인 표본을 분리한다.
- 응답 분포, 판단 어려움, 완료 시간, 문항-총점 상관, 교차 적재, 요인구조, omega, 재검사, 측정오차를 확인한다.
- 표본 수는 60문항·10 facet 구조와 분석 모형을 기준으로 사전 산출하고 임의의 작은 숫자로 고정하지 않는다.

통과 조건:

- 각 문항이 목표 facet을 안정적으로 측정한다.
- 5축·10 facet 구조가 독립 확인 표본에서도 재현된다.

### 4. 채점·문구 release

- 50점 단순 분할을 그대로 승인하지 않는다.
- 최소 응답 근거, 경계 구간, 대안 코드, quick/full 권위 차이를 데이터로 확정한다.
- 모든 리포트 문구를 점수·facet·검증 근거에 연결한다.

### 5. 제한 MVP 출시

- 활성화 RPC의 `cognitive_review`, `quantitative_pilot`, `reliability_and_structure`를 모두 `passed`로 바꾼 뒤 validated release를 활성화한다.
- 활성화 전 예비 결과는 공유·피드·비교로 전파하지 않는다.
- 기존 v0.9 결과를 새 코드로 자동 변환하지 않는다.

## 바로 이어서 구현할 한 가지

다음 개발 범위는 `동의한 베타 응답 수집`이다.

- 별도 연구 동의와 철회 가능 상태
- release ID가 고정된 문항별 응답·판단 어려움·응답 시간
- 직접 응답 client read 차단과 service-only 수집
- 중복 제출 방지와 삭제 경로
- 11개 probe 문항의 인지 질문 기록
- 분석용 익명화 export

이 수집 경로가 없으면 앱을 실행해도 M05~M07 검증을 완료할 데이터가 쌓이지 않는다.
