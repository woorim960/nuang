## 0. 결정

뉴앙의 기본 소통 단위는 검증된 대표 5글자 코드다.

```text
ERGKC
```

정밀검사에서 충분한 근거가 생긴 경우에만 관계 반응 흐름을 덧붙인다.

```text
ERGKC · 관계 반응 G → A
```

고객에게는 아래 순서로 설명한다.

```text
처음 드는 생각 G → 실제 나타나는 반응 A
```

`선택하는 반응`, `선택 행동`, `겉으로 보이는 행동`은 고객 문구로 사용하지 않는다.

## 1. G → A 두 자리의 의미

마음과 해결이 함께 있는 관계 상황에서만 아래 의미를 사용한다.

| 위치 | 고객 라벨          | G                                               | A                                          |
| ---- | ------------------ | ----------------------------------------------- | ------------------------------------------ |
| 1    | 처음 드는 생각     | 원인·결과·해결할 부분에 상대적으로 더 주의가 감 | 상대의 마음·안녕에 상대적으로 더 주의가 감 |
| 2    | 실제 나타나는 반응 | 사실·원인·해결 내용을 먼저 확인하거나 말하는 편 | 상대가 어떤 마음인지 먼저 확인하는 편      |

가능한 상세 흐름은 `G → G`, `G → A`, `A → G`, `A → A`다.

| 코드    | 쉬운 설명                                                                |
| ------- | ------------------------------------------------------------------------ |
| `G → G` | 해결할 부분이 먼저 보이고, 실제 반응도 해결 내용을 먼저 다루는 편이에요. |
| `G → A` | 해결할 부분이 먼저 보이지만, 실제로는 상대의 마음을 먼저 살펴요.         |
| `A → G` | 상대의 마음이 먼저 보이지만, 실제로는 필요한 해결을 먼저 다뤄요.         |
| `A → A` | 상대의 마음이 먼저 보이고, 실제 반응도 마음을 먼저 살피는 편이에요.      |

두 방향은 능력·도덕성·진심의 높고 낮음이 아니다.

여기서 `생각`은 머릿속에 완성된 문장만을 뜻하지 않는다. 상황을 접했을 때 무엇이 먼저 눈에 들어오거나 궁금해지고 신경 쓰였는지를 포함한다.

## 2. 화면 표기

### 2.1 처음 설명하는 결과 카드

```text
관계 반응 흐름
G → A

마음과 해결이 함께 있는 상황에서
처음 드는 생각 G → 실제 나타나는 반응 A

해결할 부분이 먼저 보이지만,
실제로는 상대의 마음을 먼저 살피는 편이에요.
```

첫 노출에서는 글자만 단독으로 보여주지 않는다. 화살표와 두 라벨을 함께 보여줘 글자 순서를 학습시킨다.

### 2.2 다시 보는 축약 카드

```text
관계 반응 G → A
G 처음 드는 생각 · A 실제 나타나는 반응
```

### 2.3 공유 문구

```text
내 뉴앙 코드는 ERGKC · 관계 반응 G → A
```

공유 이미지에는 `마음과 해결이 함께 있는 상황에서`라는 범위와 짧은 한 줄 설명을 포함한다. 화살표를 제거한 `GA` 문자열 복사는 facet 조합과 충돌할 수 있어 기본 제공하지 않는다.

## 3. 대표 코드와의 관계

- 대표 5글자는 전체 성향을 기억하고 공유하기 위한 기본 코드다.
- 대표 코드의 세 번째 `G/A`는 승인된 RO-EC 상황에서 반복되는 `처음 드는 생각`의 관계 주의 방향을 바탕으로 발급한다.
- 상세 `G → A`는 같은 RO-EC 의미축에서 처음 드는 생각과 실제 나타나는 반응을 설명한다.
- RO-RN 선택·존엄 존중은 대표 G/A나 상세 화살표에 합산하지 않는다.
- 대표 G/A는 여러 승인 상황의 안정적 방향이고, 상세 흐름은 마음·해결 상황의 반복 과정이므로 발급 기준과 측정오차를 각각 보존한다.
- 상세 흐름의 도착 방향 하나로 대표 코드의 세 번째 글자를 덮어쓰지 않는다.

UI는 `G 관계 대표 방향`과 `G → A 관계 반응 흐름`을 서로 다른 제목과 설명으로 구분하되, 두 G가 모두 `원인·해결이 먼저 보임`을 뜻한다는 점을 함께 알려준다.

## 4. 선택·존중 신호의 공개 범위

RO-RN은 거절 존중, 선택 여지, 압박·비하 억제를 다룬다. 낮은 방향이 단순한 취향처럼 소비되거나 낙인으로 공유될 위험이 있어 v0.3의 공개 관계 반응 흐름에는 포함하지 않는다. `본인 전용`은 사용자 본인에게 숨긴다는 뜻이 아니다.

- 검증 후 사용자 본인의 정밀 리포트에는 서술형 상세 신호와 상황별 설명으로 제공한다.
- 공개 프로필·피드·공유 이미지·상대 비교에는 기본 포함하지 않는다.
- 이유 응답은 본인에게만 보여주며 공유·비교에 사용하지 않는다.
- RO-RN을 별도 영문 코드로 만들려면 양쪽 방향의 가치 중립성, 이해도, 낙인 위험을 별도로 검증한다.
- `GAA`처럼 글자를 이어 붙이거나 세 번째 글자를 임의로 추가하지 않는다.

## 5. 발급 조건

관계 반응 흐름은 다음 조건을 모두 만족할 때만 발급 후보가 된다.

1. 정밀검사에서 서로 다른 중간 강도 상황의 유효 process pair가 최소 3개다.
2. 처음 드는 생각과 실제 나타나는 반응을 각각 계산할 수 있다.
3. 같은 패턴이 최소 2개 상황에서 반복된다.
4. 각 층위의 측정오차와 경계 상태가 승인 기준 안이다.
5. 인지 인터뷰에서 사용자가 두 글자의 순서를 정확히 설명한다.
6. 정량 파일럿에서 두 층위가 구분되고 재검사 시 허용 수준으로 반복된다.

조건을 충족하지 못하면 코드를 억지로 발급하지 않는다.

```text
아직 관계 반응 흐름을 정하기에는 응답이 조금 더 필요해요.
```

빠른 코어에서는 관계 반응 흐름을 발급하지 않는다.

## 6. 사용자 이해도 검증

인지 인터뷰에서 비교할 설명안:

1. `G → A`
2. `G → A · 처음 G / 실제 A`
3. `처음 G · 실제 A`

통과 질문:

- 10초 뒤 `G → A`의 왼쪽과 오른쪽 글자를 자신의 말로 설명할 수 있는가?
- 대표 코드의 `G`와 상세 흐름 왼쪽 `G`가 같은 의미축이라는 점을 이해하는가?
- `G → G`나 `A → A`를 더 좋은 사람으로 평가하지 않는가?
- `실제 나타나는 반응`을 타인이 관찰한 객관적 기록으로 오해하지 않는가?
- `나는 관계 반응이 G에서 A로 나타나`라고 자연스럽게 말할 수 있는가?

화면과 공유 기본안 모두 `G → A`를 사용한다. 최종 문구와 음성 읽기 방식은 인지 인터뷰 결과로 확정한다.

## 7. 데이터·UI 계약

```ts
type RelationshipResponseFlowResult = {
  domain: "RO";
  facet: "RO_EC";
  firstOrientation: "G" | "A" | "BOUNDARY" | "INSUFFICIENT";
  enactedResponse: "G" | "A" | "BOUNDARY" | "INSUFFICIENT";
  validPairCount: number;
  repeatedPatternCount: number;
  modelVersion: string;
  issuanceStatus:
    "ELIGIBLE" | "BOUNDARY" | "INSUFFICIENT" | "VALIDATION_BLOCKED";
};
```

- 내부 응답 층위는 `REPORTED_FIRST_ORIENTATION`, `ENACTED_RESPONSE`를 사용한다.
- 사용자 화면은 `처음 드는 생각`, `실제 나타나는 반응`을 사용한다.
- 상세 흐름이 없을 때 `--`, `XX`, 임의 중립 글자를 발급하지 않는다.
- `G → A`는 `firstOrientation`과 `enactedResponse`에서 UI가 렌더링하며 단일 DB 코드 문자열로 저장하지 않는다.
- 운영 DB·공유 payload에는 `modelVersion`과 발급 상태를 함께 저장한다.

## 8. 사용자 승인 기록

2026-07-18 KST 사용자 승인으로 다음 문구 방향을 확정했다.

1. 첫 글자는 `처음 드는 생각`을 뜻한다.
2. 두 번째 글자는 `실제 나타나는 반응`을 뜻한다.
3. 화면과 공유에서 화살표를 포함한 `G → A`를 사용한다.
4. 대표 5글자는 기본 코드로 유지하고 검증된 과정 흐름은 필요한 영역에 덧붙인다.
5. 대표 G/A와 관계 반응 흐름은 동일한 RO-EC 의미축을 사용한다.
6. RO-RN은 사용자 본인의 상세 리포트에는 보여주되 공개·공유·비교에서 기본 제외한다.

측정 검증과 운영 발급 승인은 별도 게이트다.
