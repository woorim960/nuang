## 코드 자리별 해석

### 1. 에너지가 시작되는 방식

S는 `조용히 살피는 에너지`다.

- 낯선 자리에서 분위기를 먼저 읽는다.
- 바로 튀어나가기보다 안전한 흐름을 확인한다.
- 혼자 정리할 시간이 있어야 다음 반응이 선명해진다.
- 강점은 신중함, 관찰력, 과한 반응을 줄이는 안정감이다.
- 조심할 점은 생각보다 괜찮은 기회도 너무 오래 지켜볼 수 있다는 점이다.

T는 `먼저 움직이는 에너지`다.

- 사람들과 함께 있을 때 반응이 빨라진다.
- 분위기를 열거나 선택지를 먼저 던지는 편이다.
- 생각이 정리되기 전에도 대화를 통해 방향을 찾을 수 있다.
- 강점은 추진력, 표현력, 흐름을 여는 힘이다.
- 조심할 점은 상대가 따라올 시간을 충분히 주지 못할 수 있다는 점이다.

비교 문장 기준:

- 둘 다 S이면 조용히 가까워지는 시간이 편하다.
- 둘 다 T이면 빠르게 이야기가 열리고 행동으로 이어지기 쉽다.
- S와 T가 다르면 한쪽은 여백을 원하고, 다른 한쪽은 바로 반응을 원할 수 있다.

### 2. 마음이 흔들릴 때의 반응

C는 `차분히 가라앉히는 마음`이다.

- 문제가 생겨도 우선 상황을 정리하려고 한다.
- 감정이 올라와도 표현보다 수습을 먼저 택할 수 있다.
- 안정적인 톤으로 주변을 진정시키는 역할을 한다.
- 강점은 침착함, 회복력, 과열을 낮추는 힘이다.
- 조심할 점은 속마음이 없는 것처럼 보일 수 있다는 점이다.

V는 `감정 신호가 선명한 마음`이다.

- 마음의 변화와 불편함을 빨리 알아차린다.
- 중요한 관계나 상황에서 감정의 온도가 선명해질 수 있다.
- 위험이나 어긋남을 빠르게 감지하는 편이다.
- 강점은 민감한 포착력, 진심의 표현, 관계 신호를 놓치지 않는 힘이다.
- 조심할 점은 상대가 그 반응을 부담이나 예민함으로 오해할 수 있다는 점이다.

비교 문장 기준:

- 둘 다 C이면 갈등이 커지기 전 차분히 정리하는 흐름이 맞기 쉽다.
- 둘 다 V이면 서로의 감정 신호를 빨리 알아차리고 반응한다.
- C와 V가 다르면 한쪽은 차분함으로, 다른 한쪽은 선명한 표현으로 안정감을 찾을 수 있다.

### 3. 일상을 굴리는 방식

F는 `유연하게 흘러가는 일상`이다.

- 상황이 바뀌면 계획도 함께 바꾸는 편이다.
- 정해진 틀보다 지금 필요한 선택을 우선한다.
- 여유와 즉흥성이 살아 있을 때 컨디션이 좋아진다.
- 강점은 적응력, 가벼운 전환, 압박을 낮추는 힘이다.
- 조심할 점은 마감이나 정리가 뒤로 밀릴 수 있다는 점이다.

O는 `정돈해서 완성하는 일상`이다.

- 해야 할 일을 구조화하고 마무리하려는 힘이 있다.
- 목록, 루틴, 약속, 정리된 환경에서 안정감을 얻는다.
- 계획이 있으면 에너지를 덜 낭비한다.
- 강점은 실행력, 지속성, 약속을 지키는 힘이다.
- 조심할 점은 바뀐 상황을 받아들이는 데 시간이 걸릴 수 있다는 점이다.

비교 문장 기준:

- 둘 다 F이면 계획보다 상황의 흐름을 보며 편하게 움직인다.
- 둘 다 O이면 약속과 루틴을 맞추기가 쉽다.
- F와 O가 다르면 한쪽은 숨통을, 다른 한쪽은 예측 가능성을 필요로 할 수 있다.

### 4. 관계를 맞추는 방식

D는 `거리와 선택을 존중하는 관계`다.

- 상대가 스스로 결정할 수 있는 여지를 중요하게 본다.
- 가까운 관계에서도 각자의 영역이 필요하다고 느낀다.
- 강요하지 않는 태도로 편안함을 만든다.
- 강점은 독립성, 선택 존중, 부담을 줄이는 힘이다.
- 조심할 점은 상대가 무심함이나 거리감으로 받아들일 수 있다는 점이다.

A는 `마음을 살피는 관계`다.

- 상대의 감정과 필요를 먼저 살피는 편이다.
- 가까운 사람이 힘들어하면 함께 해결하거나 챙기고 싶어진다.
- 관계의 온도와 작은 신호를 중요하게 본다.
- 강점은 공감, 배려, 따뜻한 연결감이다.
- 조심할 점은 상대가 간섭이나 부담으로 느낄 수 있다는 점이다.

비교 문장 기준:

- 둘 다 D이면 서로의 선택과 거리를 편하게 인정한다.
- 둘 다 A이면 마음을 살피고 챙기는 속도가 가까워진다.
- D와 A가 다르면 한쪽은 여백을 배려로 느끼고, 다른 한쪽은 확인을 애정으로 느낄 수 있다.

### 5. 생각이 넓어지는 방식

P는 `현실에 붙여보는 생각`이다.

- 아이디어를 실제로 쓸 수 있는지 확인하려 한다.
- 복잡한 이야기보다 지금 적용할 수 있는 기준을 선호한다.
- 감각, 사실, 구체적인 장면에서 판단이 또렷해진다.
- 강점은 실용성, 안정적인 판단, 현실 감각이다.
- 조심할 점은 가능성 탐색이 너무 빨리 닫힐 수 있다는 점이다.

E는 `가능성을 탐험하는 생각`이다.

- 한 가지 답보다 여러 가능성을 열어두는 편이다.
- 분위기, 의미, 아이디어, 원리를 탐구하는 데 흥미를 느낀다.
- 낯선 관점에서 새로운 길을 찾는 힘이 있다.
- 강점은 상상력, 연결 능력, 탐구심이다.
- 조심할 점은 상대가 너무 추상적이거나 멀게 느낄 수 있다는 점이다.

비교 문장 기준:

- 둘 다 P이면 현실적인 기준으로 빠르게 정리하기 쉽다.
- 둘 다 E이면 가능성과 의미를 넓혀가며 대화가 풍성해진다.
- P와 E가 다르면 한쪽은 바로 쓸 수 있는 답을, 다른 한쪽은 더 넓은 가능성을 원할 수 있다.

## 두 모습이 비슷한 자리

`두 모습이 비슷한 자리`는 코드 자리의 점수가 가운데 근처에 있어 양쪽 모습이 모두 나타날 수 있는 상태를 말한다.

내부 기준:

- v0.1 기준으로 45~55 구간을 가운데 근처로 본다.
- 사용자가 이 구간에 있으면 한쪽 코드 글자만으로 단정하지 않는다.
- 비교 리포트에서는 `이 자리는 상황에 따라 두 모습이 함께 보일 수 있어요`처럼 설명한다.

사용자-facing 문장:

- `이 자리는 한쪽으로 아주 강하게 기울기보다 두 모습이 함께 나타날 수 있어요.`
- `상황에 따라 다른 반응이 나올 수 있으니, 이름보다 실제 설명을 함께 보는 편이 좋아요.`
- `이 부분은 앞으로 검사를 더 하면서 더 선명해질 수 있어요.`

금지 문장:

- `경계입니다.`
- `애매한 유형입니다.`
- `불안정합니다.`
- `진짜 유형이 아직 정해지지 않았습니다.`

## Motif 규칙

첫 두 자리 조합은 motif를 정한다. motif는 캐릭터와 대표 이미지의 분위기를 정하는 시각 언어이며, 32개 성향을 4개 감정 이미지로 묶는다.

| 첫 두 자리 | Motif | 사용자-facing 이미지                        |
| ---------- | ----- | ------------------------------------------- |
| TV         | 불꽃  | 에너지가 빠르고 마음의 신호가 선명한 이미지 |
| TC         | 햇살  | 먼저 다가가지만 톤은 밝고 차분한 이미지     |
| SV         | 물결  | 조용히 살피지만 마음의 흐름은 섬세한 이미지 |
| SC         | 숲    | 조용하고 차분하게 안정감을 만드는 이미지    |

Motif는 성향의 우열이 아니다. 사용자가 자신의 코드를 쉽게 기억하도록 돕는 대표 이미지다.

## 역할명 규칙

마지막 세 자리는 코드 이름의 역할명을 정한다. 마지막 세 자리는 일상, 관계, 생각의 조합으로 해석한다.

| 마지막 세 자리 | 역할명        | 기억 문장                                         |
| -------------- | ------------- | ------------------------------------------------- |
| OAE            | 온기 탐험가   | 따뜻하게 살피며 새로운 가능성을 탐험한다.         |
| OAP            | 세심한 실천가 | 마음을 살피고 현실적인 방식으로 끝까지 챙긴다.    |
| ODE            | 새길 개척가   | 거리를 존중하면서도 새로운 길을 구조화한다.       |
| ODP            | 현실 추진가   | 필요한 거리를 두고 현실적인 실행을 밀고 간다.     |
| FAE            | 영감 동행자   | 유연하게 흐르며 상대의 마음과 가능성을 함께 본다. |
| FAP            | 편안한 조율가 | 부담을 낮추고 현실적인 편안함을 맞춘다.           |
| FDE            | 자유 발견가   | 각자의 여백 속에서 새로운 가능성을 발견한다.      |
| FDP            | 담백한 해결가 | 복잡하게 키우지 않고 현실적으로 정리한다.         |

사용자-facing 이름은 `모티프의 이름` 형태를 기본으로 한다.

예시:

- `불꽃의 온기 탐험가`
- `햇살의 현실 추진가`
- `물결의 새길 개척가`

## 1:1 비교 리포트 사전

MVP 비교 리포트는 서로의 현재 뉴앙 코드만으로 핵심 비교를 만든다. 공개 프로필은 비교 시작과 공개 범위 확인을 위한 통로이며, 비교의 핵심 의미는 두 사람의 뉴앙 코드 조합에 있다.

비교 리포트 입력:

- 내 현재 뉴앙 코드
- 상대 현재 뉴앙 코드
- 코드 이름
- 프로필 이미지
- 표시 이름
- 공개된 코드 자리별 요약
- 두 모습이 비슷한 자리 여부

비교 리포트에서 쓰지 않는 것:

- 직접 응답
- 원점수
- 비공개 정보
- 민감 검사 결과
- 상세검사 결과
- 유료검사 결과
- 궁합 점수
- 순위
- 관계 성공 확률

## 비교 리포트 화면 구조

1. 상단 요약
   - 두 사람의 프로필 이미지, 이름, 뉴앙 코드, 코드 이름
   - 한 문장 headline
   - 한 문장 body

2. 코드 리듬 요약
   - 같은 코드 자리 수
   - 다른 코드 자리 수
   - 두 모습이 비슷한 자리 안내
   - 점수표가 아니라 관계 리듬 설명으로 제공

3. 가까운 자리
   - 같은 글자이거나 차이가 작게 해석되는 자리
   - `편하게 맞는 장면`
   - `계속 유지하면 좋은 방식`

4. 조율이 필요한 자리
   - 다른 글자이거나 차이가 크게 해석되는 자리
   - `오해가 생기기 쉬운 장면`
   - `확인하면 좋은 신호`

5. 대화 가이드
   - 바로 쓸 수 있는 질문 2~3개
   - 상대를 추정하지 않는 문장

6. 조율 가이드
   - 실제 행동 제안 2~3개
   - 관계 성공을 예측하지 않고 실천 문장으로 제공

7. 공개 범위 안내
   - 서로 공개된 성향 정보만 사용했다는 안내
   - 비공개 항목과 민감 정보는 쓰지 않았다는 안내

## 비교 리포트 문장 규칙

### 같은 자리

같은 자리에서는 `같아서 좋다`로 끝내지 않고, 실제 장면을 말한다.

예시:

- `둘 다 조용히 살핀 뒤 가까워지는 흐름이 있어요. 약속을 잡을 때도 바로 결론보다 편한 분위기를 먼저 확인하면 안정적이에요.`
- `둘 다 먼저 움직이는 에너지가 있어요. 재미있는 일은 빠르게 시작되지만, 중간에 쉬어가는 신호도 함께 정하면 더 편해져요.`

### 다른 자리

다른 자리에서는 `차이가 크다`보다 서로 필요한 것이 무엇인지 말한다.

예시:

- `한 사람은 바로 반응하면서 가까워지고, 다른 한 사람은 여백이 있어야 편해져요. 답이 늦다는 이유만으로 관심이 없다고 판단하지 않는 게 좋아요.`
- `한 사람은 계획이 있어야 안심하고, 다른 한 사람은 상황을 보며 바꾸는 여유가 필요해요. 약속에서는 바뀔 수 있는 부분과 꼭 지킬 부분을 나누면 좋아요.`

### 두 모습이 비슷한 자리

두 모습이 비슷한 자리에서는 단정하지 않는다.

예시:

- `이 자리는 두 모습이 비슷하게 보여요. 상황이 편하면 한쪽 모습이, 부담이 커지면 다른 모습이 나타날 수 있어요.`
- `이 부분은 앞으로 검사가 더 쌓이면 더 선명해질 수 있어요. 지금은 이름보다 실제 장면 설명을 함께 보는 편이 좋아요.`

## 코드 자리별 비교 사전

| 코드 자리               | 같을 때                                    | 다를 때                                                       | 오해 장면                                                                | 조율 문장                                                   |
| ----------------------- | ------------------------------------------ | ------------------------------------------------------------- | ------------------------------------------------------------------------ | ----------------------------------------------------------- |
| 에너지가 시작되는 방식  | 가까워지는 속도와 표현 시작점이 비슷하다.  | 한쪽은 즉시 반응, 한쪽은 관찰 시간이 필요하다.                | 답이 늦으면 관심이 없다고 오해하거나, 빠른 반응을 압박으로 느낄 수 있다. | `바로 답이 필요해? 아니면 생각할 시간을 줄까?`              |

## Headline 생성 규칙

headline은 결과를 과장하지 않고, 두 사람의 리듬을 기억하기 쉽게 만든다.

허용 예시:

- `빠르게 열리는 에너지와 조용히 맞추는 리듬이 만났어요.`
- `비슷한 속도 위에 다른 확인 방식이 보여요.`
- `가까운 부분은 편하게 이어가고, 다른 부분은 신호를 맞추면 좋아요.`
- `서로 다른 리듬이 있지만, 대화 방식만 맞추면 오해를 줄일 수 있어요.`

금지 예시:

- `두 사람은 천생연분이에요.`
- `궁합 점수 92점`
- `이 관계는 위험해요.`
- `상대는 회피형이에요.`
- `당신이 더 맞는 사람입니다.`

## 상세검사 비교 확장 규칙

MVP에서는 상세검사나 유료검사 기반 비교를 열지 않는다.

추후 확장 조건:

- 두 사람이 모두 같은 검사를 수행했다.
- 두 사람 모두 해당 결과의 비교 사용을 허용했다.
- 비교 결과가 진단, 치료, 위험 판정으로 보이지 않는다.
- 한쪽만 수행한 검사는 비교하지 않는다.
- 서로 다른 검사끼리 억지로 비교하지 않는다.
