## 0. 결론

RO-EC와 RO-RN의 후보 은행은 각각 다음 연구 질문을 검증하는 자료로는 유지할 가치가 있다.

- `RO-EC`: 상대 마음과 원인·해결 중 어디에 자연스럽게 주의가 먼저 가는가
- `RO-RN`: 원하는 결과가 달라도 상대의 선택권과 존엄을 실제 반응에서 얼마나 지키는가

24문항 전체의 수량·방향·층위·주력/탐색 배치는 수학적으로 균형이 맞는다. 그러나 두 facet을 현재 방식으로 합쳐 대표 코드 `G/A`를 만드는 것은 통과하지 못했다.

핵심 이유는 세 가지다.

1. RO-EC의 양쪽은 비교적 가치중립적인 관심 순서지만, RO-RN의 낮은 방향에는 반복 압박과 사람 자체 평가가 포함된다.
2. 이를 한 점수로 합치면 `G = 해결·결과에 더 무게`가 낮은 선택 존중이나 비하 반응까지 중립적인 목표 지향성처럼 감싸거나, 반대로 G 사용자 전체를 무례한 사람처럼 보이게 할 수 있다.
3. 대표 `G/A`는 RO-EC+RO-RN을 뜻하지만 `G → A` 과정 흐름은 RO-EC만 뜻한다. 같은 글자가 화면에 따라 다른 측정 범위를 갖는다.

따라서 현재 판정은 다음과 같다.

```text
RO-EC 후보 연구                         유지
RO-RN 후보 연구                         수정 후 유지
24문항을 하나의 G/A 점수로 단순 합산     NO-GO
현재 의미의 G/A 고객 발급                HOLD
RO-RN 공개·공유·비교 코드화              BLOCKED
DB·seed·운영 화면 반영                   BLOCKED
```

이 판정은 `G/A 글자 자체를 폐기한다`는 뜻이 아니다. 2026-07-18 KST 사용자 승인으로 G/A를 가치중립적인 RO-EC 관계 주의 방향으로 좁히는 A안을 채택했다. RO-RN은 대표 G/A에 합산하지 않는 본인 전용 상세 신호로 분리한다.

## 3. 방향과 가치중립성 감사

### 3.1 RO-EC

RO-EC의 두 방향은 다음처럼 설명할 수 있다.

```text
HIGH: 상대의 마음·안녕이 먼저 궁금함
LOW: 일어난 일의 원인·결과·해결이 먼저 궁금함
```

둘 중 어느 쪽도 능력이나 도덕성으로 번역하지 않으면 비교적 가치중립적인 양극 설명이 가능하다. `공감 능력이 높다/낮다`, `논리적이다/차갑다`로 확대하지 않는 계약은 유지한다.

판정: `PASS_AS_VALUE_NEUTRAL_CANDIDATE`

### 3.2 RO-RN

RO-RN은 양쪽이 같은 종류의 중립적 선호가 아니다.

```text
HIGH: 거절과 선택 여지를 두고 사람 자체에 대한 평가를 억제함
LOW: 설득을 계속하거나 태도·성격까지 문제로 말할 수 있음
```

특히 RORN-P04-B와 P06-B는 바람직하지 않은 답이 너무 선명해 실제 성향보다 자기표현 관리 능력을 측정할 위험이 크다. 반대로 `계속 설명한다`는 문구는 정상적인 협의·책임 수행까지 낮은 존중으로 오채점할 수 있다.

RO-RN은 중요한 관계 안전·행동 신호지만, 이를 G라는 가치중립적 정체성 코드로 포장해서는 안 된다. 또한 낮은 점수를 `추진력`, `솔직함`, `리더십`으로 보상해 설명해서도 안 된다.

판정: `RESEARCH_VALUE_PRESENT_PUBLIC_IDENTITY_NO_GO`

## 4. 대표 G/A와의 의미 충돌

### 4.1 현재 대표 코드의 범위

현재 M01 계약은 다음 두 facet을 합쳐 G/A 한 글자를 만들도록 제안한다.

```text
RO-EC 공감적 관심
+ RO-RN 선택·존엄 존중
→ 관계 조율 방식 G/A
```

그러나 G 고객 문구 `해결·결과에 더 무게`는 RO-EC의 낮은 방향은 설명해도 RO-RN의 낮은 방향인 반복 압박·사람 평가를 정확히 설명하지 못한다.

### 4.2 과정 흐름과의 범위 불일치

현재 `G → A`는 RO-EC에서만 검증한다.

```text
처음 드는 생각 G = 원인·해결이 먼저 보임
실제 나타나는 반응 A = 상대 마음을 먼저 확인함
```

즉 같은 G/A가 다음 두 의미를 동시에 갖는다.

| 위치            | G/A가 뜻하는 범위        |
| --------------- | ------------------------ |
| 대표 5글자 코드 | RO-EC + RO-RN 상위 점수  |
| 관계 반응 흐름  | RO-EC의 생각 + 반응 순서 |

사용자는 `내 대표 코드는 G인데 왜 상세 화면에는 A 반응이 나오지?`, `선택 존중 점수도 화살표에 들어가나?`라는 질문에 한 문장으로 답을 얻기 어렵다. 표현층의 같은 문자는 같은 측정 범위를 가져야 한다.

판정: `NO_GO_SAME_SYMBOL_DIFFERENT_SCOPE`

## 5. 다층 프로필 감사

생각과 행동을 나누는 방식 자체는 의미가 있다. 다만 다음 조건을 지켜야 한다.

- 같은 또는 동등한 상황에서 두 층위를 별도로 묻는다.
- 한 pair가 아니라 최소 3개 유효 pair와 2개 이상의 반복 차이가 필요하다.
- 생각과 행동 차이를 `본성/가면`, `진짜/가짜`, `좋음/나쁨`으로 해석하지 않는다.
- 사용자가 직접 고른 이유만 비채점 맥락으로 보여준다.
- RO-EC와 RO-RN의 핵심 층위가 다르므로 두 process score를 하나로 평균 내지 않는다.

RO-EC의 `G → A` 가설은 독립 연구 후보로 유지할 수 있다. RO-RN 과정은 사회적 바람직성과 관계 안전 영향이 더 커 사용자 본인의 상세 리포트용 연구 후보로 두고 공개·공유·비교에서 기본 제외한다.

판정: `PASS_AS_SEPARATE_VALIDATION_HYPOTHESES`

## 6. 빠른·정밀 검사 사용 판정

### 6.1 빠른 코어

- 빠른 코어에서 process flow를 발급하지 않는다.
- G/A 의미 결정 전 RO 문항으로 대표 글자를 발급하지 않는다.
- RO-RN 후보는 현재 형태로 빠른 코드의 반대 극을 만드는 데 사용하지 않는다.
- 운영 UX 시뮬레이션에서는 fixture임을 내부적으로 명시하고 실제 결과처럼 공개하지 않는다.

### 6.2 정밀 코어

- RO-EC는 서로 다른 관계 거리와 상황에서 최소 3개 유효 pair를 확보한다.
- RO-RN은 거절 존중, 반복 설득 조절, 사람 평가 억제를 별도 하위 신호로 분석한다.
- RO-RN 최종 문항은 도덕적 정답이 덜 보이는 행동 빈도 문구와 간접 장면을 경쟁시킨다.
- 사회적 바람직성, 갈등 회피, 주도적 표현, 감정 반응, 역할 책임과의 변별타당도를 확인한다.
- 대표 코드와 상세 신호의 공개 범위는 G/A 의미 결정을 마친 뒤 확정한다.

## 8. 사용자 의사결정 결과

### A안 — 승인: 대표 G/A를 가치중립적인 관심 방향으로 좁힌다

```text
대표 G/A
G = 원인·해결이 먼저 보이는 편
A = 상대 마음이 먼저 보이는 편

RO-RN
= 선택과 경계 존중하기
= 대표 글자와 분리된 본인 전용 상세 신호
= 공개·공유·비교 기본 제외
```

필요한 변경:

- 세 번째 고객 영역을 `관계에서 먼저 보는 것`과 같은 좁은 명칭으로 재검토한다.
- G/A 점수는 검증된 RO-EC 문항만 사용한다.
- `결과`와 `선택`을 대표 양극 문구에서 제거한다.
- `G → A`는 대표 G/A와 동일한 측정 범위를 사용하므로 의미가 일치한다.
- RO-RN은 관계 안전과 선택 존중을 설명하는 별도 상세 신호로 검증한다.
- 5개 상위 영역이 기존 Big Five와 동일하다고 주장하지 않고 뉴앙의 맞춤 구조로 재검증한다.

장점:

- G와 A를 누구나 한 문장으로 이해하기 쉽다.
- 양쪽을 장점·차이로 설명할 수 있다.
- 대표 코드와 과정 흐름의 글자 의미가 같아진다.
- 해로운 행동을 재미있는 정체성 코드로 미화하지 않는다.

대가:

- 세 번째 축의 범위가 좁아진다.
- 기존 `관계 조율 방식`, `마음·선택`, Big Five Agreeableness 대조를 수정해야 한다.
- RO-RN은 대표 5글자 밖의 상세 신호가 된다.

### B안 — 미채택: 두 facet을 대표 코드에 유지하되 코드 의미와 흐름 표기를 다시 만든다

```text
대표 관계 코드
= 공감적 관심 + 선택·존엄 존중

RO-EC 과정 흐름
= 대표 코드와 다른 전용 표기 사용
```

필요한 변경:

- G/A 글자와 한국어 양극명이 두 facet 전체를 정직하게 설명하는지 처음부터 다시 검토한다.
- 낮은 선택 존중을 `목표 지향`으로 미화하지 않는 리포트 계약을 만든다.
- RO-EC 과정 흐름에는 G/A가 아닌 별도 쉬운 표기를 사용한다.
- 사회적 바람직성에 덜 취약한 RO-RN 문항을 재작성한다.

장점:

- Agreeableness 계열의 넓은 관계 성향을 대표 코드에 유지할 수 있다.
- 정밀 결과에서 두 facet의 분리 프로필을 풍부하게 설명할 수 있다.

대가:

- 한 글자로 쉽게 외우고 긍정적으로 공유하기가 더 어렵다.
- 낮은 극의 정확하고 비낙인적인 명칭을 찾기 어렵다.
- 기존 승인 코드와 과정 표기를 더 크게 바꿔야 할 수 있다.

### C안 — 기각: 현재 G/A와 합산을 그대로 유지한다

판정: `REJECT`

문항이 실제로 측정하는 낮은 선택 존중을 `해결·결과형`이라는 표현으로 덮게 되고, 대표 코드와 과정 흐름의 글자 범위도 달라진다. 제품의 핵심인 검사 신뢰를 해칠 위험이 가장 크다.

## 9. 다음 게이트

승인된 A안을 다음 순서로 반영한다.

1. M01-RO 관계 주의 방향·G/A 의미 계약 수정
2. 5글자 최종 의미 감사와 코드 신뢰 페이지의 RO 부분 수정
3. 전체 코드 세부 성향·과정 프로필 아키텍처의 G/A 범위 수정
4. RO-EC·RO-RN 후보 은행의 점수 역할·공개 범위 수정
5. RBL-01B 재감사 후 `PASS_TO_SM_CANDIDATE_DESIGN` 여부 결정
6. 통과 시 SM 후보 설계로 이동

문서 재기준화가 완료돼도 정량 검증과 scoring release 전에는 G/A 또는 G→A를 운영 사용자에게 발급하지 않는다.

## 10. A안 적용 후 RO 코어 재감사

대표 G/A와 과정 흐름 후보를 RO-EC 안에서 다시 계산했다.

대표 후보는 `3G·3A`, 과정 후보도 `3G·3A`로 방향 수가 균형을 이룬다.

| 후보층                                  | 문항 수 | G 방향 | A 방향 | 상황 범위             | 판정                                     |
| --------------------------------------- | ------: | -----: | -----: | --------------------- | ---------------------------------------- |
| 대표 G/A용 `REPORTED_FIRST_ORIENTATION` |       6 |      3 |      3 | P01~P05, 5개 상황     | `PASS_TO_EXPERT_AND_COGNITIVE_REVIEW`    |
| 과정 흐름용 `ENACTED_RESPONSE`          |       6 |      3 |      3 | P01~P04·P06, 5개 상황 | `PASS_AS_PROCESS_FLOW_CANDIDATE`         |
| 빠른 코어 우선 후보 P01~P04 처음 생각   |       4 |      2 |      2 | 4개 상황              | `PASS_AS_QUICK_FORM_CANDIDATE_NOT_FINAL` |

이 균형은 문항의 타당성이나 최종 채점을 승인한다는 뜻이 아니다. 다음 위험은 전문가 검토·인지 인터뷰·정량 파일럿에서 계속 확인한다.

- 마음 쪽 문장이 좋은 사람처럼 답하기 쉬운가
- 원인·해결 쪽 문장이 능력·논리성 자기평가로 읽히는가
- `먼저`가 시간 순서와 중요도 중 무엇으로 이해되는가
- 관계 거리, 해결 요청 여부, 상황 심각도가 응답을 지배하는가
- 처음 드는 생각과 실제 나타나는 반응이 통계적으로 구분되는가

RO-RN은 대표 5글자 코어에서 빠졌으므로 `RBL-01B` 코어 통과를 막지 않는다. 대신 별도 `DET-RO-RN` 연구 트랙에서 다음을 완료해야 본인 전용 상세 리포트에 연결할 수 있다.

1. 도덕적 정답이 지나치게 선명한 P04-B·P06-B 대체 문구 작성
2. 정상적인 설명과 반복 압박의 경계를 분명히 하는 상황 재작성
3. 갈등 회피·주도적 표현·감정 반응·역할 책임과 blind mapping
4. 본인 결과 문구의 낙인·방어 반응·행동 합리화 위험 검증
5. `SELF_ONLY` 가시성 계약과 삭제·동의·접근 통제 검증

최종 RBL-01B 판정:

```text
대표 G/A 후보 설계       PASS_TO_NEXT_M03_DOMAIN
G → A 과정 후보          PASS_TO_VALIDATION_TRACK
RO-RN 본인 전용 상세     SEPARATE_TRACK_REQUIRES_REVISION
운영 점수·DB·UI 발급      BLOCKED_UNTIL_SCORING_RELEASE
```
