## 0. 제품 결정

뉴앙은 대표 5글자만 보여주고 끝내지 않는다. 정밀 결과에서는 다섯 코드 자리 모두에 사용자가 자신의 차이를 이해할 수 있는 세부 정보를 제공한다.

다만 세부 정보는 세 종류를 구분한다.

```text
대표 뉴앙 코드
└─ 세부 성향 지도: 모든 코드 자리에 제공
   └─ 과정 프로필: 실제로 두 단계가 분리 측정된 영역에만 제공

본인 전용 상세 신호
└─ 대표 코드에 합산하지 않는 민감하거나 별도인 성향 정보
```

예시:

```text
ERGKC

사람 사이 에너지
├─ 함께할 때의 에너지
└─ 먼저 표현하기

관계에서 먼저 보는 것
├─ 관계 주의 방향 G/A
└─ 관계 반응 흐름 G → A

본인 전용 관계 상세
└─ 선택과 경계 존중하기
```

`세부 성향이 존재한다`와 `화살표 코드가 필요하다`는 같은 결정이 아니다.

## 1. 세 상세 층의 의미

### 1.1 세부 성향 지도 `FACET_MAP`

하나의 대표 코드에 실제로 기여하는 한 개 이상의 세부 성향이나 점수 근거를 보여준다.

- 시간 순서를 뜻하지 않는다.
- 원인과 결과를 뜻하지 않는다.
- 각 세부 성향의 연속 점수·선명도·근거 상황을 따로 보존한다.
- 세부 성향이 갈리면 평균으로 숨기지 않는다.
- 모든 코드 자리에 제공하는 기본 상세층이다.

### 1.2 과정 프로필 `PROCESS_FLOW`

같거나 동등한 상황에서 서로 다른 두 반응 단계가 반복 측정됐을 때만 화살표로 보여준다.

- 화살표 왼쪽과 오른쪽의 의미가 명확히 달라야 한다.
- 기존 facet 점수 두 개를 임의로 이어 붙여 만들지 않는다.
- 한 문항의 응답 차이나 한 번의 사례로 발급하지 않는다.
- 환경, 역할, 불안, 능력 같은 대체 설명을 검증한다.
- 화살표는 우열·성장·악화·인과를 뜻하지 않는다.

### 1.3 본인 전용 상세 신호 `PRIVATE_DETAIL_SIGNAL`

사용자 이해에는 중요하지만 대표 코드에 합산하지 않거나 공개 시 낙인·오해·관계 피해가 생길 수 있는 정보를 분리해 보여준다.

- 사용자 본인의 정밀 리포트에는 검증 후 쉬운 설명으로 제공한다.
- 대표 글자·점수·선명도를 바꾸지 않는다.
- 공개 프로필·피드·공유 이미지·상대 비교에는 기본 포함하지 않는다.
- `PRIVATE`는 사용자에게 숨긴다는 뜻이 아니라 본인 외 자동 공개를 막는 가시성 등급이다.
- 선택 공유를 검토하려면 별도의 이해도·동의·안전성 검증이 필요하다.
- 현재 첫 적용 후보는 `RO-RN 선택과 경계 존중하기`다.

## 3. 코드 자리별 상세 계약

### 3.1 E/I — 세부 성향 지도만 제공

현재 측정 구조:

- `SE-RE`: 함께할 때의 에너지
- `SE-AI`: 먼저 표현하기

사람과 함께할 때 에너지가 살아나는 것과 먼저 의견을 꺼내는 것은 다를 수 있다.

```text
사람들과 함께하는 시간은 좋아하지만
먼저 의견을 꺼내기까지는 시간이 필요해요.
```

현재 `E → I`를 발급하지 않는다.

`실제로 조용히 있었다`는 행동은 내향성 외에도 걱정·주저, 역할, 관계 거리, 대화 규칙의 영향을 받을 수 있다. SE-AI의 낮은 방향을 곧바로 `I 반응`으로 바꾸면 E/I의 원래 의미와 ER-WD가 섞인다.

후속 과정 연구를 열려면 `교류하고 싶은 생각`과 `실제 참여 반응`을 같은 상황에서 별도 측정하고, SE-AI·ER-WD·사회불안·역할 규범과의 변별타당도를 먼저 확인한다.

### 3.2 R/N — 3개 세부 성향 지도를 제공

현재 측정 구조:

- `OE-AE`: 미적 경험
- `OE-CI`: 상상 확장
- `OE-IE`: 지적 탐구

세 성향은 시간 순서가 아니라 관심이 향하는 서로 다른 대상이다.

```text
분위기와 아름다움에는 관심이 많이 가요.
장면의 앞뒤 이야기도 자주 상상해요.
새로운 원리를 더 알아보는 정도는 중간에 가까워요.
```

현재 `R → N`을 발급하지 않는다. 이를 사용하면 `구체적으로 이해한 뒤 가능성을 확장한다`는 새로운 사고 과정이 생기지만, 현재 OE 문항은 그 순서를 측정하지 않는다.

### 3.3 G/A — 같은 관계 주의 방향에서 대표 글자와 반응 흐름을 검증

현재 측정 구조:

- 대표 G/A 원본 `RO-EC`: 원인·해결과 상대 마음 중 무엇이 먼저 보이는가
- RO-EC process pair: 처음 드는 생각과 실제 나타나는 반응
- 분리 상세 신호 `RO-RN`: 선택과 경계 존중하기; G/A에 합산하지 않음

조건을 충족한 RO-EC 결과에서 다음 흐름을 파일럿한다.

```text
관계 반응 흐름
G → A

처음 드는 생각 G
해결할 부분이 먼저 떠올라요.

실제 나타나는 반응 A
상대의 마음을 먼저 살피는 편이에요.
```

가능한 흐름은 `G → G`, `G → A`, `A → G`, `A → A`다. 화면과 공유에서 화살표를 생략한 `GG`, `GA`, `AG`, `AA`를 기본 코드로 사용하지 않는다. 문자열만 보면 세부 facet 조합인지 과정 순서인지 구분하기 어렵기 때문이다.

RO-RN의 선택·존중 신호는 사용자 본인의 정밀 리포트에는 제공할 수 있다. 다만 낮은 방향이 가벼운 취향 코드로 소비되거나 낙인이 될 위험이 있어 대표 글자·과정 화살표·공개 프로필·피드 공유·상대 비교에서는 기본 제외한다. 추후 공유를 열더라도 사용자의 명시적 선택과 별도 이해도·안전성 검증이 필요하다.

### 3.4 K/M — 세부 성향 지도 우선, 실행 흐름은 후속 후보

현재 측정 구조:

- `SM-EP`: 시작하고 이어가기
- `SM-OS`: 정리하고 계획하기
- `SM-RL`: 맡은 일 이행 후보

세부 성향은 정밀 결과에서 각각 보여준다. 적용되는 facet 수는 승인된 `facet_model_version`을 따른다.

과정 연구 후보:

```text
의도한 방식 K → 실제 실행 M

계획은 꾸준히 이어가려고 세우지만,
실제 실행은 상황에 따라 달라지는 편이에요.
```

이 흐름은 `처음 드는 생각 → 실제 반응`이 아니라 `의도한 방식 → 실제 실행`이다. 시간·건강·자원·외부 일정·과제 난이도의 영향을 별도로 확인해야 하며, 기존 SM facet 점수만으로 `K → M`을 발급하지 않는다.

### 3.5 C/Q — 세부 성향 지도 필수, 감정 과정은 기본 비공개 후보

현재 측정 구조:

- `ER-IR`: 감정 동요
- `ER-WD`: 걱정·주저

두 세부 성향은 정밀 결과에서 따로 보여준다.

과정 연구 후보:

```text
마음속 반응 Q → 실제 나타나는 반응 C

마음속에서는 걱정이나 감정이 빠르게 커지지만,
말과 행동은 비교적 차분하게 나타나는 편이에요.
```

감정 경험과 외부 표현은 구분될 수 있지만, `Q → C`를 감정조절을 잘함, 감정을 숨김, 회복력이 좋음으로 해석하지 않는다. 공개 프로필·피드·1:1 비교·코드별 통계에는 기본 비공개이며, 본인 리포트에서도 별도 동의와 안전 문구를 검토한다.

## 4. 표기 문법

### 4.1 대표 코드

대표 코드는 공백이나 구분자 없이 5글자를 유지한다.

```text
ERGKC
```

### 4.2 세부 성향 지도

MVP에서는 세부 성향 이름과 방향·선명도를 표시한다. 검증되지 않은 영문 축약 코드를 만들지 않는다.

```text
함께할 때의 에너지
E 쪽 · 비교적 선명

먼저 표현하기
중간에 가까움
```

세부 성향마다 새 영문 문자쌍을 만들려면 다음을 별도 통과해야 한다.

- 양쪽 의미가 가치중립적이고 실제 측정 범위와 일치함
- 기존 10개 대표 글자와 혼동되지 않음
- 사용자가 10초 뒤 글자 의미를 설명할 수 있음
- 문자 순서를 외우지 않아도 카드만으로 이해 가능함
- 공유했을 때 능력·도덕·진단으로 오해하지 않음

### 4.3 과정 흐름

과정은 화살표를 포함해 표시한다.

```text
G → A
Q → C
K → M
```

- 화살표 없는 `GA`, `QC`, `KM`은 정식 과정 표기로 사용하지 않는다.
- `→`는 UI 표시 기호이며 DB의 단일 코드 문자열이 아니다.
- 저장할 때 시작 층위·방향과 도착 층위·방향을 별도 필드로 보존한다.
- 접근성 라벨은 `G에서 A로`가 아니라 `처음 드는 생각 G, 실제 나타나는 반응 A`처럼 의미를 읽는다.

## 5. 결과 화면 정보 순서

정밀 결과의 기본 순서:

```text
1. 대표 뉴앙 코드와 한 줄 설명
2. 다섯 코드 자리 요약
3. 선택한 코드 자리의 세부 성향 지도
4. 근거가 충분한 경우 과정 프로필
5. 본인 전용 상세 신호
6. 상황에 따른 차이와 선명도
7. 사용자가 직접 고른 비채점 이유
8. 코드 지도 또는 다음 검사
```

한 화면에서 다섯 영역의 모든 세부 정보를 펼치지 않는다. 다섯 자리 요약에서 하나를 선택하면 해당 상세 카드가 열리는 점진적 공개 방식을 사용한다.

## 6. 발급·노출 게이트

### 6.1 세부 성향 지도

- 빠른 코어: 대표 예비 코드와 짧은 요약만 제공하고 facet을 확정하지 않는다.
- 정밀 코어: 승인된 facet별 최소 문항·신뢰도·경계 기준을 통과했을 때 제공한다.
- facet 증거가 부족하면 빈 글자나 임의 중립 코드를 만들지 않는다.

### 6.2 과정 프로필

다음을 모두 통과해야 한다.

1. 동일하거나 동등한 중간 강도 상황의 유효 process pair가 최소 3개다.
2. 각 반응 층위에 HIGH·LOW 방향 문항이 포함된다.
3. 같은 흐름이 최소 2개 상황에서 반복된다.
4. 각 층위의 측정오차와 경계 상태가 승인 기준 안이다.
5. 인지 인터뷰에서 두 단계와 화살표를 정확히 이해한다.
6. 대체 설명과 인접 facet의 교차 적재가 허용 범위 안이다.
7. 정량 파일럿에서 두 층위 분리와 반복성이 확인된다.
8. 공개 범위와 고객 문구가 별도 승인된다.

## 9. 검증 우선순위

```text
1. G → A 관계 반응 흐름
2. Q → C 마음 반응 흐름의 측정·안전 가능성
3. K → M 의도·실행 흐름의 환경 오염 가능성
4. E/I 과정 모형 필요성 재검토
5. R/N 과정 모형 필요성 재검토
```

E/I와 R/N은 세부 성향 지도만으로 충분한 자기이해가 제공되는지 먼저 확인한다. 사용자가 화살표를 원한다는 이유만으로 측정하지 않은 과정을 추가하지 않는다.

## 10. 검증 질문

- 대표 코드와 세부 성향 지도를 서로 다른 해상도로 이해하는가?
- `G → A`를 G에서 A로 성격이 변했다는 뜻으로 오해하지 않는가?
- 화살표 왼쪽과 오른쪽의 라벨을 10초 뒤 설명할 수 있는가?
- 세부 성향이 갈려도 대표 코드가 오류라고 느끼지 않는가?
- 상세 정보가 자신을 더 정확히 설명한다고 느끼는가?
- 세부 영문 코드가 없어도 카드 내용을 쉽게 기억하고 말할 수 있는가?
- 공유하고 싶지 않은 상세 정보가 기본 비공개임을 이해하는가?

## 11. 연구 근거와 적용 범위

- [Soto & John (2017), BFI-2](https://escholarship.org/uc/item/16x6n05t) — 넓은 5개 domain 아래 15개 facet을 둔 위계 구조와 facet 수준 정보의 유용성
- [DeYoung, Quilty, & Peterson (2007), Between Facets and Domains](https://doi.org/10.1037/0022-3514.93.5.880) — 각 Big Five domain 아래 관련되지만 구분되는 두 aspect가 존재할 수 있다는 근거
- [Gross (1998), Antecedent- and response-focused emotion regulation](https://pubmed.ncbi.nlm.nih.gov/9457784/) — 감정 경험·행동 표현·생리 반응을 구분해 살펴야 한다는 C/Q 과정 후보의 근거
- [Horstmann et al. (2021), Assessing Personality States](https://doi.org/10.1002/per.2266) — 상황별 성향 상태를 측정할 때 개인 내 변산과 변별타당도를 별도로 확인해야 한다는 방법론

위 연구는 뉴앙의 문자·한국어 문항·화살표 코드가 이미 타당하다는 증거가 아니다. 뉴앙의 각 상세층은 자체 전문가 검토·인지 인터뷰·정량 파일럿을 통과해야 한다.

## 12. 사용자 승인 기록

2026-07-18 KST 사용자 승인으로 다음 방향을 확정했다.

1. 다른 코드 자리도 대표 글자보다 자세한 성향을 제공한다.
2. 모든 코드 자리에 세부 성향 지도를 제공한다.
3. 모든 영역을 같은 화살표 구조에 억지로 맞추지 않는다.
4. G/A와 `G → A`를 동일한 RO-EC 범위에서 우선 파일럿하고, RO-RN은 본인 전용 상세 신호로 분리한다.
5. C/Q, K/M 과정 흐름은 후속 후보로 검토한다.
6. E/I와 R/N은 현재 세부 성향 지도를 우선한다.

운영 점수·상세 영문 문자·공개 범위의 최종 승인은 별도 게이트다.
