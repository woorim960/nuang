# NUANG Free Topic Assessment Mapping Proposal

작성일: 2026-07-10 KST

> 코드 체계 주의: 아래 `S/T · C/V · F/O · D/A · P/E`와 OE-AS 표기는 현재 구현된 v0.1 매핑을 설명하는 역사적 값이다. 2026-07-17 현재 작업 범례는 `E/I · R/N · G/A · K/M · C/Q`이며, OE는 OE-AE 미적 경험·OE-CI 상상 확장·OE-IE 지적 탐구의 3-facet을 사용자 선택으로 채택했다. 전체 측정 검증과 migration 설계가 끝나기 전에는 이 문서만 근거로 운영 데이터나 DB를 변환하지 않는다.

이 문서는 Stage 026 구현 전 사용자 승인을 받기 위한 제안서다. 무료 주제 검사가 단순 재미 콘텐츠로 끝나지 않고, 사용자가 검사를 더 할수록 현재 대표 성향과 세부 해석이 더 정확해지도록 5축·10세부 성향 매핑 기준을 제안한다.

승인·구현 상태:

- 2026-07-10 KST 사용자 승인 완료.
- v0.1 카탈로그와 축·세부 성향 매핑은 `src/features/assessment/free-topic-assessments.ts`에 구현했다.
- 첫 실행 범위는 10개 무료 주제 검사로 확정했다.
- 결과 저장은 로컬 우선이며, 로그인 사용자는 온라인 상태에서 서버 동기화를 시도한다.
- `supabase/migrations/202607100002_free_topic_results.sql`은 Supabase에 적용했고, 원격 DB에서 `assessment.free_topic_result` 접근을 확인했다.
- 결과 URL을 다시 열 때 로컬 결과가 없으면 서버 저장 결과를 `localResultId`로 찾아 복원한다.
- 사용자 화면에는 `RO-EC`, `SE-AI` 같은 내부 채점 키를 노출하지 않고 한국어 세부 신호명과 해석 문구만 보여준다.
- 공개 프로필과 1:1 비교 연결은 아직 열지 않는다.

## 목적

무료 주제 검사는 뉴앙의 핵심 체류 콘텐츠다. 사용자는 코어 검사를 한 번 끝내고 떠나는 것이 아니라, 짧은 무료 검사를 계속 해보며 `내 성향이 더 선명해지고 있다`는 경험을 해야 한다.

이 단계의 핵심 결정:

- 무료 주제 검사는 기본적으로 `free_topic` source weight 0.35를 사용한다.
- 대표 코드는 즉시 흔들지 않고, 승인된 매핑과 충분한 반복 증거가 있을 때만 갱신한다.
- 직접 응답은 공개 프로필, 피드, 공유 리포트, 1:1 비교에 노출하지 않는다.
- 민감/치료성 주제는 무료 주제 검사로 열지 않는다.

## 5축·10세부 기준

| 5축 | 코드 방향 | 10세부 |
|---|---|---|
| 사람 사이 에너지 | S/T | 함께하는 에너지 `SE-RE`, 먼저 표현하기 `SE-AI` |
| 마음의 반응 | C/V | 감정이 커지는 정도 `ER-IR`, 걱정과 망설임 `ER-WD` |
| 일상 리듬 | F/O | 실행과 마무리 `SM-EP`, 정리와 계획 `SM-OS` |
| 관계 방식 | D/A | 상대 마음 살피기 `RO-EC`, 기준과 선택 존중 `RO-RN` |
| 감각과 생각 | P/E | 감각과 상상 `OE-AS`, 아이디어 탐구 `OE-IE` |

## 사용자 경험 원칙

검사 탭:

- 사용자는 `코드에 반영되는 검사`, `코드에 반영되지 않는 검사` 같은 내부 용어를 볼 필요가 없다.
- 대신 `내 성향을 더 선명하게 보는 짧은 검사`처럼 이해 쉬운 문구를 쓴다.
- 첫 화면 선택지는 적게 유지한다.

결과 화면:

- 무료 주제 검사 결과는 한 화면에서 바로 읽히게 만든다.
- 필요할 때만 `이 결과가 내 성향 해석을 더 선명하게 만들었어요` 정도로 안내한다.
- 대표 코드가 바뀌는 경우에는 `최근 여러 검사에서 반복 확인되어 현재 가장 가까운 코드가 달라졌어요`처럼 설명한다.

성향지도:

- 무료 검사 신호는 `최근 더 선명해진 부분`으로 보여준다.
- 균형 구간에 가까운 자리는 양쪽 모습이 모두 나타날 수 있음을 설명한다.

마이:

- 최신 대표 성향은 상단에 간결하게 보여준다.
- 이전 리포트는 `내 리포트`에서 보존한다.

## 저장과 공개 범위

저장 후보:

- 검사 종류
- 결과 요약
- 축·세부 성향 증거
- 유효 가중치
- 알고리즘 버전
- 결과 생성 시각

공개 금지:

- 직접 문항 응답
- 원점수 payload
- 응답 품질 플래그
- 민감/도움 연결 기록
- OAuth, 이메일, 연락처

1:1 비교:

- v0.1에서는 무료 주제 검사 신호를 1:1 비교에 넣지 않는다.
- 비교에는 기본적으로 현재 공개된 대표 성향과 공식 축만 사용한다.
- 무료 주제 검사 신호를 비교에 포함하려면 별도 공개 범위와 사용자 승인 후 설계한다.
