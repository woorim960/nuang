## 0. 실행 결론

M04와 M05는 한 번의 회의나 사용성 테스트가 아니다.

```text
M04 독립 전문가 blind mapping
→ 목표 facet·방향·오염을 모르는 상태에서 개별 판단
→ 제출 잠금 후 정답 키 공개·합의 조정
→ 수정·보류·제외 및 새 revision 발행
→ 세 form 재생성·자동 균형 검증

M05 2030 사용자 인지 인터뷰
→ 실제 검사처럼 먼저 자연 응답
→ 문항을 다시 보며 이해·회상·판단·응답 선택 확인
→ 2~3회 반복, 크게 고친 문항은 새 참여자에게 재검증
→ 파일럿 후보만 M06으로 이동
```

두 단계의 목적은 `좋은 문장처럼 보이는가`가 아니라 다음을 증명하는 것이다.

- 의도한 facet을 다른 성향·능력과 구분해 측정하는가?
- 한국어 사용자가 같은 종류의 상황과 반응으로 이해하는가?
- 특정 학력·매체·감각·디지털 환경 때문에 응답이 달라지지 않는가?
- HIGH/LOW 중 한 방향이 더 좋은 사람처럼 보이지 않는가?
- 상황 라벨·질문·응답 척도가 하나의 판단 과정으로 이어지는가?

M04·M05를 통과해도 R/N 코드를 바로 발급하지 않는다. 이후 M06 정량 파일럿과 M07 채점 모형 검증이 남는다.

## 1. 연구 거버넌스와 시작 전 게이트

### 1.1 필수 책임자

| 역할                     | 필수 책임                                               |
| ------------------------ | ------------------------------------------------------- |
| 측정 책임자              | 구성개념·판정 기준·변경 승인, 최종 연구 게이트 서명     |
| 심리측정 전문가          | blind mapping 설계, 기준 사전 등록, 분석·해석 감독      |
| 성격심리 전문가          | OE 구조·인접 성향·리포트 주장 범위 검토                 |
| UX 리서처                | 사용자 모집·인터뷰·UI 검증·분석 진행                    |
| 한국어 문항 검토자       | 쉬운 말·문법·상황 범위·세대 언어 검토                   |
| 접근성·편향 검토자       | 감각·읽기·디지털·문화 편향과 대체 방식 검토             |
| 개인정보·연구윤리 책임자 | 동의·녹음·보관·삭제·외부 공유·IRB 필요 여부 결정        |
| 제품 오너                | 범위·예산·일정 승인, 측정 판단을 단독으로 덮어쓰지 않음 |

한 사람이 여러 역할을 맡을 수 있지만 `문항 작성자 = 유일한 검토자 = 최종 승인자` 구조는 금지한다. 독립된 심리측정 책임자의 서명 없이 결과를 운영 검사로 연결하지 않는다.

### 1.2 모집 시작 전 하드 게이트

다음 값이 모두 채워지기 전 모집·녹음·개인정보 수집을 시작하지 않는다.

- 연구 목적과 허용되는 결과 사용 범위
- 책임자·담당자·접근 권한
- 전문가·사용자 모집 기준과 보상
- 동의서·철회·질문 건너뛰기 절차
- 녹음·화면 기록 여부와 별도 동의
- 원본 녹음·메모·연락처·가명 데이터의 정확한 보관 기간
- 삭제·익명화·접근 기록 절차
- 외부 연구자·업체 제공 범위
- 기관 연구윤리 심의 또는 면제 판단 필요 여부
- 사고·민원·철회 요청 대응 담당자

제품 연구라는 이유로 연구윤리 검토가 불필요하다고 가정하지 않는다. 학술 발표·외부 공동연구·민감정보 수집 계획이 생기면 모집 전 공식 판단을 받는다.

## 2. 사전 등록할 연구 질문

### M04 전문가 질문

1. 36문항의 첫 번째 target facet을 blind 상태에서 구분할 수 있는가?
2. HIGH/LOW 방향을 의도대로 구분할 수 있는가?
3. 문항이 관심·관여가 아니라 능력·교육·접근·다른 성향을 측정하는가?
4. 각 facet의 내용 범위가 충분하고 특정 장면에 몰리지 않는가?
5. R/N의 양쪽 방향이 우열·낙인 없이 해석 가능한가?

### M05 사용자 질문

1. 사용자는 상황 라벨과 질문을 의도한 하나의 장면으로 이해하는가?
2. 1·3·5점의 차이를 반복 빈도로 일관되게 선택하는가?
3. 6개월 회상 지시가 실제 응답 근거로 사용되는가?
4. `판단하기 어려워요`의 이유가 접근·경험·문장·개인정보 중 무엇인가?
5. 분위기 관심·상상 확장·원리 탐구를 서로 다른 질문으로 느끼는가?
6. 부정문과 `~만·~까지만` 문장에서 응답 방향을 뒤집어 이해하지 않는가?
7. 특정 방향이 감수성·창의성·지능·현실성·효율성처럼 더 좋아 보이는가?
8. 모바일 UI가 이해와 응답을 방해하거나 특정 답을 유도하는가?

연구 질문·판정 기준·제외 규칙을 첫 응답을 보기 전에 버전 고정한다. 변경하면 이유와 시점을 기록하고 탐색적 변경으로 표시한다.

## 3. M04 독립 전문가 구성

### 3.1 모집 목표

작업 모집 목표는 독립 검토자 6~8명이다. 숫자만 채우지 않고 다음 역할을 모두 포함한다.

- 심리측정·검사 개발 경험자 최소 2명
- 성격·사회심리 연구 경험자 최소 1명
- 한국어 설문·문항·언어 검토 경험자 최소 1명
- 2030 모바일 UX 리서처 최소 1명
- 접근성·공정성·문화 편향 검토 경험자 최소 1명

한 검토자가 두 역할을 충족할 수 있다. 그러나 전체 검토자가 6명 미만이거나 심리측정·성격심리 역할이 비어 있으면 문항을 `PASS`로 확정하지 않는다.

### 3.2 독립성·이해상충

- 문항 작성자는 blind reviewer 집계에서 제외하고 adjudication에만 참여한다.
- 뉴앙 투자·고용·납품 관계와 문항 개발 참여 여부를 받는다.
- 검토자끼리 1차 응답을 공유하지 않는다.
- 원문 척도나 기존 뉴앙 seed 문항을 복원해 맞히는 과제가 아님을 알린다.
- 보상은 특정 결과·통과 문항 수와 연결하지 않는다.

## 4. M04 blind packet

### 4.1 검토자에게 제공하는 것

- 연구 목적과 비밀유지·이해상충 안내
- 공통 응답 지시와 1~5점·판단 어려움 의미
- 세 OE facet과 인접 구성개념의 중립적 정의
- 무작위 순서의 `contextLabel + promptText`
- 아래 평가 항목과 자유 서술란

facet 정의 목록:

| 선택지                       | 중립 정의                                                         |
| ---------------------------- | ----------------------------------------------------------------- |
| 미적 경험                    | 실제 장면·소리·공간의 분위기와 인상에 관심이 머무는 경향          |
| 상상 확장                    | 주어진 내용 바깥의 장면·이야기·가능성을 새로 이어서 떠올리는 경향 |
| 지적 탐구                    | 개념·이유·배경·서로 다른 설명을 필요한 범위보다 더 알아보는 경향  |
| 감정 동요                    | 좌절·변화에서 불편 감정이 빠르게 커지는 경향                      |
| 걱정·주저                    | 불확실성·평가에서 잘못될 가능성이 반복되어 주저하는 경향          |
| 실행·지속                    | 정한 행동을 시작·복귀·지속·마무리하는 경향                        |
| 선택·존엄 존중               | 관계에서 상대의 마음·선택 여지를 먼저 고려하는 경향               |
| 능력·접근·방법 효과          | 감각·심상·지능·기억·읽기·검색·문장 형식 영향                      |
| 어느 것에도 명확히 맞지 않음 | 위 정의 중 하나로 안정적으로 분류하기 어려움                      |

### 4.2 검토자에게 가리는 것

- 실제 candidate ID와 target facet
- HIGH/LOW key
- 주력·탐색 상태
- 작성 근거·예상 오염·이전 사용자 피드백
- 다른 검토자의 판단
- 기존 빠른·정밀 포함 여부

관리자는 `opaque_item_id`와 실제 ID의 대응표를 별도 접근 영역에 보관한다.

## 5. M04 두 단계 평가표

### 5.1 Stage 1 — blind mapping

첫 제출이 잠기기 전에는 actual target·방향·작성 근거를 보여주지 않는다.

| 순서 | 평가                      | 응답 형식                                              |
| ---: | ------------------------- | ------------------------------------------------------ |
|    1 | 가장 가까운 구성개념      | 위 목록 중 하나, 필수                                  |
|    2 | 두 번째로 가까운 구성개념 | 목록 중 하나 또는 없음                                 |
|    3 | 방향 추정                 | 첫 선택 구성개념의 높은 방향 / 낮은 방향 / 구분 어려움 |
|    4 | 문장 명확성               | 1 매우 모호 ~ 4 매우 명확                              |
|    5 | 단일 반응성               | 1 여러 반응 ~ 4 하나의 반응                            |
|    6 | 일상 보편성               | 1 특정 경험 의존 ~ 4 넓게 답변 가능                    |
|    7 | 응답 척도 적합성          | 1 빈도로 답하기 어려움 ~ 4 자연스러움                  |
|    8 | 좋아 보이는 방향          | HIGH / LOW / 비슷함 / 모름                             |
|    9 | 오염 flag                 | 능력·접근·교육·매체·SM·RO·ER·기억·주의·기타 복수 선택  |
|   10 | 치명적 공정성·낙인 위험   | 없음 / 있음 + 근거                                     |

### 5.2 Stage 2 — target 공개 검토

Stage 1 제출을 수정할 수 없게 잠근 뒤 actual target·방향·의도한 응답 과정만 공개한다. 기존 작성 근거·예상 위험·다른 reviewer 결과는 아직 보여주지 않는다.

| 순서 | 평가                      | 응답 형식                                          |
| ---: | ------------------------- | -------------------------------------------------- |
|    1 | target 관련성             | 1 전혀 아님 ~ 4 매우 직접적                        |
|    2 | 방향 적합성               | 1 맞지 않음 ~ 4 매우 직접적                        |
|    3 | facet 범위 기여           | 이미 있는 내용 반복 / 일부 기여 / 중요한 범위 기여 |
|    4 | 인접 구성개념 분리 가능성 | 1 거의 분리 안 됨 ~ 4 명확히 분리됨                |
|    5 | 권고                      | 유지 / 문구 수정 / 구성개념 재작성 / 보류 / 제외   |
|    6 | 최종 근거                 | 자유 서술, 필수                                    |

blind mapping은 구분 가능성을, target 공개 평정은 의도한 내용과의 관련성·포괄성을 확인한다. 두 결과를 하나의 평균으로 합쳐 문제를 숨기지 않는다.

## 6. M04 제시 순서와 품질관리

1. reviewer마다 36문항 순서를 독립 무작위화한다.
2. 같은 상황의 HIGH/LOW 쌍은 최소 8문항 이상 떨어뜨린다.
3. facet별 문항을 한 덩어리로 보여주지 않는다.
4. 직접 부정문 3개가 연속되지 않게 한다.
5. 답변을 시작한 뒤 정의표는 다시 볼 수 있지만 target key는 볼 수 없다.
6. 제출 전 누락만 확인하고 응답 방향을 교정해주지 않는다.
7. 제출 시 packet hash·순서 seed·프로토콜 버전을 저장한다.
8. Stage 1 제출이 잠긴 뒤 actual target·방향·의도한 응답 과정만 공개한다.
9. Stage 2 제출까지 잠긴 뒤 작성 근거·예상 위험·집계 결과를 공개한다.
10. adjudication 회의에서는 소수 의견과 공정성 우려를 삭제하지 않고 기록한다.

## 7. M04 사전 판정 기준안

아래는 데이터 확인 전 심리측정 책임자와 사용자가 승인해야 하는 작업 기준안이다. 절대적 학문 표준으로 주장하지 않는다.

### 7.1 문항별 기준

| 판정                 | 제안 조건                                                                                                                     |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `PASS_TO_COGNITIVE`  | 첫 facet target 일치가 `ceil(검토자 수 × .75)` 이상, 관련성 중앙값 3 이상, 명확성 중앙값 3 이상, 해결되지 않은 치명 위험 없음 |
| `COPY_REVISE`        | target은 대체로 맞지만 같은 표현 오해·응답 척도 문제·방향 혼동이 반복됨                                                       |
| `CONSTRUCT_REWRITE`  | target 일치 50~74% 또는 하나의 인접 성향으로 반복 매핑됨                                                                      |
| `HOLD_FOR_ACCESS`    | 접근·노출 부족 때문에 내용 판정이 불안정함                                                                                    |
| `EXCLUDE_OR_REBUILD` | target 일치 50% 미만, 능력·다른 성향이 주 표적으로 반복 판단됨, 치명적 낙인·공정성 문제를 문구 수정으로 해결하기 어려움       |

추가 규칙:

- 관련성 평균 하나로 통과시키지 않는다.
- 한 명의 구체적인 치명 위험 제기도 증거와 함께 adjudication한다.
- 같은 비target을 검토자의 30% 이상이 첫 선택하면 seam 위험으로 표시한다.
- agreement 계수는 보조로 사용하고 원자료 분포와 자유 서술을 함께 본다.
- set의 포괄성은 문항별 점수와 별도로 판단한다.

### 7.2 set 수준 기준

- 세 facet 각각 최소 네 개 상황 버킷이 M05 후보에 남는다.
- HIGH/LOW·주력/탐색·접근 방식이 한쪽으로 붕괴하지 않는다.
- AE↔CI, CI↔IE 우선 seam을 검증할 후보가 남는다.
- LOW 제한 표현과 부정문이 모두 한 facet에 몰리지 않는다.
- R 설명을 지능·실용성·현실성으로 확장하지 않는다.

## 8. M04 adjudication과 revision

### 8.1 독립 판단 후 회의

1. blind 결과를 item별로 집계한다.
2. target 일치·혼동 행렬·방향 일치·평정 분포·오염 flag를 본다.
3. 다수 의견보다 먼저 치명적 공정성·접근성 의견을 검토한다.
4. 문항 작성자는 의도를 설명할 수 있지만 결과를 재작성하지 못한다.
5. 각 문항을 `PASS / COPY_REVISE / CONSTRUCT_REWRITE / HOLD / EXCLUDE`로 결정한다.
6. 변경 이유와 반대 의견을 decision log에 남긴다.

### 8.2 revision 규칙

| 변경                     | revision 처리     | 다음 단계                            |
| ------------------------ | ----------------- | ------------------------------------ |
| 띄어쓰기·명백한 오탈자   | patch revision    | lint 후 기존 판정 유지 가능          |
| 쉬운 말·동사·범위 수정   | minor revision    | 새 문구 인지 인터뷰 필수             |
| 상황 라벨·표적 반응 변경 | major revision    | 전문가 mapping 재검토 후 인지 인터뷰 |
| target facet·방향 변경   | new candidate     | M03 후보 작성 단계부터 다시 시작     |
| 제외                     | retired candidate | 이유 보존, ID 재사용 금지            |

크게 수정한 문항을 같은 ID의 최신 문장으로 덮어쓰지 않는다. `revision_id`, 이전 문구, 변경 이유, 근거 reviewer, 승인자를 보존한다.

## 9. M04에서 M05로 넘어가는 게이트

다음을 모두 만족해야 한다.

- 필수 전문가 역할과 최소 검토자 수 충족
- 36문항 모두 독립 blind 평가 완료
- target key 누출·협의 응답·누락 검토 완료
- 모든 문항에 하나의 결정 상태와 근거 존재
- major revision 문항의 재검토 완료
- 해결되지 않은 치명적 공정성·낙인 위험 0건
- M05 후보가 facet별 최소 네 상황 버킷을 유지
- M05 세 form을 새 revision으로 재생성하고 자동 균형 검사 통과
- 개인정보·연구윤리·보상·보관 계획 승인

게이트를 충족하지 못하면 사용자 인터뷰 일정 때문에 문항을 임의로 통과시키지 않는다.

## 10. M05 사용자 모집 설계

### 10.1 목적표집 원칙

인지 인터뷰는 인구 비율을 추정하는 조사가 아니다. 서로 다른 해석·응답 오류·접근 문제를 발견하도록 의도적으로 다양한 참여자를 모집한다.

대상:

- 한국어 앱을 사용하는 만 20~39세
- 뉴앙 검사 문항 작성에 참여하지 않은 사람
- 동일 protocol의 이전 round에 참여하지 않은 사람
- 자신의 경험을 말로 설명하는 인터뷰에 동의한 사람

내부 직원 2~3명의 리허설은 가능하지만 외부 사용자 근거로 세지 않는다.

### 10.2 모집 다양성 매트릭스

첫 두 round 합계에서 최소 다음 범위를 모두 포함한다.

- 20대·30대 각각 전체의 약 40% 이상
- 특정 성별 범주가 전체의 70%를 넘지 않도록 노력
- 학생과 비학생 모두 포함
- 사무·서비스·생산·프리랜서·구직·돌봄 등 생활 맥락 분산
- 수도권·비수도권 또는 대도시·중소도시 생활환경 분산
- 고졸 이하·직업교육·대학 재학·대학 이상 등 교육 경험 분산
- 독서·영상·검색 사용 빈도가 낮은 사람과 높은 사람 모두 포함
- 시각·청각·읽기·디지털 이용 방식이 다른 사람 포함
- 성향 검사에 익숙한 사람과 거의 사용하지 않은 사람 모두 포함

의학적 진단이나 장애 공개를 참여 조건으로 요구하지 않는다. 접근 지원이 필요한지와 선호 정보 방식만 선택적으로 받는다.

### 10.3 round 계획

| 단계                | 외부 참여 목표 | form 배치                  | 목적                               |
| ------------------- | -------------: | -------------------------- | ---------------------------------- |
| Round 1 발견        |        12~15명 | form당 4~5명               | 큰 오해·오염·UI 문제 발견          |
| Round 2 수정 검증   |        12~15명 | form당 4~5명               | 수정 문구와 반복 문제 검증         |
| Round 3 표적 재검증 |         9~12명 | 문제가 남은 form·문항 중심 | major revision·접근·seam 종료 확인 |

예상 전체는 33~42명이다. 이는 모집 목표이지 통계적 대표성을 보장하는 고정 표본 수가 아니다. 문제가 반복되면 숫자를 채웠다는 이유로 종료하지 않는다.

각 유지 문항은 다음을 기본으로 한다.

- 최소 두 개 외부 round에서 관찰
- 합계 최소 8명의 응답 과정 근거
- 최종 major revision 뒤 새 참여자 최소 4명에게 재검증
- 관련 접근·노출 subgroup의 응답 근거 포함

숫자를 충족해도 같은 오해가 남으면 통과하지 않는다. 단 한 건이어도 치명적 낙인·접근·개인정보 문제라면 즉시 검토한다.

## 11. M05 인터뷰 방식

### 11.1 세션 길이와 순서

권장 55~65분:

| 구간            |    시간 | 내용                                   |
| --------------- | ------: | -------------------------------------- |
| 동의·안내       |     5분 | 목적·철회·녹음·보상·결과 미제공 안내   |
| 연습            |     3분 | 연구용 예시 1개로 응답·회상 방식 연습  |
| 자연 응답       |  8~12분 | 한 form 12문항을 실제 앱처럼 먼저 완료 |
| 문항 재생·probe | 30~35분 | 응답 과정과 우선 seam·위험 문항 확인   |
| 세트·UI debrief |  8~10분 | 중복·바람직성·접근·UI 전체 인상        |
| 종료            |     2분 | 질문·철회 안내·보상 절차               |

### 11.2 자연 응답을 먼저 하는 이유

첫 문항부터 면접자가 뜻을 묻거나 설명하면 이후 문항의 해석이 학습된다. 따라서 먼저 12문항을 스스로 완료하고, 이후 문항을 다시 보여주며 응답 과정을 재구성한다.

- 자연 응답 중 면접자는 단어 뜻을 설명하지 않는다.
- 참여자가 질문하면 `평소 이해한 방식대로 답한 뒤 나중에 함께 이야기한다`고 안내한다.
- 응답 시간·변경은 동의 범위 안에서 최소한으로 기록한다.
- 첫 응답 이후에도 개인 성향 코드·HIGH/LOW·target facet은 보여주지 않는다.

### 11.3 진행자 기본 문구

핵심 안내:

> 참여자를 평가하는 자리가 아니라 문장이 의도대로 이해되는지 확인하는 자리입니다. 이해하기 어렵거나 어색한 부분을 찾는 것이 연구에 가장 도움이 됩니다. 답하고 싶지 않은 질문은 건너뛸 수 있고 언제든 중단할 수 있습니다.

진행자는 방어하거나 정답을 설명하지 않는다. `제가 쓴 문장이 아니므로 편하게 비판해 달라`는 중립적 태도를 유지한다.

## 12. M05 공통 probe

### 12.1 이해

- 이 문장을 본인 말로 다시 설명해 주세요.
- 상황 라벨은 언제의 일이라고 이해했나요?
- 질문은 그 상황에서 무엇을 묻는다고 느꼈나요?
- 낯설거나 범위가 불분명한 단어가 있었나요?

### 12.2 회상

- 어떤 실제 경험을 떠올렸나요?
- 한 번의 경험인가요, 최근 여러 경험을 종합했나요?
- 최근 6개월이라는 기준을 실제로 사용했나요?
- 비슷한 상황이 거의 없었다면 어떤 답을 골랐을 것 같나요?

### 12.3 판단

- 왜 그 응답을 선택했나요?
- 관심이 갔기 때문인가요, 잘했거나 못했기 때문인가요?
- 시간·일·학업·기기·검색 환경 때문에 반응이 달라졌나요?
- 상황이 달라지면 답이 어떻게 달라지나요?

### 12.4 응답 선택

- 1점과 3점, 3점과 5점은 어떻게 다르다고 생각했나요?
- 선택하려던 답과 버튼 문구가 잘 맞았나요?
- 부정문에서 `거의 그렇지 않아요`를 고르면 어떤 뜻이 되나요?
- `판단하기 어려워요`가 필요했나요? 선택한다면 이유는 무엇인가요?

### 12.5 바람직성

- 어느 방향이 더 좋은 사람처럼 보였나요?
- 감수성·창의성·지능·현실성·효율성을 판단하며 답했나요?
- 결과가 공개된다면 답을 바꾸고 싶어질 것 같나요?

## 13. facet별 probe

### OE-AE 미적 경험

- 아름다움·분위기에 관심이 간 것인가요, 작은 차이를 정확히 알아챘다는 뜻인가요?
- 감정이 강하게 올라온 것과 인상에 관심이 머문 것을 구분했나요?
- 시각·청각·공간 중 어떤 경험으로 답했나요?
- 예술 취향이나 지식이 있어야 높은 답이라고 느꼈나요?

### OE-CI 상상 확장

- 실제 이미지가 보였나요, 말이나 이야기 흐름으로 생각했나요?
- 새 장면을 만든 것인가요, 실제 내용을 기억하거나 예측한 것인가요?
- 창의적인 결과를 잘 만들 수 있는지를 묻는다고 느꼈나요?
- 걱정·계획·주의 산만 때문에 장면이 이어진 것은 아닌가요?

### OE-IE 지적 탐구

- 궁금해진 것과 실제로 찾아본 것을 구분했나요?
- 필요해서 찾았나요, 이유·배경 자체가 궁금해서 더 봤나요?
- 내용을 이해하거나 정답을 찾는 능력을 묻는다고 느꼈나요?
- 다른 설명에 동의하거나 사람을 존중해야 높은 답이라고 느꼈나요?

## 14. UI·응답 형식 검증

### 14.1 모바일 UI

- 상황 라벨이 장식이 아니라 응답 조건으로 인식되는가?
- 상황 라벨과 질문이 중복되지 않고 자연스럽게 연결되는가?
- 질문을 읽은 뒤 상황 조건을 잊지 않는가?
- 이어진 진행 그래프가 현재 위치를 충분히 알려주는가?
- 하단 응답 버튼과 다음 동작을 엄지로 편하게 선택할 수 있는가?
- `판단하기 어려워요`가 숨은 기능처럼 보이지 않는가?
- 선택 변경·뒤로 가기·종료가 의도대로 이해되는가?
- 결과나 캐릭터 반응이 특정 답을 유도하지 않는가?

Round 1은 의미 검증용 연구 프로토타입, Round 2 이후는 S03 모바일 UI와 같은 구조의 고충실도 프로토타입을 사용한다. 프로덕션 앱과 운영 DB에 연결하지 않는다.

### 14.2 회상 기간과 척도 A/B

문항 의미가 안정되기 전에 회상 기간·척도 문구까지 동시에 바꾸지 않는다.

1. Round 1은 M02 기준 6개월·빈도 척도를 사용한다.
2. Round 2 이후 새 참여자를 form 안에서 3개월/6개월 조건으로 분리할 수 있다.
3. 같은 참여자에게 두 회상 기간을 연속으로 보여 학습시키지 않는다.
4. 회상 구체성·부담·판단 어려움 이유를 질적으로 비교한다.
5. 응답 형식이 바뀌면 `response_format_id`를 새로 발행한다.

정량적 우열은 M05 소표본으로 확정하지 않고 M06 파일럿에서 확인한다.

## 15. issue code와 심각도

### 15.1 issue taxonomy

| code                  | 의미                                          |
| --------------------- | --------------------------------------------- |
| `CONTEXT_MISREAD`     | 상황 라벨을 다른 조건으로 이해                |
| `PROMPT_MISREAD`      | 질문 반응을 다른 의미로 이해                  |
| `DOUBLE_RESPONSE`     | 두 가지 반응을 동시에 묻는다고 느낌           |

| `ABILITY_INFERENCE`   | 감각·창의·지능·기억 능력으로 답함             |

| `NEGATION_DIRECTION`  | 부정문과 척도 방향을 반대로 이해              |

| `PRIVACY_SENSITIVITY` | 불필요하게 민감하거나 공개가 꺼려짐           |

### 15.2 severity

| 등급          | 의미                                            | 예시 처리                      |
| ------------- | ----------------------------------------------- | ------------------------------ |
| `S0_NOTE`     | 의미를 바꾸지 않는 선호·표현 의견               | 기록 후 유지 가능              |
| `S1_MINOR`    | 일부 망설임이나 자연스러움 문제                 | A/B 또는 minor revision        |
| `S2_MATERIAL` | 응답 근거·facet·방향이 달라지는 문제            | 수정·재검증 전 통과 금지       |
| `S3_CRITICAL` | 낙인·권리·접근 차별·개인정보·회복 어려운 오측정 | 즉시 보류, 책임자 adjudication |

같은 S1/S2가 두 명 이상에게 반복되면 dominant pattern으로 표시한다. 한 명에게만 나타나도 타당성을 직접 깨뜨리거나 S3이면 discovery issue로 별도 처리한다.

## 16. M05 판정과 stop rule

| 판정                 | 조건                                                                           |
| -------------------- | ------------------------------------------------------------------------------ |
| `KEEP_FOR_PILOT`     | 의도한 상황·반응으로 이해, 반복 S2 없음, 최종 문구 재검증, 접근·방향 문제 해결 |
| `COPY_REVISE_RETEST` | target은 유지되나 표현·범위·척도 매핑 문제 반복                                |
| `CONSTRUCT_REWRITE`  | 다른 facet·능력으로 반복 해석                                                  |
| `HOLD_FOR_SUBGROUP`  | 특정 접근·노출 집단 근거 부족                                                  |
| `EXCLUDE`            | 수정 후에도 target 오해·낙인·접근 차별이 반복되거나 상황이 지나치게 제한적     |

종료 조건:

- 최소 두 개 외부 round 완료
- 유지 문항이 최종 문구로 충분한 참여자·subgroup 근거 확보
- 모든 major revision 재검증
- 반복 S2·미해결 S3 0건
- 세 facet seam 오해가 사전 기준 아래
- 부정문 방향 혼동이 해결되거나 해당 문항 제외
- UI에서 상황 라벨·응답 척도·판단 어려움의 반복 장애 0건
- M06에 보낼 후보가 facet별 여러 상황 버킷과 HIGH/LOW를 유지

issue가 새로 계속 발견되면 3회차가 끝나도 종료하지 않는다. 반대로 문제 포화가 보여도 final revision을 새 참여자에게 확인하기 전 통과시키지 않는다.

## 17. M05 분석 절차

CDC의 다단계 분석 원칙을 뉴앙에 다음처럼 적용한다.

1. 세션 중: 관찰·후속 probe 기록
2. 세션 직후: interviewer와 note-taker가 요약·severity 초안 작성
3. round 중: item별로 참여자 간 공통점·차이 비교
4. subgroup 비교: 연령·교육·매체·접근 방식에 따른 해석 차이 확인
5. round 종료: 문항 성능·수정·제외 결론과 다음 round 계획

정량 빈도처럼 보이는 issue count는 문제의 보편적 발생률이 아니다. 목적표집 질적 근거로만 해석한다.

## 18. 개인정보·안전 계약

- 연구 연락처와 인터뷰 데이터는 분리 보관한다.
- 앱 계정 ID·실명·성향 코드와 연구 participant ID를 연결하지 않는다.
- 필요한 최소 인구통계만 범주형으로 수집한다.
- 녹음·화면 기록은 각각 별도 동의를 받는다.
- 녹음을 거부해도 상세 메모 방식으로 참여할 수 있는지 모집 전에 정한다.
- 자유 서술의 제3자 이름·회사·학교·정확한 장소는 기록 단계에서 제거한다.
- 개인 성향 결과·진단·상담을 제공한다고 안내하지 않는다.
- 질문 건너뛰기·중단·철회가 보상 불이익으로 이어지지 않게 한다.
- 원본 보관 기간이 끝나면 검증 가능한 방식으로 삭제하고 로그를 남긴다.
- 인용문은 별도 동의 범위 안에서만 비식별 형태로 사용한다.
- 외부 업체를 쓰면 재위탁·국외 이전·접근 권한·삭제 증빙을 계약으로 제한한다.

## 23. 사용자 승인 기록

2026-07-17 사용자 승인으로 다음 다섯 항목을 확정했다. 프로토콜을 설계 기준으로 고정하고 실제 실행 자료 제작 단계로 이동한다.

1. M04는 6~8명의 독립 전문가와 필수 역할 범위를 기준으로 준비한다.
2. 전문가 PASS 작업 기준을 target mapping 75% 이상·관련성/명확성 중앙값 3 이상으로 사전 검토한다.
3. M05는 3개 균형 form, 2~~3개 round, 외부 참여자 약 33~~42명의 목적표집 계획으로 준비한다.
4. final major revision은 새 참여자에게 다시 확인하고, 숫자를 채워도 반복 오해가 남으면 종료하지 않는다.
5. 개인정보·연구윤리·보상·보관 기간이 승인되기 전에는 실제 모집과 기록을 시작하지 않는다.

승인 후 다음 산출물은 `M04 전문가 blind review 실행 키트`다. 이 승인만으로 모집·연락·외부 전송은 하지 않는다.
