## 0. 감사 결론

SM의 세 후보 facet은 정의 수준에서 구분할 수 있다.

```text
SM-EP 시작하고 이어가기
일 자체를 언제 시작하고, 끊긴 뒤 돌아오며, 어느 정도 계속하는가?

SM-OS 정리하고 계획하기
물건·시간·절차에 반복해서 쓸 위치·순서·때를 만드는가?

SM-RL 약속한 일 챙기기
다른 사람과 연결된 맡은 일을 언제 확인·전달·재조율하는가?
```

36문항의 수량·방향·후보 상태는 균형을 이룬다. 그러나 `더 자세하므로 3-facet`을 바로 채택하면 안 된다.

핵심 이유:

1. SM-RL은 SM-EP를 관계 상황에서 다시 물은 것일 수 있다.
2. RL의 HIGH는 좋은 사람, LOW는 무책임한 사람처럼 보여 자기보고를 왜곡할 위험이 크다.
3. RL을 K/M에 합산하면 M의 승인 의미인 `상황에 따라 달라짐`이 `약속을 못 지킴`으로 바뀔 수 있다.
4. RL은 SE-AI의 먼저 알림, RO-RN의 상대 선택 존중, 관계 규범과도 교차 적재될 수 있다.
5. 반대로 RL이 독립적이고 행동·관계 결과에 추가 설명력을 보이면 뉴앙의 관계 서비스에 유용한 상세 신호가 될 수 있다.

따라서 현재 권장안은 아래와 같다.

> **2-facet을 공개 K/M의 의미 기준선으로 유지하고, 세 후보 facet을 모두 파일럿한다. SM-RL은 독립성·추가 설명력·비낙인성이 함께 확인될 때만 K/M 포함 여부를 다시 승인한다. 독립적이지만 도덕 낙인 위험이 남으면 K/M과 분리한 본인 전용 상세 신호로 제공한다.**

현재 판정은 `PASS_FOR_EXPERT_AND_COGNITIVE_REVIEW_WITH_RL_PUBLIC_CODE_HOLD`다. 빠른·정밀 문항 선정과 K/M 발급은 계속 `NOT_READY`다.

## 2. 2-facet·3-facet·분리 상세 세 가지 경쟁 모형

| 모형         | 구조                          | 의미                             | 채택 시 제품 영향                            |
| ------------ | ----------------------------- | -------------------------------- | -------------------------------------------- |
| `SM-M2`      | EP + OS                       | 실행 지속과 생활 구조의 2-facet  | K/M이 현재 승인 의미를 가장 직접 유지        |
| `SM-M3`      | EP + OS + RL                  | 실행·구조·맡은 일 이행의 3-facet | 더 넓은 설명, 검사 길이·도덕 낙인 위험 증가  |
| `SM-M2+RL-D` | EP + OS → K/M, RL → 별도 상세 | 주 코드와 관계 이행 신호 분리    | 대표 코드는 단순하게 유지하며 상세 정보 제공 |

`SM-M2+RL-D`는 통계 모형만의 문제가 아니다. RL이 독립 요인으로 확인되더라도 주 코드 의미와 공개 적합성이 다르면 제품 표현층에서 분리하는 대안이다.

### 2.1 무엇이 확인돼야 하는가

`SM-M3`가 문항 수가 많아 적합도가 조금 좋아지는 것만으로 채택하지 않는다. 아래를 함께 확인한다.

- RL 문항이 EP·OS가 아닌 하나의 반복 가능한 요인으로 모이는가?
- RL이 일반 실행·정리 점수를 넘는 추가 정보를 제공하는가?
- 그 정보가 자기보고의 좋은 사람 효과만 반영하지 않는가?
- 관계 맥락·성별·연령·직업·돌봄 부담에 따라 문항 기능이 크게 달라지지 않는가?
- LOW를 비난하지 않고 사실대로 설명할 수 있는가?
- 빠른 검사에서도 세 facet에 최소 근거를 확보할 수 있는가?

## 5. HIGH·LOW와 K/M 의미 대칭성

### 5.1 EP와 OS

| 방향 | 관찰 가능한 공통 의미                            | 금지 해석                     |
| ---- | ------------------------------------------------ | ----------------------------- |
| HIGH | 시작·지속 또는 구조 사용이 비교적 일관됨         | 우수함, 유능함, J형           |
| LOW  | 관심·기운·때·현재 단서에 따라 실행·구조가 달라짐 | 게으름, 지저분함, P형, 유연함 |

EP·OS는 완전히 가치 중립적이지 않지만, 현재 K `꾸준히 이어감`과 M `상황에 따라 달라짐`의 언어로 비교적 직접 설명할 수 있다.

### 5.2 RL이 만드는 의미 변화

RL을 합산하면 다음 변환이 발생할 수 있다.

```text
현재 M
실행 시점과 생활 구조가 상황의 영향을 더 받음

RL 포함 후 사용자가 들을 수 있는 M
다른 사람이 독촉해야 맡은 일을 처리함
```

두 문장은 도덕적 무게가 다르다. 통계적으로 같은 상위 요인에 묶인다는 이유만으로 이 의미 차이를 무시하지 않는다.

### 5.3 RL 주 코드 포함을 위한 의미 게이트

- LOW 응답자도 결과 문구를 사실 설명으로 받아들이고 인격 평가로 느끼지 않는다.
- M 글자 설명을 들은 사용자가 `무책임`, `약속을 안 지킴`을 첫 연상으로 말하지 않는다.
- HIGH 사용자를 `믿을 만한 좋은 사람`으로 과장하지 않는다.
- RL이 낮다는 이유로 관계 비교에서 경고·궁합 불리함을 부여하지 않는다.
- 행동 개선이 필요한 상황과 성향 정체성을 분리해 안내할 수 있다.

이 게이트를 통과하지 못하면 RL은 K/M에서 제외한다.

## 6. 상황·맥락 균형

| facet | 주요 상황                                 | 현재 장점                    | 남은 위험                        |
| ----- | ----------------------------------------- | ---------------------------- | -------------------------------- |
| EP    | 시작·중단·낮은 보상·여러 날·마무리·재개   | 학생·직장인 전용 표현이 적음 | 피로·업무량·흥미 영향            |
| OS    | 물건·자료·할 일·반복 절차·일정·확인       | 특정 앱·도구를 요구하지 않음 | 공유환경·일정 통제권·디지털 경험 |
| RL    | 공동 몫·약속·전달·재조율·재확인·작은 부탁 | 관계 서비스에 직접 관련      | 권력·관계 친밀도·도덕 평가       |

필수 공변량·결측 이유:

- 건강·피로·기분·업무량·돌봄 부담
- 자기 일정과 작업 방식에 대한 통제권
- 문항 상황의 경험 빈도
- 공동 활동의 관계 유형과 권력 차이
- 물리·디지털 접근 방식
- `좋아 보이는 답` 평정과 도덕 판단 평정

이 값은 성향 점수를 보정하기 위한 임의 감점·가점이 아니라 문항이 다른 집단에서 같은 기능을 하는지 확인하는 연구 정보다.

## 7. 문장 형식·방법효과 감사

### 7.1 확인된 균형

- 세 facet 모두 HIGH 6개·LOW 6개다.
- `안 한다`, `못 한다`, `잊어버린다` 같은 직접 부정문을 사용하지 않았다.
- LOW도 현재 단서·기회·요청·기억에 따라 행동하는 직접 반응으로 작성했다.
- 같은 상황 쌍은 문법 구조가 유사하지만 한 form에는 함께 넣지 않는다.

### 7.2 남은 비대칭

| 위험                 | 예                                  | 처리                         |
| -------------------- | ----------------------------------- | ---------------------------- |
| HIGH의 자기관리 언어 | 정해둔다, 확인한다, 이어간다        | 좋은 사람 평정 별도 수집     |
| LOW의 외부 계기 언어 | 물어볼 때, 요청한 뒤, 다시 말해주면 | 비난·수치심·응답 회피 확인   |
| HIGH의 사전 시점     | 전에, 미리                          | 불안·완벽주의 교차 적재 확인 |
| LOW의 상태 의존      | 관심·기운·손이 가는 날              | 우울·번아웃·현재 상태 분리   |

HIGH/LOW 숫자만 맞춘 것은 심리적 대칭성의 증거가 아니다.

## 8. 인지 인터뷰용 3개 균형 form

세 form은 3-facet 채택용 점수를 계산하지 않는다. 문장 이해·seam·도덕 인상·응답 과정을 확인한다.

| Form | SM-EP 4개       | SM-OS 4개       | SM-RL 4개       |
| ---- | --------------- | --------------- | --------------- |
| A    | C01·C04·C09·C12 | C03·C06·C08·C11 | C02·C05·C07·C10 |
| B    | C03·C06·C08·C11 | C02·C05·C07·C10 | C01·C04·C09·C12 |
| C    | C02·C05·C07·C10 | C01·C04·C09·C12 | C03·C06·C08·C11 |

검증된 수량:

| 검사                       | Form A | Form B | Form C |
| -------------------------- | -----: | -----: | -----: |
| 전체 문항                  |     12 |     12 |     12 |
| facet별 문항               |  4·4·4 |  4·4·4 |  4·4·4 |
| HIGH / LOW                 |  6 / 6 |  6 / 6 |  6 / 6 |
| 주력 / 탐색                |  8 / 4 |  8 / 4 |  8 / 4 |
| 같은 HIGH/LOW 쌍 동시 포함 |      0 |      0 |      0 |

2-facet 인지 인터뷰만 먼저 실행해야 할 경우 RL 열을 제외하지 않고, 별도의 비교 표본에서 아래 8문항 form을 사용한다.

| 2-facet Form | SM-EP | SM-OS | 전체 / HIGH·LOW |
| ------------ | ----- | ----- | --------------: |
| 2A           | EP-α  | OS-β  |         8 / 4·4 |
| 2B           | EP-β  | OS-γ  |         8 / 4·4 |
| 2C           | EP-γ  | OS-α  |         8 / 4·4 |

2-facet과 3-facet form의 단순 완료 시간 차이를 곧바로 모형 품질 차이로 해석하지 않는다. 3-facet은 내용이 추가되므로 더 길어지는 것이 당연하다.

## 9. 전문가·인지 인터뷰 게이트

### 9.1 전문가 blind mapping

각 문항에 대해 다음을 받는다.

- 첫 번째 목표 facet, 두 번째 후보 facet, 어느 facet에도 맞지 않음
- HIGH/LOW 방향
- 행동 하나만 묻는지
- 능력·임상 상태·환경 제약·규범·사회적 바람직성 오염
- 수정 없이 유지 / 수정 후 유지 / 제외
- 수정 제안이 구성개념을 바꾸는지

통과 수치는 데이터를 보기 전에 심리측정 책임자가 사전 등록한다.

### 9.2 사용자 인지 인터뷰

각 문항에서 다음을 기록한다.

- 실제 떠올린 상황과 행동
- 질문을 자기 말로 다시 설명한 내용
- 답의 근거가 성향·능력·현재 상태·환경·좋은 사람 이미지 중 무엇인지
- HIGH와 LOW 중 더 바람직해 보이는 방향
- LOW를 골랐을 때 비난·수치·방어 느낌
- `이 상황은 답하기 어려워요`의 이유

세트 종료 질문:

1. 시작·지속, 정리·계획, 약속 챙기기가 서로 다른 성향으로 느껴졌는가?
2. RL 질문이 EP 질문을 사람 관계로 바꿔 반복한 것처럼 느껴졌는가?
3. M을 `상황에 따라 달라지는 편`으로 설명했을 때 RL LOW와 자연스럽게 연결되는가?
4. RL 결과를 본인만 보는 것과 친구·연인에게 공유하는 것 중 어디까지 괜찮은가?
5. 어느 문항에서 사실대로 답하기보다 좋은 사람처럼 답하고 싶었는가?

## 10. 정량 파일럿 모형 비교

인지 검토를 통과한 후보로 다음을 비교한다.

1. `M2-CORRELATED`: EP와 OS 두 상관 요인
2. `M2-HIERARCHICAL`: EP·OS 위 상위 SM
3. `M3-CORRELATED`: EP·OS·RL 세 상관 요인
4. `M3-HIERARCHICAL`: 세 facet 위 상위 SM
5. `M2+RL-D`: EP·OS 상위 SM과 분리된 RL 상세 신호
6. 방향 문구·사전 시점·사회적 바람직성 방법 요인을 포함한 민감도 모형

비교 근거:

- 탐색·확인 표본을 분리한 요인구조 재현
- 교차 적재와 요인 간 구분 가능성
- 모형 적합도·정보 기준·교차검증 성능
- facet 및 상위 점수의 omega와 측정오차
- 문항정보·문턱값·국소 의존성
- 연령·성별·관계 맥락·생활 제약별 측정 불변성 및 DIF
- 사회적 바람직성 통제 전후 RL 구조 안정성
- holdout 행동·자기설명 적합성에 대한 RL의 증분 설명력
- 결과 문구 공감도와 낙인·공개 수용도

CFI·RMSEA 같은 하나의 수치나 Cronbach alpha 하나로 결정하지 않는다. 수치 임계값, 표본 크기, 제외 규칙은 데이터를 보기 전에 M04 연구 계획에 사전 등록한다.

## 11. 빠른·정밀 검사 영향

### 11.1 2-facet 채택 시

- 빠른 K/M은 EP와 OS 양쪽에서 최소 근거를 가져야 한다.
- 한 facet만 응답한 경우 K/M을 확정하지 않는다.
- 정밀 결과는 EP·OS를 각각 보여주고 분리 프로필을 평균으로 숨기지 않는다.

### 11.2 3-facet 채택 시

- 빠른 검사 길이를 유지하려고 RL을 1문항만 넣고 세부 결과를 확정하지 않는다.
- RL까지 K/M 의미에 포함한다면 빠른 검사에도 세 facet의 근거가 필요하다.
- 충분한 근거를 담을 수 없으면 빠른 K/M은 `예비 방향`으로 낮추거나 정밀 후 확정한다.
- 정밀 결과는 EP·OS·RL 세 점수와 불확실성을 따로 보존한다.

### 11.3 분리 상세 채택 시

- K/M은 EP·OS만으로 설명한다.
- RL은 정밀 검사 뒤 본인 전용 상세 신호로 보여준다.
- 공개 프로필·피드·공유·비교에는 기본 제외하고 사용자가 명시적으로 선택해도 별도 정책 승인을 요구한다.
- RL이 낮다는 사실을 관계 위험 점수나 궁합 불이익으로 자동 변환하지 않는다.

정확한 빠른·정밀 문항 수는 M04·M05 이후 정보량·응답 부담·세부 결과 최소 근거량으로 결정한다. 기존 빠른 코어 4개 SM 문항을 자동 승계하지 않는다.

## 14. 사용자 승인 기록

2026-07-18 KST 사용자 승인으로 다음 세 문장을 확정했다.
