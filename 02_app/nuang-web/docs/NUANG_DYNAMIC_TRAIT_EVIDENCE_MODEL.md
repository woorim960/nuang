## 핵심 원칙

대표 성향 코드는 고정된 평생 유형이 아니라, 현재까지의 검사 증거로 계산한 `현재 가장 가까운 대표 성향`이다.

정밀 코어는 대표 성향의 기준점을 만든다. 이후 무료 주제 검사, 승인된 상세검사, 반복 검사에서 얻은 관찰 증거가 쌓이면 성향 점수가 더 선명해지거나, 반대 방향의 증거가 충분할 때 대표 코드가 바뀔 수 있다.

결제 여부는 성향 정확도 가중치가 아니다. 무료 검사와 유료 검사는 같은 기준을 사용하며, 가중치는 문항 수, 구성개념 직접성, 응답 품질, 승인 상태, 최근성, 반복 할인으로 결정한다.

## 계산 개념

```text
현재 성향 점수
=
Σ(검사 관찰 점수 × 유효 가중치)
÷
Σ(유효 가중치)
```

유효 가중치:

```text
측정량
× 구성개념 직접성
× 응답 품질
× 승인 상태
× 최근성
× 반복 할인
```

결제 여부:

```text
항상 1.0
```

## 검사별 반영 수준

| 검사 분류 | 기본 반영 | 대표 코드 영향 | 설명 |
|---|---:|---|---|
| 빠른 코어 | 낮음 | 예비 코드만 | 첫 방향을 보여주되 정밀 코어 전까지 임시 기준이다. |
| 정밀 코어 | 높음 | 기준점 | 뉴앙 코드, 코드 지도, 세부 신호, 공개 프로필, 1:1 비교의 기준이다. |
| 무료 주제 검사 | 중간~낮음 | 가능 | 승인된 축·세부 성향 매핑이 있을 때 성향 보강 증거로 반영한다. |
| 별난 성향 연구소 | 없음~낮음 | 기본 불가 | 재미형 검사는 기본적으로 코드에 반영하지 않고, 별도 승인된 일부만 낮은 가중치로 반영한다. |
| 상세검사 | 중간~높음 | 가능 | 결제 여부가 아니라 문항 수와 구성개념 직접성 때문에 가중치가 달라질 수 있다. |
| 도움 연결 | 없음 | 불가 | 민감/위기 주제는 점수화하거나 대표 코드에 반영하지 않는다. |
| 그룹 검사 | 미정 | 미정 | 추후 별도 기획과 승인 후 결정한다. |

## 대표 코드 변경 조건

대표 성향 코드는 다음 조건을 만족할 때만 바뀔 수 있다.

- 반대 방향의 증거가 한 번이 아니라 여러 검사에서 반복된다.
- 변경 대상 축의 유효 가중치 합이 최소 기준을 넘는다.
- 응답 품질이 낮은 검사 결과는 변경 근거에서 제외하거나 낮은 가중치로 반영한다.
- 점수 변화가 작은 경우에는 코드 변경 대신 `두 방향이 비슷한 자리`로 표시한다.
- 45~55 구간은 가운데 근처로 취급하고 가까운 대안 코드를 함께 보여준다.
- 작은 변화로 코드가 계속 왕복하지 않도록 최소 증거 조건과 변경 쿨다운을 둔다.
- 현재 결과와 과거 결과를 모두 보존한다.

v0.1 코드 변경 조건:

- 변경 대상 축의 유효 가중치 합이 1.2 이상이어야 한다.
- 이전 방향과 반대되는 유효 증거가 최소 2개 이상이어야 한다.
- 새 점수가 45~55 가운데 근처에 있으면 코드 변경 대신 두 방향이 비슷하게 나타날 수 있다고 안내한다.
- 3점 미만 변화는 의미 있는 변화로 강조하지 않는다.
- 이전 리포트는 삭제하지 않고 최신 대표 성향만 갱신한다.

## 사용자 안내

변화는 사용자가 이해할 수 있는 말로만 안내한다.

좋은 안내:

- `최근 검사에서도 같은 방향이 반복 확인되어 더 선명해졌어요.`
- `이 자리는 두 방향이 비슷하게 함께 나타날 수 있어요.`
- `최근 여러 검사에서 반대 방향이 반복 확인되어 현재 가장 가까운 코드가 달라졌어요.`

피해야 할 안내:

- `진짜 유형이 바뀌었습니다.`
- `이전 결과가 틀렸습니다.`
- `유료 검사를 해야 정확해집니다.`
- `당신은 이제 다른 사람입니다.`

## 화면 반영

성향지도:

- 현재 뉴앙 코드와 코드 지도를 보여준다.
- 무료 검사로 더 선명해진 코드 자리는 `최근 더 선명해진 부분`으로 표시한다.
- 두 방향이 비슷한 자리는 양쪽 특성이 모두 나타날 수 있음을 설명한다.

결과 리포트:

- 현재 알고리즘 버전과 마지막 업데이트일을 제공한다.
- 의미 있는 변화만 안내한다.
- 반올림 전 소수점만 달라진 변화나 3점 미만 변화는 강조하지 않는다.

마이:

- 대표 성향 영역은 최신 현재 결과를 보여준다.
- 이전 대표 코드와 리포트는 `내 리포트`에서 다시 볼 수 있게 보존한다.

피드:

- 무료 검사 결과와 성향 보강 신호를 추천 품질 개선에 활용할 수 있다.
- 직접 응답과 원점수는 피드에 공개하지 않는다.

1:1 비교:

- 기본적으로 현재 공개된 대표 성향과 공식 축만 사용한다.
- 무료 주제 검사 신호를 비교에 포함하려면 별도 공개 범위와 사용자 승인 후 설계한다.

## 구현 승인 게이트

이 문서는 제품 기준이다. v0.1 순수 채점 엔진은 승인 후 구현했다. 실제 앱 화면, DB 저장, 공개 프로필, 1:1 비교에 연결하기 전에는 아래 항목을 다시 사용자에게 승인받는다.

1. 검사별 기본 가중치 표
2. 축·세부 성향 매핑 방식
3. 코드 변경 최소 증거 기준
4. 두 방향이 비슷한 구간과 변경 쿨다운
5. 과거 리포트 보존 방식
6. 공개 프로필과 1:1 비교에 최신 코드가 반영되는 시점
7. 사용자에게 변화 안내를 보여주는 화면과 문구
