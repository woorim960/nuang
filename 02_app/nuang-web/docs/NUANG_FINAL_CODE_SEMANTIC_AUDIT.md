> 후속 RO 24문항 감사와 2026-07-18 사용자 승인으로 G/A를 RO-EC의 가치중립적인 관계 주의 방향으로 재기준화했다. RO-RN은 대표 코드에 합산하지 않는 본인 전용 상세 신호다. 이 v2 계약은 이전 `해결·결과 ↔ 마음·선택` 2-facet 합산안을 대체한다.

## 0. 감사 결론

현재 5글자 구조는 G/A 재기준화를 포함해 `설계 의미 계약` 수준에서 정합성이 있다.

```text
E/I · R/N · G/A · K/M · C/Q
사람 → 생각 → 관계 → 일상 → 마음
```

통과한 내용:

- 다섯 자리가 서로 다른 생활 질문을 담당한다.
- 열 개 글자가 중복되지 않는다.
- MBTI와 비슷한 위치 문법은 활용하되 같은 검사를 가장하지 않는다.
- 각 글자는 능력·도덕성·진단이 아니라 반복 경향을 요약한다.
- 코드보다 연속 점수·세부 성향·불확실성을 원본으로 둔다.
- 세부 성향이 갈리는 프로필과 실제 경계 상태를 구분한다.

아직 통과하지 않은 내용:

- 한국어 문항의 내용타당도·요인구조·신뢰도·수렴·변별타당도
- 점수 경계와 신뢰구간 계산 방식
- SM 2-facet·3-facet 최종 선택
- 빠른·정밀 검사 문항 수와 정보량
- 연령·성별·직업·문화 집단별 측정동일성과 문항편향
- 32개 조합의 이름·캐릭터·추천 정확성

따라서 현재 허용되는 표현은 `연구 근거를 바탕으로 설계하고 검증 중인 코드`다. `과학적으로 정확성이 입증된 검사` 또는 `신뢰도 검증 완료`라고 말하지 않는다.

## 1. 코드의 역할

5글자 코드는 측정 원본이 아니라 기억·공유를 위한 표현층이다.

```text
문항 응답
→ 세부 성향별 관찰 증거
→ 연속 점수와 측정오차
→ 상위 영역 점수와 상태
→ 기억용 5글자 코드
→ 검증된 범위 안의 리포트 문구
```

사용자의 성향을 다섯 개 상자로 고정하지 않는다. 글자가 같아도 세부 성향의 조합과 선명도는 다를 수 있고, 추가 검사를 통해 점수와 설명이 갱신될 수 있다.

## 3. 자리별 의미 계약

### 3.1 E/I — 사람 사이 에너지

측정한다:

- 사람과 함께할 때 활력이 올라오고 교류에 관여하는 경향
- 필요한 의견·요청·선택지를 먼저 표현하는 경향

측정하지 않는다:

- 친구 수, 인기, 대화 기술, 리더십 능력
- 사람을 좋아하거나 싫어하는 정도
- 수줍음·사회불안·우울 진단

고객 해석:

- E 쪽: 교류에서 활력이 올라오고 먼저 표현하는 모습이 상대적으로 자주 나타날 수 있음
- I 쪽: 혼자 정리할 때 회복되고 표현 전에 상황·확신·차례를 더 살피는 모습이 상대적으로 자주 나타날 수 있음

주의:

- 교류 활력과 주도적 표현이 갈리면 `두 반응이 다르게 나타나요`라고 설명한다.
- E를 관계 능력, I를 소통 부족으로 해석하지 않는다.

### 3.2 R/N — 생각과 탐색

측정한다:

- 분위기와 아름다움에 관심을 두는 경향
- 장면·이야기·가능성을 머릿속에서 펼치는 경향
- 새로운 개념·원리·관점을 알아보는 경향

측정하지 않는다:

- 현실 판단, 실용 능력, 지능, 창의적 성과
- 직관의 정확도, 예술적 재능, 미래 예측력

고객 해석:

- R: 익숙하고 구체적으로 접할 수 있는 대상에 관심이 더 머무는 편
- N: 분위기·상상·새로운 원리와 관점에 관심이 더 머무는 편

주의:

- N은 MBTI N과 방향이 유사하지만 같은 검사 결과가 아니다.
- R을 `현실형`, N을 `창의형` 또는 `몽상가`로 이름 붙이지 않는다.
- 세 facet이 갈리면 세 점수를 따로 설명한다.

### 3.3 G/A — 관계에서 먼저 보는 것

측정한다:

- 마음과 해결이 함께 있는 관계 상황에서 원인·결과·해결할 부분이 먼저 보이는지
- 같은 상황에서 상대의 감정 상태와 안녕이 먼저 보이는지
- 처음 드는 생각과 실제 나타나는 반응의 순서가 반복해서 다르게 나타나는지

측정하지 않는다:

- 착함, 도덕성, 공감 정확도, 상담 능력
- 논리력, 목표 달성 능력, 솔직함, 독립성
- 타인의 말을 모두 들어주는 자기희생
- 상대의 거절·선택권을 존중하는 정도
- 압박·비하를 억제하는 정도

고객 해석:

- G: 원인·결과·해결할 부분이 먼저 눈에 들어오는 편
- A: 상대가 어떤 마음인지 먼저 눈에 들어오는 편

주의:

- G/A는 MBTI T/F와 같은 검사가 아니다.
- G를 낮은 존중, A를 높은 존중으로 해석하지 않는다.
- RO-RN 선택·존엄 존중은 G/A에 합산하지 않고 본인 전용 상세 신호로 분리한다.

### 3.4 K/M — 일상을 꾸리는 방식

측정한다:

- 해야 할 일을 시작하고 다시 돌아와 이어가는 경향
- 물건·시간·절차에 만든 구조를 유지하는 경향
- 채택될 경우 맡기로 한 일을 필요한 때 처리하는 경향

측정하지 않는다:

- 실제 성과, 지능, 업무 능력, 도덕적 책임감
- 창의성, 즉흥성, 변화 적응 능력
- ADHD·우울·번아웃 등 임상 상태

고객 해석:

- K: 시작·지속·정리 정도가 비교적 꾸준히 이어지는 편
- M: 시작·지속·정리 정도가 흥미·에너지·마감·환경 같은 상황에 더 영향을 받는 편

주의:

- K/M은 MBTI J/P가 아니다. 특히 M은 P의 유연성·선택지 개방을 뜻하지 않는다.
- SM-RL 제3 facet은 파일럿에서 채택 여부를 결정한다.

### 3.5 C/Q — 걱정과 감정 반응

측정한다:

- 좌절·갈등·예상 밖 문제에서 짜증과 동요가 커지는 경향
- 불확실성·평가·실패 가능성 앞에서 걱정이 반복되고 행동을 주저하는 경향

측정하지 않는다:

- 긍정 감정의 풍부함, 공감, 감수성
- 위험 탐지 정확도, 감정 조절 기술, 회복력
- 정신건강 상태나 불안·우울 장애 진단

고객 해석:

- C: 불편한 상황에서도 걱정과 감정이 급격히 커지는 일이 상대적으로 적은 편
- Q: 불편한 상황에서 걱정과 감정이 상대적으로 빨리 커지는 편

주의:

- C를 정신건강이 좋음, Q를 예민함·질환으로 설명하지 않는다.
- 감정 동요와 걱정·주저가 갈리면 두 반응을 따로 보여준다.

## 4. MBTI와의 위치 정합성

| 자리 | MBTI의 익숙한 주제 | 뉴앙                 | 관계                                                               | 금지되는 변환                              |
| ---- | ------------------ | -------------------- | ------------------------------------------------------------------ | ------------------------------------------ |
| 1    | E/I 에너지 방향    | E/I 사람 사이 에너지 | 방향이 가장 가까워 글자를 재사용                                   | MBTI E/I 결과를 뉴앙 E/I로 복사            |
| 2    | S/N 정보 수용      | R/N 생각과 탐색      | N의 가능성 방향은 유사, 뉴앙은 미적 경험·상상·지적 탐구를 포함     | MBTI S=N의 반대이므로 뉴앙 R이라고 변환    |
| 3    | T/F 판단 선호      | G/A 관계 주의        | 코드 위치만 활용하며, 뉴앙은 관계 상황에서 처음 보이는 대상만 측정 | T=G, F=A로 변환                            |
| 4    | J/P 외부 생활 선호 | K/M 일상 실행·구조   | J의 일부가 K와 겹칠 수 있으나 P와 M은 다름                         | J=K, P=M으로 변환                          |
| 5    | 대응 없음          | C/Q 불편 정서 반응성 | 뉴앙 고유 추가 영역                                                | MBTI 뒤에 보너스 글자만 붙인 검사라고 설명 |

뉴앙은 MBTI의 기억하기 쉬운 순서 문법을 UX 참고로 사용했지만, 독자 문항·연속 점수·세부 성향을 사용하는 별도 모델이다.

## 5. 영역 간 중복 감사

| 혼동 위험                            | 구분 질문                                                              | 판정                               |
| ------------------------------------ | ---------------------------------------------------------------------- | ---------------------------------- |
| SE-AI 주도적 표현 ↔ RO의 G           | 먼저 말을 꺼냈는가, 원인·해결이 먼저 눈에 들어왔는가                   | 구분 가능                          |
| SE-RE 교류 활력 ↔ ER 정서 반응       | 사람과 있을 때 활력이 올라오는가, 불편한 상황에서 걱정·감정이 커지는가 | 구분 가능                          |
| OE-IE 지적 탐구 ↔ SM-EP 실행·지속    | 새로운 원리를 알아보는 일을 즐기는가, 이미 정한 일을 이어가는가        | 구분 가능                          |
| OE-AE 미적 경험 ↔ ER-IR 감정 동요    | 미적 인상에 관심을 두는가, 좌절·갈등에서 불편 감정이 커지는가          | 구분 가능                          |
| RO-EC 공감적 관심 ↔ ER 감정 반응     | 상대의 감정에 관심을 두는가, 자신의 불편 정서가 쉽게 커지는가          | 구분 가능                          |
| RO-RN 선택 존중 ↔ SM-RL 맡은 일 이행 | 상대의 권리와 선택을 인정하는가, 맡은 일을 관리해 처리하는가           | 구분 가능, SM-RL 채택 시 추가 검증 |
| SM-EP 실행·지속 ↔ ER-WD 걱정·주저    | 평소 실행을 이어가는 정도인가, 걱정 때문에 행동이 멈추는가             | 구분 가능, 문항 상황 통제 필요     |

구분 가능하다는 판정은 정의 수준의 결과다. 실제 문항이 이 경계를 지키는지는 전문가 blind mapping과 파일럿 교차 적재로 다시 검증한다.

## 6. 코드 상태 규칙

모든 영역은 같은 네 상태 문법을 사용한다.

| 내부 상태               | 의미                                               | 고객 설명                             | 코드 처리                                    |
| ----------------------- | -------------------------------------------------- | ------------------------------------- | -------------------------------------------- |
| `CLEAR`                 | 세부 성향들이 같은 방향이고 불확실성이 낮음        | 해당 방향이 비교적 선명해요           | 점추정치 방향의 글자 제공                    |
| `SPLIT_PROFILE`         | 세부 성향 방향이 다르지만 응답 근거는 충분함       | 반응들이 다르게 나타나요              | 상위 점수 글자와 세부 차이를 함께 제공       |
| `NEAR_BOUNDARY`         | 상위 점수가 중심 부근이거나 신뢰구간이 양쪽에 걸침 | 두 방향이 비슷해요                    | 글자와 낮은 선명도 표시, 필요 시 적응형 문항 |
| `INSUFFICIENT_EVIDENCE` | 응답 부족 또는 품질 실패                           | 결과를 만들기 위한 응답이 더 필요해요 | 해당 글자 미발급                             |

`SPLIT_PROFILE`은 검사 실패가 아니다. `NEAR_BOUNDARY`와 구분해 저장하고 설명한다.

## 7. 빠른 검사와 정밀 검사의 계약

두 검사는 같은 구성개념 버전을 사용한다.

| 항목 | 빠른 코어                                | 정밀 코어                                          |
| ---- | ---------------------------------------- | -------------------------------------------------- |
| 목적 | 예비 5축 방향과 다음 행동 제공           | 세부 성향과 불확실성을 더 자세히 설명              |
| 문항 | 검증된 정보량 높은 문항의 축약 세트      | 각 세부 성향을 여러 상황에서 반복 측정             |
| 결과 | 예비 코드, 축 방향, 낮은 해상도의 설명   | 코드, 세부 성향, 분리 프로필, 선명도               |
| 금지 | 적은 문항으로 세부 성향 확정             | 많은 문항이라는 이유만으로 검증되지 않은 주장 제공 |
| 갱신 | 정밀검사 후 결과가 갱신될 수 있음을 안내 | 새 근거가 쌓이면 versioned update                  |

OE 3-facet을 유지하기 위해 빠른 코어 20문항 고정을 우선하지 않는다. 세부 성향을 빼기보다 문항 수를 늘리거나 빠른 결과의 설명 범위를 줄인다.

## 8. 고객 언어 공통 규칙

사용한다:

- `~하는 모습이 비교적 자주 나타날 수 있어요`
- `두 방향이 비슷해요`
- `반응들이 다르게 나타나요`
- `현재 결과`, `선명도`, `더 알아보기`

사용하지 않는다:

- `당신은 원래 ~한 사람입니다`
- `완벽한 유형`, `진짜 성격`, `과학적으로 증명된 코드`
- `더 좋은/나쁜 성향`, `상위/하위 유형`
- `현실형/창의형`, `착한 유형/냉정한 유형`, `성실형/게으른 유형`
- `정신적으로 건강/불안정`, 질병·진단 암시
- MBTI 글자와의 자동 변환표

## 9. 데이터 의미 계약

실제 구현에서는 최소한 아래를 버전으로 분리한다.

- `construct_definition_version`
- `facet_model_version`
- `item_bank_version`
- `scoring_model_version`
- `report_copy_version`
- `code_scheme_version`

결과에는 아래를 보존한다.

- 다섯 상위 영역의 연속 점수
- 각 세부 성향의 연속 점수
- 표준오차 또는 신뢰구간
- `CLEAR`, `SPLIT_PROFILE`, `NEAR_BOUNDARY`, `INSUFFICIENT_EVIDENCE`
- 응답 품질과 누락 정보
- 어떤 버전의 문항·채점·문구로 계산했는지

과거 결과를 새 코드 문자열로 단순 치환하지 않는다. 새 구성개념과 문항으로 재검사했는지, 호환 가능한 점수인지 구분한다.

## 10. 최종 위험과 차단책

| 위험                                  | 현재 차단책                               | 남은 검증                           |
| ------------------------------------- | ----------------------------------------- | ----------------------------------- |
| R을 현실 능력으로 오해                | 필수 한국어, Real-world 미노출, 금지 문구 | R/N 이해도 테스트, 필요 시 F/N 비교 |
| A를 더 착한 방향으로 선택             | 능력·도덕성 금지, 상황 강도 분산          | 사회적 바람직성·반응편향 검증       |
| M을 유연성 또는 게으름으로 오해       | 상황 영향으로 제한, J/P 분리              | 사례 분류·회상 테스트               |
| Q를 예민함·정신질환으로 오해          | 불편 정서 활성화로 제한                   | 낙인·안전 문구 사용자 조사          |
| 5글자가 세부 차이를 숨김              | facet 점수와 split 상태 동시 제공         | 결과 카드 이해도 테스트             |

| 연구 기반이라는 말이 검증 완료로 읽힘 | 공개 검증 상태와 버전 표시                | 신뢰 페이지 이해도 테스트           |

## 11. 최종 판정

| 감사 항목                  | 판정                           |
| -------------------------- | ------------------------------ |
| 다섯 자리 순서             | `PASS_AS_UX_HYPOTHESIS`        |
| 열 개 글자 중복            | `PASS`                         |
| 상위 영역의 개념 구분      | `PASS_AT_DEFINITION_LEVEL`     |
| 쉬운 한국어 연결           | `PASS_FOR_OWNER_REVIEW`        |
| MBTI 동일 검사 오해 차단   | `PASS_WITH_USER_TEST_REQUIRED` |
| 능력·도덕성·임상 과장 차단 | `PASS_AT_COPY_CONTRACT_LEVEL`  |
| 세부 성향·경계 상태 계약   | `PASS_AT_DESIGN_LEVEL`         |
| 문항·채점·통계 타당성      | `NOT_YET_VALIDATED`            |
| 운영 배포                  | `BLOCKED_UNTIL_VALIDATION`     |

결론:

> `E/I · R/N · G/A · K/M · C/Q`는 재기준화된 문항 설계 의미 계약으로 사용할 수 있다. 그러나 고객에게 신뢰도 높은 검사라고 제공하려면 M02 이후의 내용타당도·인지 인터뷰·정량 파일럿·신뢰도·타당도·공정성 검증을 통과해야 한다.

## 12. 사용자 승인 기록

2026-07-17 사용자 승인으로 코드 순서와 기본 의미 계약을 확정했고, 2026-07-18에는 RO 문항 감사 결과 G/A 의미를 v2로 재기준화했다.

1. 최종 작업 코드를 `E/I · R/N · G/A · K/M · C/Q`로 유지한다.
2. 코드는 기억용 요약이며 연속 점수·세부 성향·불확실성을 측정 원본으로 둔다.
3. 본 문서의 측정·금지·경계 규칙을 빠른·정밀 검사와 코드지도에 공통 적용한다.
4. 현재 상태를 `설계 근거 확보, 실제 측정 검증 전`으로 정직하게 공개한다.
5. G/A는 `원인·해결이 먼저 보임 ↔ 상대 마음이 먼저 보임`의 RO-EC 관계 주의 방향만 나타낸다.
6. RO-RN은 G/A에 합산하지 않는 본인 전용 상세 신호이며 공개·공유·비교에서 기본 제외한다.

이 승인으로 운영 코드나 DB를 바꾸지 않는다. 다음 단계는 M02 문항 사양·상황 라벨·응답 문법과 파일럿 프로토콜 설계다.
