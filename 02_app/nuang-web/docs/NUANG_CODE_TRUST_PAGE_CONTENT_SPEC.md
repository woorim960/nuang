## 0. 화면의 목적

이 페이지는 사용자가 코드 지도를 보다가 아래 질문에 답을 얻는 곳이다.

1. 뉴앙 코드는 무엇을 참고해 만들었는가?
2. MBTI를 그대로 사용하지 않은 이유는 무엇인가?
3. Big Five·BFAS·BFI-2를 어떻게 반영했는가?
4. 지금 믿어도 되는 범위와 아직 검증 중인 범위는 어디까지인가?
5. 앞으로 어떤 절차로 신뢰 근거를 쌓는가?

목표는 권위를 과시하는 것이 아니라 `근거와 한계를 함께 공개하는 태도`로 신뢰를 만드는 것이다.

## 1. 페이지가 지켜야 할 신뢰 계약

### 1.1 현재 공개 가능한 표현

- `성격 연구와 검사 개발 기준을 비교해 코드의 의미를 설계했어요.`
- `MBTI의 기억하기 쉬운 코드 방식과 Big Five 계열의 연속 성향 구조를 서로 다른 목적으로 참고했어요.`
- `뉴앙의 한국어 문항과 점수 기준은 별도의 사용자 검증이 필요해요.`
- `현재는 코드 의미를 정리했고, 문항과 점수는 검증을 준비하고 있어요.`
- `검사가 쌓이면 결과가 더 자세해지거나 달라질 수 있어요.`

### 1.2 현재 공개하면 안 되는 표현

- `과학적으로 증명된 뉴앙 코드`
- `정확도 00%`, `신뢰도 검증 완료`, `전문가 인증 완료`
- `Big Five 기반이므로 이미 타당한 검사`
- `MBTI의 단점을 완전히 해결한 검사`
- `당신의 진짜 성격`, `평생 변하지 않는 유형`
- `C는 정신적으로 건강하고 Q는 불안정하다`와 같은 가치·진단 표현

참고 이론의 근거와 `뉴앙 자체 문항의 근거`를 한 문장 안에서 섞지 않는다.

## 2. 코드 지도에서 들어오는 방식

### 2.1 진입 카드

위치:

- 코드 자리와 세부 신호를 한 번 본 뒤
- 32개 코드 탐색 목록이 시작되기 전
- 사용자의 탐색을 끊지 않도록 화면 최상단에는 배치하지 않음

카드 문구:

```text
뉴앙 코드는 어떻게 만들었나요?
참고한 연구와 현재 검증 단계를 확인해요.

[근거 살펴보기]
```

보조 표시:

```text
최근 검토 2026.07
```

금지:

- `100% 신뢰`, `과학 인증` 배지
- 논문 수를 품질 점수처럼 과시하는 문구
- 캐릭터에게 흰 가운·청진기·인증 메달을 주는 표현

### 2.2 돌아오기

- 상단 왼쪽: `코드 지도로`
- 페이지 하단 고정 CTA 없음
- 마지막 본 코드 지도 스크롤 위치를 복원
- 외부 원문 링크를 열어도 앱의 현재 위치를 잃지 않음

## 3. 정보 구조

```text
코드 지도 진입 카드
→ 01 한눈에 보는 결론
→ 02 무엇을 참고했나요?
→ 03 무엇을 보완했나요?
→ 04 신뢰는 어떻게 확인하나요?
→ 05 지금 어디까지 왔나요?
→ 06 뉴앙이 말하지 않는 것
→ 07 연구 원문 보기
→ 버전·최근 검토일

```

처음 네 섹션만 읽어도 핵심을 이해할 수 있어야 한다. 연구 원문과 전문 용어는 접힌 상세 영역에서 선택적으로 제공한다.

## 4. 섹션별 최종 카피

### 4.1 01 한눈에 보는 결론

상단 eyebrow:

```text
뉴앙 코드의 설계 원칙
```

제목:

```text
한 가지 검사를 그대로 복사하지 않았어요
```

본문:

```text
MBTI의 기억하기 쉬운 코드 방식,
Big Five의 연속적인 성향 구조,
BFAS·BFI-2의 세부 성향 연구를 비교했어요.

그 위에 뉴앙만의 다섯 자리 의미와 한국어 문항을 새로 설계하고,
실제 사용자에게 맞는지는 별도로 확인합니다.
```

핵심 도식:

```text
기억하기 쉬운 코드
        +
연속 점수와 세부 신호
        +
한국어 사용자 검증
        =
뉴앙이 만들고 있는 성향 이해 방식
```

현재 상태 카드:

```text
현재 단계
코드 의미 설계 완료 · 문항과 점수 검증 전

연구를 참고했다는 사실과 뉴앙 검사가 검증됐다는 것은 달라요.
검증이 끝난 범위만 결과에 사용하고, 상태와 버전을 계속 공개할게요.
```

상태 표시에는 아이콘·텍스트·색을 함께 사용한다. 보라색 하나로 완료와 진행 중을 모두 표현하지 않는다.

### 4.2 02 무엇을 참고했나요?

섹션 제목:

```text
각 연구에서 잘하는 점을 따로 배웠어요
```

#### 카드 A — MBTI

```text
MBTI
쉽게 기억하고 대화하는 방법

참고한 점
네 글자로 차이를 빠르게 기억하고, 어느 쪽도 더 좋거나 나쁘지 않게 설명하는 방식

뉴앙의 적용
사람 → 생각 → 관계 → 일상으로 이어지는 익숙한 읽기 흐름을 참고했어요.
하지만 글자만으로 사람을 고정하지 않고 점수·선명도·세부 신호를 함께 봐요.
걱정과 감정 반응을 이해하는 다섯 번째 자리도 더했어요.
```

하단 안내:

```text
같은 E/I나 N이 보여도 MBTI 결과를 뉴앙 코드로 바꿀 수는 없어요.
```

#### 카드 B — Big Five

```text
Big Five
성향을 연속선으로 보는 방법

참고한 점
사람을 둘 중 하나로만 자르지 않고, 다섯 개의 넓은 성향을 정도의 차이로 이해하는 구조

뉴앙의 적용
각 코드 자리의 원본은 글자가 아니라 연속 점수와 현재의 선명도예요.
두 방향이 비슷하거나 세부 반응이 갈리면 그 차이를 숨기지 않아요.
```

#### 카드 C — BFAS·BFI-2

```text
BFAS · BFI-2
넓은 성향을 더 자세히 보는 방법

참고한 점
같은 넓은 성향 안에도 서로 다른 세부 모습이 함께 존재할 수 있다는 구조

뉴앙의 적용
예를 들어 사람 사이의 에너지는 함께할 때의 활력과 먼저 표현하는 모습으로,
생각과 탐색은 미적 경험·상상 확장·지적 탐구로 나누어 살펴봐요.

연구의 넓은 영역을 그대로 코드에 복사하지도 않아요.
G/A는 관계에서 원인·해결과 상대 마음 중 무엇이 먼저 보이는지만 나타내고,
선택과 경계를 존중하는 반응은 본인에게 보여주는 별도 상세 신호로 확인해요.
```

#### 카드 D — HEXACO·IPIP

```text
HEXACO · IPIP
다른 설명 방식과 문항 자료까지 비교

참고한 점
Big Five 외의 대안 구조와 공개 문항 자료를 함께 검토했어요.

뉴앙의 적용
HEXACO의 정직·겸손은 도덕 등급으로 오해될 위험이 있어 공개 코어에 넣지 않았어요.
IPIP는 문항 아이디어와 권리 확인 자료로만 참고하고, 번역 문항을 그대로 사용하지 않아요.
```

카드의 기본 화면에는 위 요약만 노출한다. 연구명 옆 `자세히` 버튼을 누르면 공식 설명·원 연구·뉴앙의 채택/수정/제외 판단을 펼친다.

### 4.3 03 무엇을 보완했나요?

섹션 제목:

```text
쉽게 이해하되, 단순하게 단정하지 않아요
```

| 기존 방식만 사용할 때 생길 수 있는 아쉬움                     | 뉴앙의 보완 원칙                                                |
| ------------------------------------------------------------- | --------------------------------------------------------------- |
| 한 글자가 경계에 가까운 차이를 숨길 수 있음                   | 글자와 함께 연속 점수·선명도·두 방향이 비슷한 상태를 제공       |
| 넓은 성향 하나가 서로 다른 생활 모습을 뭉갤 수 있음           | 코드 아래의 세부 신호를 따로 보여주고 서로 갈리는 모습도 설명   |
| 짧은 검사 결과가 지나치게 확정적으로 들릴 수 있음             | 빠른 결과는 예비 결과로 제한하고 정밀검사 후 갱신 가능성을 안내 |
| 전문 용어나 가치가 섞인 이름이 낙인을 만들 수 있음            | 능력·도덕·진단이 아닌 중립적인 일상 언어 사용                   |
| 영어권 문항을 번역하는 것만으로 문화 적합성을 보장하기 어려움 | 한국어 문항을 새로 작성하고 인지 인터뷰·파일럿으로 별도 검증    |
| 유명 이론을 참고했다는 사실이 자체 검증처럼 보일 수 있음      | 참고 근거와 뉴앙 자체 검증 상태를 화면에서 분리해 공개          |

모바일에서는 표를 좌우 스크롤시키지 않는다. 각 항목을 `아쉬움 → 뉴앙의 보완` 2단 카드로 변환한다.

### 4.4 04 신뢰는 어떻게 확인하나요?

섹션 제목:

```text
좋은 설명이 아니라, 실제 근거로 확인해요
```

도입:

```text
연구 이름을 붙이는 것만으로 검사가 완성되지는 않아요.
뉴앙의 한국어 문항과 결과가 실제로 의도한 성향을 설명하는지 다음 순서로 확인합니다.
```

검증 흐름:

1. `의미 정하기` — 각 코드 자리와 세부 신호가 무엇을 포함하고 제외하는지 정함
2. `문항 만들기` — 다양한 일상 상황에서 한 번에 하나의 반응만 묻는 후보를 만듦
3. `전문가 확인` — 문항이 의도한 성향에 정확히 연결되는지 독립적으로 검토
4. `사용자와 읽어보기` — 2030 한국어 사용자가 문장을 같은 뜻으로 이해하는지 인터뷰
5. `파일럿 검사` — 충분한 응답으로 문항 구조·신뢰도·측정오차를 분석
6. `공정성 확인` — 집단에 따라 같은 문항이 다르게 작동하지 않는지 확인
7. `버전 공개` — 통과한 문항·점수·설명만 버전과 함께 적용

단계 카드의 도움말:

- `전문가 확인`은 전문가가 결과를 인증한다는 뜻이 아니라 내용타당도를 독립적으로 점검한다는 뜻이다.
- `신뢰도`는 결과가 항상 옳다는 뜻이 아니라 측정의 일관성과 오차 범위를 평가하는 한 근거다.
- `공정성 확인`은 모든 차이가 사라진다는 뜻이 아니라, 측정하려는 성향 외의 집단 차이가 결과를 왜곡하는지 점검하는 과정이다.

### 4.5 05 지금 어디까지 왔나요?

섹션 제목:

```text
현재 검증 상태를 숨기지 않을게요
```

초기 공개 상태:

| 단계                    | 고객 표시      | 상태        |
| ----------------------- | -------------- | ----------- |
| 연구 비교·영역 범위     | 참고 근거 정리 | `완료`      |
| 5글자 코드 의미         | 코드 의미 설계 | `완료`      |
| 한국어 문항 사양        | 문항 설계      | `다음 단계` |
| 전문가·사용자 이해 검토 | 문항 이해 확인 | `시작 전`   |
| 정량 파일럿             | 점수 구조 확인 | `시작 전`   |
| 신뢰도·타당도·공정성    | 결과 근거 확인 | `시작 전`   |

설명:

```text
그래서 지금은 ‘연구 근거를 바탕으로 설계하고 검증 중인 코드’라고 안내해요.
아직 ‘정확성이 입증된 검사’라고 말하지 않습니다.
```

운영 구현에서는 이 상태를 정적 카피에 하드코딩하지 않는다. 승인된 evidence release에서 읽고, 각 상태에 `검토일`, `근거 문서 버전`, `책임 승인 상태`를 연결한다.

검증 완료 뒤에도 단순히 모든 배지를 `완료`로 바꾸지 않는다. 사용 목적별 근거를 구분한다.

- 자기이해 리포트
- 코드 발급
- 관계 비교
- 커뮤니티 공개 요약
- 빠른 검사와 정밀 검사

### 4.6 06 뉴앙이 말하지 않는 것

섹션 제목:

```text
코드가 말할 수 없는 것도 분명해요
```

카피:

```text
뉴앙 코드는 능력, 도덕성, 정신건강을 평가하지 않아요.
누군가를 완벽히 정의하거나 관계의 성공을 예언하지도 않아요.

현재까지 관찰된 반복 경향을 이해하고,
나와 원하는 사람들의 공통점과 차이를 대화하기 위한 출발점이에요.
```

보조 목록:

- 의료·심리 진단이 아님
- 취업·선발·평가에 사용하지 않음
- 더 좋은 글자와 더 나쁜 글자가 없음
- 같은 코드라도 세부 신호와 생활 맥락이 다를 수 있음
- 새 검사가 쌓이면 현재 결과가 더 자세해지거나 갱신될 수 있음

### 4.7 07 연구 원문 보기

섹션 제목:

```text
참고한 자료를 직접 확인할 수 있어요
```

기본은 모두 접힌 아코디언이다. 각 자료에는 `무엇을 확인할 수 있는지`를 한 줄로 먼저 설명한다.

#### MBTI 공식 설명·비교 연구

## 5. 시각 디자인 방향

### 5.1 전체 인상

- 전문적이지만 딱딱하지 않은 `제품 안의 공개 연구 노트`
- 따뜻한 무채색 canvas, cream card, deep ink text
- 보라는 제목·상태 강조·성향 라인에만 제한적으로 사용
- 카드마다 다른 파스텔 배경을 무작위로 배치하지 않음
- 그림보다 정보 구조를 우선하고, 캐릭터는 한 섹션에 한 번 이하로 사용

### 5.2 대표 비주얼

뉴앙 캐릭터가 다섯 개의 얇은 카드 조각을 하나의 연결선 위에 올려두는 장면을 사용한다.

- 카드 조각: `기억`, `점수`, `세부`, `검증`, `버전`
- 실험실·뇌 이미지·DNA·수식·인증 방패를 사용하지 않음
- AI 생성 이미지 특유의 부유하는 장식물 대신 편집 그리드에 맞춘 단순한 소품 사용
- 이미지 안에 설명 문구를 넣지 않고, 접근 가능한 HTML 텍스트로 제공

온보딩 가이드처럼 이미지 자체가 메시지인 화면이 아니므로, 이 페이지는 텍스트 접근성과 업데이트 가능성을 우선한다.

### 5.3 모션

- 진입 시 다섯 카드 조각이 40~60ms 간격으로 연결선에 놓임
- 검증 흐름은 사용자가 해당 섹션에 도달했을 때 선이 한 번 이어짐
- 상태 배지는 튀거나 반짝이지 않음
- 아코디언은 180~240ms 높이·opacity 전환
- `prefers-reduced-motion`에서는 즉시 상태 전환
- 논문 수, 정확도, 완료도를 게임 점수처럼 올리는 카운터 모션 금지

## 6. 모바일 레이아웃

- 기준 폭: 360 / 390 / 430px, 콘텐츠 최대 폭 520px
- 상단 safe area와 하단 브라우저 영역 고려
- 전체 한 열 구성
- 본문 16px 이상, 주석 13px 이상
- 본문 한 줄 길이 약 24~32자
- 비교표는 카드로 변환하고 가로 스크롤 금지
- 아코디언 터치 영역 최소 44×44px
- 외부 링크에는 `새 화면에서 열림`을 스크린리더 라벨로 제공
- 긴 연구명은 두 줄 허용, 말줄임표로 핵심 정보를 숨기지 않음

## 7. 상태·데이터 모델

화면의 연구·검증 상태를 앱 카피와 분리한다.

## 9. 사용자 검증 시나리오

5~8명씩 두 차례의 이해도 테스트를 권장한다.

### 과제

1. 페이지를 읽고 뉴앙이 MBTI와 같은 검사인지 설명한다.
2. 뉴앙이 Big Five를 어떻게 반영했는지 설명한다.
3. 현재 뉴앙 검증이 끝났는지 판단하고 근거를 찾는다.
4. 코드가 말할 수 없는 것을 두 가지 말한다.
5. 관심 있는 연구의 공식 원문을 찾는다.

### 성공 기준

- 80% 이상이 `MBTI를 그대로 복사한 검사가 아님`을 설명
- 80% 이상이 `글자 아래 연속 점수와 세부 신호가 있음`을 설명
- 90% 이상이 현재 상태를 `검증 완료`로 오해하지 않음
- 80% 이상이 의료·능력·도덕 평가가 아님을 한 가지 이상 회상
- 80% 이상이 30초 안에 검증 상태를 찾음
- 연구명 미인지 사용자도 핵심 설명을 이해

오해가 발생하면 면책 문구를 추가하기보다 정보 순서와 핵심 문장을 먼저 수정한다.

## 10. 접근성·윤리 점검

- 완료·진행 상태를 색만으로 구분하지 않음
- 아코디언은 실제 `button`과 `aria-expanded` 사용
- 외부 링크는 출처명과 목적이 링크 텍스트에 포함됨
- 영어 연구명에 짧은 한국어 설명을 항상 동반
- `낮은 점수`, `나쁜 성향`, `정상`, `비정상` 사용 금지
- MBTI 상표와 BFI·BFI-2·IPIP의 이용 조건·출처를 별도 법무·권리 검토
- 연구 참여나 사용자 응답을 향후 근거로 활용할 경우 별도 동의·비식별화·보관 정책 적용
