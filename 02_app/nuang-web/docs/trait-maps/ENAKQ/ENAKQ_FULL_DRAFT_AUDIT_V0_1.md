상태: `INTERNAL_AUDIT_COMPLETE_EXTERNAL_REVIEW_PENDING`

## 1. 감사 결론

ENAKQ 전체 15개 장의 연구 초안과 문단별 근거 원장이 완성됐다. 공백 제외 55,513자로 프로토콜의 허용 범위 45,000–60,000자 안에 있으며, 106개 문단 블록과 158개 고유 claim을 포함한다.

내부 자동·수동 감사에서 아래 항목을 확인했다.

- 원문 claim과 각 Part 근거 원장의 매핑 누락·여분 0건
- 중복 block ID 0건
- Part 사이 동일 문장 재사용 0건
- 과거 코드 문자 S/T·C/V·F/O·D/A·P/E 사용 0건
- 사용자 금지 표현 `리듬`, `마음 먼저`, `정답 없는`, `일상 질문`, `연구 중인`, `베타` 0건
- 동일 claim의 근거 상태 충돌 0건
- `APPROVED`로 잘못 승격된 사용자 claim 0건

이 감사는 에이전트 내부 검토다. 외부 자격 전문가 검토, 사용자 인지 인터뷰, 한국어 정량 파일럿을 대신하지 않는다. 현재 원문은 운영 확정 문구가 아니라 연구 초안이다.

## 4. 구성개념 충돌 감사

| 영역 | 감사 질문 | 판정 | 남은 위험 |
|---|---|---|---|
| E | 교류 활력·표현 시작을 사교 능력·인기·리더십으로 확대했는가 | 통과 | SE 두 facet 구조 검증 필요 |
| N | 가능성·관점·원리 탐색을 지능·창의 능력·현실성으로 확대했는가 | 통과 | OE 3-facet 구조와 행동 준거 필요 |
| A | 상대 마음으로 향하는 관계 주의를 공감·착함·자기희생으로 확대했는가 | 통과 | Agreeableness 전체와의 판별 필요 |
| K | 시작·지속·정리를 책임감·생산성·성과·완벽주의로 확대했는가 | 통과 | 2-vs-3 facet과 환경 오염 검증 필요 |
| Q | 걱정·감정 활성화를 진단·감수성·위험 탐지·조절·회복으로 확대했는가 | 통과 | ER 구조·최근 상태·과정 분리 필요 |

Part 5의 강점 장도 위 범위를 넘지 않는다. 각 글자의 장점은 능력 명사가 아니라 관찰 가능한 후보 행동으로 쓰였고, 도움이 되는 조건·반대 사례·과사용 가능성·확인 기준이 함께 제공됐다. Q에는 긍정적 재능을 억지로 배정하지 않았다.

## 5. 생각·행동·회복 경계 감사

### 처음 드는 생각과 실제 나타나는 반응

- G/A만 내부 process pair 후보가 있으며 개인 과정 자료가 있을 때만 A→A·A→G·G→A·G→G를 제공한다.
- E→I, N→R, K→M, Q→C는 직접 과정 문항과 반복 자료가 없어 발급하지 않는다.
- 대표 코드, 표정, 말투, 게시물로 속마음과 실제 반응을 추정하는 문장을 차단했다.
- 공개 비교에서 상대의 직접 응답·원점수·과정 자료를 추정하거나 노출하지 않는다.

### 반응성과 회복

- Q는 불편 정서가 활성화되는 경향이며 회복 속도·조절 기술·지원 선호가 아니다.
- C를 높은 회복력·정신건강·침착 능력으로 설명하지 않는다.
- 감정, 걱정, 행동, 생활 기능이 서로 다른 속도로 달라질 수 있게 설명했다.
- 개인 회복·지원 claim은 `HOLD`로 유지했다.

판정: **경계 충돌 없음.** 과정 코드와 회복 개인화는 후속 측정·데이터 계약 없이는 구현하면 안 된다.

| A×Q·N×A×Q → 애착 불안·집착·질투 | 조합은 `HOLD`, 애착·사회불안·호감 강도 추정 금지 |

## 7. 중복과 canonical 정보 구조

동일 문장은 없지만 핵심 경계 claim은 여러 장에 반복 연결된다. 이는 연구 원문의 요약·상세·상황별 역할을 추적하기 위한 것이다. 운영 UI에서는 아래 canonical 위치를 따른다.

| 정보 종류 | Canonical 원문 | 다른 화면의 역할 |
|---|---|---|
| 다섯 자리 정의·facet | Part 1, Section 3 | 한 문장 요약 또는 해당 자리 링크 |
| 역할명 의미·한계 | Part 1, Section 2 | 역할명 옆 짧은 경계만 표시 |
| 2자·3자 조합 가설 | Part 2, Section 4 | 검증된 개인 입력이 있을 때만 상황 카드에 연결 |
| 처음 생각·실제 반응 | Part 2, Section 5 | 본인 상세 전용, 공개 비교는 승인된 요약만 |
| 일상·관계·업무 장면 | Part 2–4 해당 Section | 해당 상황 진입 시에만 노출 |
| 강점·과사용·실험 | Part 5, Section 13 | 동일 claim 재출력 없이 사용자 사례와 연결 |
| 오해·대화 질문 | Part 5, Section 14 | 비교·커뮤니티에서 필요한 질문만 한 번 표시 |
| 근거·검증·한계 | Part 5, Section 15 | 요약 배지→근거 상세 페이지로 단계적 공개 |

가장 자주 연결된 claim은 `boundary.variability` 13회, `general.q.activation`·`general.q.expression`·`stress.recovery` 각 7회, `general.role.conductor`·`process.non_inference` 각 6회다. 장문 연구 문서에서는 맥락별 깊이가 다르므로 유지하지만, 제품 렌더링에서는 같은 `contentKey`를 한 뷰에 한 번만 허용한다.

## 8. 근거·문화·개인정보 감사

### 근거 투명성

- 외부 연구, 뉴앙 매핑, 제품 문구 검증을 분리했다.
- 표본·측정·문화·연구 설계의 제한을 근거마다 기록했다.
- 외부 Big Five 연구를 뉴앙 한국어 문항의 타당화로 표현하지 않았다.
- 자기일치감과 만족도를 정확성의 증거로 사용하지 않았다.

### 문화와 규준

- 특정 가족 형태, 성 역할, 연애 방식, 연락 횟수, 직업을 정상 기준으로 만들지 않았다.
- 한국어·문화 적응과 측정동일성 검증 전 외국 규준을 사용하지 않는다.
- 앱 이용자 분포를 한국인 전체 분포로 부르지 않는다.

### 개인정보와 AI

- 본인·공개·비교·연구 정보의 사용 목적을 분리했다.
- 직접 응답·원점수·과정·최근 상태·위기 정보는 기본 비공개다.
- 관계가 가깝거나 마음 가는 상대라는 이유로 접근권을 넓히지 않는다.
- 게시물·사진·활동 기록으로 성향·호감·정신건강을 AI 추정하지 않는다.

판정: 연구 원문의 정책 경계는 일관된다. 실제 앱의 RLS·동의 철회·삭제·AI 입력 계약은 별도 구현 테스트가 필요하다.
